# 代码审查记录

本文记录 AI 多页课件生成项目首轮结构搭建后的代码审查结论。

## 2026-04-16

### 审查范围

- `App.xaml` 与 `App.xaml.cs`
- `MainWindow.xaml` 与 `MainWindow.xaml.cs`
- `Models` 目录
- `Services` 目录
- `ViewModels` 目录

### 审查结论

本轮实现已经完成从 WPF 空模板到“可生成逐页提示词、支持单页确认图片生成、支持导出已选图片”的基础结构搭建，代码能够成功构建。

### 已确认的设计点

- 使用 `MainViewModel + SlidePageViewModel + Service` 组织主流程。
- 使用 `Microsoft Agent Framework` 调用 `ep-20260306101224-c8mtg` 生成逐页图片提示词。
- 使用独立图片生成服务调用 `ep-20260120102721-c4pxb` 生成候选图片。
- 图片生成前保留用户确认步骤，只有单页点击按钮后才触发图片模型。
- 导出只依赖每页当前已选图片，并按时间命名子目录输出。

### 当前约束

- 豆包 API Key 通过环境变量 `DOUBAO_API_KEY` 读取。
- 图片生成接口按 OpenAI 兼容路径 `/images/generations` 调用。
- 导出目录当前采用手工输入路径的方式预设。

### 审查补充：结构重构后新增确认点

- 逐页拆分不再依赖按行或按空行规则，而是由 `PromptAgentService` 将完整大纲交给 Agent 处理。
- Prompt Agent 通过函数工具回传“一个页面一个字符串”的提示词列表，并在工具调用后终止对话，避免再将工具结果回灌给模型。
- 页面列表默认可见，用户可以直接新增页面、删除页面和编辑提示词。
- 服务配置改为由界面输入，不再依赖环境变量读取 API Key。
- 图片生成链路改为使用 OpenAI SDK 的图片生成客户端，避免继续维护手写 DTO 与请求拼接代码。
- `MainWindow.xaml` 已补充设计时 `d:DataContext` 与 `d:SampleData`，设计器可以推断主要绑定关系。

### 后续可继续优化项

- 增加导出目录选择器。
- 增加基于当前页面列表的批量重新生成能力。
- 增加更细粒度的取消操作与超时控制。
- 根据真实接口返回结果补充更细的错误处理与日志记录。
