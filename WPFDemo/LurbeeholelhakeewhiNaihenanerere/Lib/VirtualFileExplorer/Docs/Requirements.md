# VirtualFileExplorer 需求分析

## 背景

当前项目已经具备虚拟文件、虚拟文件夹和虚拟文件管理器的基础类型，但缺少可以直接使用的文件管理器界面、完整的文件系统操作抽象以及物理文件系统验证实现。目标是把 `VirtualFileExplorer` 打造成一个可复用的 WPF 文件管理器库。

## 核心目标

1. 提供一个开箱即用的 `FileExplorerUserControl`。
2. 支持从虚拟文件提供器获取文件和文件夹列表。
3. 支持面包屑导航、返回上级、刷新和目录切换。
4. 支持至少两套显示方式：平铺模式、详细列表模式。
5. 支持文件和文件夹的重命名、移动、复制、删除。
6. 允许这些能力通过 `VirtualFileManager` 统一提供，也允许作为虚拟文件或虚拟文件夹的可选能力对外暴露。
7. 允许 `FileExplorerUserControl` 内部部件被替换，例如项模板、工具区模板、空状态模板。
8. 提供默认样式，使库在没有额外资源的情况下也能直接使用。
9. 补全 `PhysicalFileManager`，并通过示例应用做真实文件系统验证。
10. 补充文档，包括经验、需求、设计、实现回顾。

## 功能性需求

### 1. 虚拟文件系统抽象

- `VirtualFileManager` 需要能枚举指定文件夹下的文件和文件夹。
- `VirtualFileManager` 需要能返回根目录或初始目录。
- `VirtualFileManager` 需要支持新建文件夹、重命名、复制、移动、删除等操作。
- 操作目标既可以是文件，也可以是文件夹。
- 若某个实现不支持某项操作，应能显式表达能力状态，而不是让界面猜测。

### 2. 文件与文件夹模型

- 文件对象至少要包含：唯一标识、名称、所属目录、扩展名、大小、创建时间、修改时间、访问时间。
- 文件夹对象至少要包含：唯一标识、名称、上级目录、创建时间、修改时间。
- 文件和文件夹对象都需要声明是否允许重命名、复制、移动、删除。
- 文件夹对象需要支持生成面包屑链路。

### 3. 界面能力

- 控件初始化后能从 `FileExplorerUserControl.xaml.cs` 中暴露的 `FileManager` 获取数据。
- 支持展示当前目录下的文件夹和文件。
- 支持双击或命令进入文件夹。
- 支持点击面包屑跳转到上级目录。
- 支持切换为平铺视图和详细列表视图。
- 支持选中单项，并对选中项执行文件操作。
- 支持输入新名称完成重命名。
- 支持通过目标目录完成复制和移动。
- 支持删除文件或文件夹。

### 4. 可定制性

- 控件应允许宿主替换项目模板。
- 控件应允许宿主替换工具栏区域。
- 控件应允许宿主替换空状态展示。
- 默认样式不依赖宿主额外定义资源即可工作。

### 5. 物理文件系统验证

- `PhysicalFileManager` 需要映射实际磁盘目录。
- 示例程序需要构建演示目录，并把控件绑定到物理文件管理器。
- 演示场景至少覆盖浏览、切换显示模式、重命名、复制、移动、删除。

## 非功能性需求

- 保持库代码与示例代码分离。
- 遵循 MVVM，避免把业务逻辑堆在控件代码后置中。
- 保持现有项目结构风格，做最小但完整的扩展。
- 优先使用 .NET 6/WPF 可稳定支持的语法和能力。
- 对失败操作返回明确异常，避免静默吞掉错误。

## 约束与边界

- 本轮实现优先支持单选操作，不强制实现多选批处理。
- 本轮实现以同步文件系统操作为主，界面层保留后续异步扩展空间。
- 本轮不引入额外第三方 MVVM 框架，沿用项目现有轻量基础设施。
- 本轮暂不实现文件内容预览和拖放排序。

## 验收标准

- 示例程序启动后可直接浏览演示目录。
- 面包屑导航、返回上级、刷新可用。
- 平铺模式和详细列表模式都能显示同一批数据。
- 对文件和文件夹执行重命名、复制、移动、删除时，目录列表会同步刷新。
- 宿主可通过模板属性替换默认项模板或空状态模板。
- 项目能够成功构建。
