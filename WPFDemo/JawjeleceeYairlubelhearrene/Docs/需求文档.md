# 需求文档

## 1 项目目标
将 WPF 程序 `JawjeleceeYairlubelhearrene` 实现为一个 PPT 讲稿视频生成工具。程序需要读取用户输入的 PowerPoint 文件，提取每页文本和页面截图，再结合 OpenAI 兼容接口与 OpenSpeech 语音合成能力，为每页生成讲稿并合成完整讲稿视频。

## 2 用户场景
* 用户选择或拖入一个 PPT 文件；
* 程序读取每页文本内容；
* 程序通过 PowerPoint COM 导出每页截图；
* 程序调用大模型为每页生成中文讲稿；
* 程序调用 OpenSpeech 合成逐页语音；
* 程序使用 FFmpeg 合成最终视频；
* 用户在界面中查看页面内容、生成日志与最终视频，并可直接播放生成结果。

## 3 功能需求

### 3.1 PPT 输入
* 支持手工输入 PPT 文件路径；
* 支持按钮选择 PPT 文件；
* 支持将 PPT 文件拖入窗口；
* 支持常见 PowerPoint 文件扩展名，如 `ppt`、`pptx`、`pptm`、`ppsx`、`ppsm`、`potx`、`potm`。

### 3.2 PPT 解析
* 使用 OpenXML 读取每页文本；
* 尽量提取标题、普通文本框、表格、备注信息；
* 使用 COM 调用 Microsoft PowerPoint 导出每页截图；
* 若 PowerPoint COM 不可用，需要明确提示用户设备未正确安装或不可调用 PowerPoint。

### 3.3 配置输入
界面中需要提供以下配置项：

|配置项 |说明 |默认值来源 |
|---|---|---|
|OpenSpeech API Key |语音合成鉴权 |本地文件读取 |
|resourceId |OpenSpeech 资源标识 |默认 `seed-tts-2.0` |
|speaker |演讲者音色 |默认 `zh_female_vv_uranus_bigtts` |
|OpenAI API Key |OpenAI 兼容接口鉴权 |本地文件读取 |
|OpenAI EndPoint |OpenAI 兼容接口地址 |参考现有控制台程序 |
|OpenAI 模型 |聊天模型名称 |参考现有控制台程序 |
|FFmpeg 路径 |视频合成执行文件 |本地路径读取 |
|输出目录 |生成视频输出路径 |程序默认输出目录 |

### 3.4 默认值和本地文件
* 默认值参考 `KadefihalldokaiChairwedone` 与 `JiyinunalcheWaqerehoqarlijear` 的现有实现；
* 读取本地文件时，如路径不存在或文件内容为空，不抛出初始化异常，界面保留为空；
* 当前默认参考路径包括但不限于：
  * `C:\lindexi\Work\Doubao.txt`
  * `C:\lindexi\Work\Key\OpenSpeech API Key.txt`
  * `C:\lindexi\Application\ffmpeg.exe`

### 3.5 Speaker 下拉框
* Speaker 使用下拉框选择；
* 数据来源为 `Docs\VolcEngineSdk\在线音色列表.txt`；
* 只取“豆包语音合成模型2.0”音色列表中：
  * 通用场景；
  * 支持“情感变化、指令遵循、ASMR”；
* 默认选择 `zh_female_vv_uranus_bigtts`。

### 3.6 视频生成
* 根据页面文本和截图生成逐页讲稿；
* 根据讲稿生成逐页语音；
* 使用 FFmpeg 将页面截图与语音片段合成视频；
* 生成完成后在界面中显示最终视频，并支持播放、暂停、停止；
* 支持打开输出目录。

### 3.7 预览与日志
* 界面中展示每页截图；
* 展示每页提取文本；
* 展示每页生成的讲稿；
* 展示执行状态与日志信息。

## 4 非功能需求
* 采用 MVVM 结构；
* 界面风格统一、美观、现代；
* 控件样式完整，包含下拉框样式；
* 对外部依赖失败给出明确提示；
* 保持代码修改最小化并可编译通过。

## 5 异常处理要求
* PPT 文件不存在时，阻止执行并提示；
* PowerPoint COM 导出失败时，提示需要安装或修复 Microsoft PowerPoint；
* FFmpeg 路径无效时，阻止执行并提示；
* API Key、模型、EndPoint 等关键配置为空时，阻止执行并提示。
