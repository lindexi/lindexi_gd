## 一、C1 裁决修订：旋转保存对话框

### 1.1 裁决结论

**采纳 UI 立场：v1.0 不设计「是否保存旋转结果」对话框。**

旋转行为严格遵循需求 v1.1 §4.3 的"仅视图旋转，不修改原文件；关闭即丢失；不弹出任何形式确认对话框"。

### 1.2 修订动作清单

| # | 修订点（涉及文档） | 修订前 | 修订后 |
|---|---|---|---|
| 1 | UX 05 低保真线框图 S4 旋转后的关闭场景 | 包含「是否保存」对话框草图 | **删除该对话框草图**，关闭路径改为"直接关闭，无任何中间态" |
| 2 | UX 02 信息架构 §4 关闭流程段落 | "关闭时若有旋转状态，询问是否保存" | **删除该段落**，改为"关闭即丢失旋转状态，下次打开恢复原方向（与 v1.1 §4.3 一致）" |
| 3 | UX 03 交互流程图 M3 模块的 Close 节点 | 分支含"有旋转 → 弹确认对话框" | **删除该分支**，改写为节点直连 Exit，附注释："v1.0 不设计保存对话框" |

### 1.3 一致性验证

| 比对项 | 结果 |
|---|---|
| 需求 v1.1 §4.3「仅视图旋转」 | ✅ |
| 需求 v1.1 §2.2「不包含编辑/导出」 | ✅ |
| UI 微调 M2 / M4（仅装饰、无存储） | ✅ |
| 测试风险 R3（无二次确认对话框） | ✅ |
| 需求 AC-05 / AC-13 | ✅ |

## 二、C2 裁决修订：Ctrl+S / Ctrl+Shift+S 快捷键

### 2.1 裁决结论

**v1.0 不提供 `Ctrl+S`（保存）和 `Ctrl+Shift+S`（另存为）快捷键。**

依据 v1.1 §2.2 明确排除"编辑/导出"，且与 C1 同源——若不设计保存对话框，则保存/另存为快捷键没有触发对象，必须同步移除避免"死键"。

### 2.2 修订动作清单

| # | 修订点（涉及文档） | 修订前 | 修订后 |
|---|---|---|---|
| 1 | UX 04 §2.1 文件操作表 | 含 `Ctrl+S` 和 `Ctrl+Shift+S` 两行 | **删除这两行**，仅保留 `Ctrl+O` / `Ctrl+Q` / `Alt+F4` |
| 2 | UX 04 §3 快捷键冲突矩阵 | `Ctrl+S` 与 M1 的关联行 | **删除该行** |
| 3 | UX 04 §6 被排除的快捷键表 | 不含 `Ctrl+S` / `Ctrl+Shift+S` | **添加这两条**，排除原因 = "v1.0 不提供编辑/导出" |

### 2.3 v2.0 占位声明（与 §8.1 对齐）

在 UX 04 §6 末尾新增"v2.0 快捷键预留位"小节，与需求 v1.1 §8.1 字字对齐：

```
| 快捷键        | v2.0 预留功能                          | v1.0 状态 |
|---------------|----------------------------------------|-----------|
| Ctrl+S        | 保存（v2.0「另存为」时启用）           | 不实现    |
| Ctrl+Shift+S  | 另存为                                 | 不实现    |
| Delete        | 删除（v2.0 删除功能时启用）            | 不实现    |
| Ctrl+X        | 剪切                                   | 不实现    |
| Ctrl+N        | 新建窗口（v2.0 多窗口时启用）          | 不实现    |
| Ctrl+T        | 新建标签页（v2.0 标签页时启用）        | 不实现    |
| Ctrl+W        | 关闭标签页                             | 不实现    |
| Ctrl+P        | 打印（v2.0 打印时启用）                | 不实现    |
```

### 2.4 一致性验证

| 比对项 | 结果 |
|---|---|
| 需求 v1.1 §2.2 | ✅ |
| 需求 v1.1 §8.1 v2.0 快捷键预留位 | ✅ |
| 测试风险 R2（v2.0 快捷键预留） | ✅ |
| 测试 M13 §13.2 不响应清单 | ✅ |
| 与 C1 裁决的同源一致性（避免设计自相矛盾） | ✅ |

## 三、C3 裁决修订：Delete 删除键

### 3.1 裁决结论

**v1.0 不提供 `Delete` 快捷键、不提供任何删除入口。**

依据 v1.1 §2.2「明确不包含『删除图片功能』」这一产品边界设定——产品是"查看器"不是"文件管理器"，删除操作应交由 OS 原生回收站机制处理。

### 3.2 修订动作清单

| # | 修订点（涉及文档） | 修订前 | 修订后 |
|---|---|---|---|
| 1 | UX 04 §2.6 编辑操作表 | `Delete` 行带 ⚠️ 待需求确认标识 | **完全删除该行**（连占位都不留灰色态，保持绝对洁净） |
| 2 | UX 02 信息架构 §3 工具栏布局 | 不含删除按钮 | **无需改动**（已合规） |
| 3 | UX 04 §3 冲突矩阵 | 不涉及 Delete | **无需改动** |

### 3.3 v2.0 占位声明

并入 §2.3 占位表（`Delete` 行）。

### 3.4 一致性验证

| 比对项 | 结果 |
|---|---|
| 需求 v1.1 §2.2「删除图片功能」明确排除 | ✅ |
| 测试风险 R3（无二次确认对话框） | ✅ |
| 需求 v1.1 §8.1 占位声明 | ✅ |
| 与 OS 文件管理器职责划分（删除应走回收站） | ✅ |
