# 04 — 键盘快捷键体系

> **文档定位**：定义图片查看器完整的键盘快捷键体系，对齐行业惯例，避免冲突。  
> **设计原则**：DD-2（键盘优先）、DD-1（零学习成本）  
> **关联 AC**：AC-10（键盘操作覆盖所有功能）

---

## 1. 设计目标

1. **肌肉记忆兼容**：快捷键组合尽量对齐 Windows 照片、macOS Preview、Photoshop、浏览器图片查看
2. **单手可操作**：高频操作尽可能只用左手（右手握鼠标）
3. **无歧义**：一个快捷键只对应一个功能
4. **不拦截系统键**：不与 OS 全局快捷键冲突
5. **板正克制**：不做自定义快捷键（v1.0），保持产品气质

---

## 2. 完整快捷键表

### 2.1 文件操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `Ctrl+O` | 打开文件 | Windows 通用 | 打开文件对话框 |
| `Ctrl+S` | 保存 | Windows 通用 | 保存当前图片（若有修改） |
| `Ctrl+Shift+S` | 另存为 | 通用 | 另存为对话框 |
| `Ctrl+P` | 打印 | Windows 通用 | 打印对话框 |
| `Ctrl+Q` | 退出 | Linux 通用 | ⚠️ Windows 习惯 `Alt+F4`，两者都支持 |
| `Alt+F4` | 退出 | Windows 通用 | 关闭窗口 |

### 2.2 缩放操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `Ctrl+=` / `Ctrl++` | 放大 | Photoshop / 浏览器 | `=` 键不需要 Shift（等同于 `+`） |
| `Ctrl+-` | 缩小 | Photoshop / 浏览器 | |
| `Ctrl+0` | 1:1 原始大小 | Photoshop / 浏览器 | |
| `Ctrl+9` | 适应窗口 | 自定义 | Photoshop 用 `Ctrl+0`（第二次），这里独立出来 |
| `Ctrl+滚轮↑` | 放大 | 浏览器 / Windows 照片 | 连续平滑缩放 |
| `Ctrl+滚轮↓` | 缩小 | 浏览器 / Windows 照片 | |

### 2.3 旋转操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `Ctrl+,` | 逆时针旋转 90° | 自定义 | `,` 和 `.` 在键盘相邻，视觉上对应 `↺` `↻` |
| `Ctrl+.` | 顺时针旋转 90° | 自定义 | |
| `Ctrl+Shift+,` | 重置旋转为 0° | 自定义 | 反转旋转状态 |

### 2.4 翻看操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `←` | 上一张 | Windows 照片 / 所有图片查看器 | |
| `→` | 下一张 | Windows 照片 / 所有图片查看器 | |
| `Home` | 第一张 | 通用 | |
| `End` | 最后一张 | 通用 | |
| `PageUp` | 上一张（备选） | 部分图片查看器 | 兼容习惯 |
| `PageDown` | 下一张（备选） | 部分图片查看器 | 兼容习惯 |
| `滚轮↑`（无 Ctrl） | 上一张 | Windows 照片 | ⚠️ 全屏模式下也适用 |
| `滚轮↓`（无 Ctrl） | 下一张 | Windows 照片 | |

### 2.5 视图操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `F11` | 全屏 开/关 | 浏览器 / Windows 通用 | |
| `F` | 全屏 开/关 | 视频播放器惯例 | 备选快捷键 |
| `Esc` | 退出全屏 / 退出特殊状态 | 通用 | 最高优先级的"退出"键 |
| `Space` | 下一张（全屏时） | macOS Preview | 大按键，全屏时方便 |
| `Shift+Space` | 上一张（全屏时） | macOS Preview | |

### 2.6 编辑操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `Ctrl+C` | 复制图片到剪贴板 | Windows 通用 | 复制图片数据，非文件路径 |
| `Ctrl+X` | 剪切（⚠️ 可能不适合） | — | ⚠️ 图片查看器通常不提供剪切 |
| `Delete` | 删除当前图片 | — | ⚠️ 待需求确认是否提供删除功能 |

### 2.7 窗口操作

| 快捷键 | 功能 | 行业对齐 | 备注 |
|--------|------|---------|------|
| `Alt+Enter` | 图片属性/详情 | Windows 通用 | 文件属性对话框或内置详情面板 |

## 4. 平台差异适配

### 4.1 Windows vs Linux

| 功能 | Windows 标准 | Linux 标准 | 本产品策略 |
|------|------------|-----------|-----------|
| 退出 | `Alt+F4` | `Ctrl+Q` | **两者都支持** |
| 打开 | `Ctrl+O` | `Ctrl+O` | 一致 |
| 全屏 | `F11` | `F11` | 一致 |
| 菜单 | `Alt` | `Alt` | 一致 |

### 4.2 与系统全局快捷键的冲突

| 系统快捷键 | 冲突风险 | 处理方式 |
|-----------|---------|---------|
| `Ctrl+Alt+Del` (Windows) | 无 | 未占用 |
| `Alt+Tab` | 无 | 系统优先，不拦截 |
| `Win+D` | 无 | 系统优先，不拦截 |
| `Ctrl+Shift+Esc` | 无 | 未占用 |
| 截图工具 (Win+Shift+S) | 无 | 系统优先 |

## 5. 全屏模式下的特殊键位

全屏模式下所有快捷键**与默认模式保持一致**，额外增加：

| 快捷键 | 功能 | 原因 |
|--------|------|------|
| `Space` | 下一张 | 大按键，方便全屏时用拇指 |
| `Shift+Space` | 上一张 | 自然配对 |
| `Esc` | 退出全屏（最高优先级） | 通用"退出"键 |
| `双击图片` | 退出全屏 | 与默认模式双击行为一致 |

## 7. "不可自定义"的设计决策

> **v1.0 决策**：快捷键不可自定义。  
> **理由**：
> 1. 板正气质——产品不应该让用户去"配置"，而是直接告诉他们"就是这样"
> 2. 当前快捷键体系经过精心设计，与行业惯例对齐，无需自定义
> 3. 减少设置界面的开发成本和维护复杂度
> 4. 避免"设置太多"导致的用户困惑（DD-1：零学习成本）
>
> **v2.0 展望**：如有足够用户反馈，可加入"快捷键配置"面板。
