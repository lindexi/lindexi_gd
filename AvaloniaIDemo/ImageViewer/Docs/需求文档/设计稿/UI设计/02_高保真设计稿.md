### 逐区域详述

#### 标题栏
- **实现方式**：系统原生标题栏（不自定义绘制）
- **颜色**：由 OS 决定（暗色模式下通常接近 `#2C2C2C`）
- **标题文字**：`{文件名} - 图片查看器`，12px Regular
- **窗口图标**：应用图标（16×16px），由系统标题栏渲染

#### 工具栏
- **背景**：`--surface-chrome`（#2C2C2C）
- **高度**：36px
- **按钮布局**（从左到右）：
  - **文件组**：`[📂 打开]` — 图标 16px + 标签 12px，间距 4px
  - **分隔线**：1px 竖线，`--divider`（#3A3A3A），高 20px，上下居中
  - **缩放组**：`[-] [+] [100%▼] [1:1] [适应]` — "100%▼"为文字按钮，带下拉箭头图标
  - **分隔线**
  - **旋转组**：`[↺] [↻]` — 纯图标按钮
  - **分隔线**
  - **视图组**：`[⊞ 全屏]` — 图标+标签按钮
  - **弹性空间**（spring）
  - **更多**：`[⋮]` — 纯图标按钮，右对齐
- **按钮状态**：
  - 默认：背景透明，文字 `--text-primary`，图标 `--text-primary`
  - 悬停：背景 `--surface-hover`，100ms ease 过渡
  - 按下：背景 `--surface-pressed`
  - 禁用：文字 `--text-disabled`，图标 `--text-disabled`，无悬停效果
- **"100%▼" 下拉按钮**：点击展开下拉菜单（见 §S1-附加）

#### 图片显示区
- **背景**：`--surface-image-area`（#1A1A1A）
- **图片**：等比缩放至适应窗口，居中显示
- **光标**：默认箭头（图片未放大时不需要抓手）

#### 状态栏
- **背景**：`--surface-chrome`（#2C2C2C）
- **高度**：28px
- **信息布局**（从左到右）：
  - `● 第 N/M 张` — `--text-primary`，12px。当 M=1 时不显示此项
  - `│` 分隔符 — `--text-tertiary`（#666）
  - `文件名` — `--text-primary`，12px
  - `│` 分隔符
  - `宽×高` — `--text-secondary`（#999），11px
  - `│` 分隔符
  - `文件大小` — `--text-secondary`，11px
  - `│` 分隔符
  - `格式` — `--text-secondary`，11px
  - 弹性空间
  - `缩放比例` — 仅在非 100% 时显示，使用 `--accent`（#4A90D9），12px Semibold

#### S1-附加：缩放下拉菜单

点击 `[100%▼]` 展开：

```
┌──────────────┐  ← 背景 #353535，边框 1px #454545，radius 6px
│  25%         │  ← 文字 #E8E8E8 14px，行高 32px
│  50%         │     悬停行背景 rgba(255,255,255,0.06)

│  200%        │

│  原始大小    │  ← Ctrl+0

  宽度 140px

```

### S1 动效

| 元素 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 按钮悬停 | 背景色过渡 | 100ms | ease |
| 图片加载完成 | 200ms 渐显 | 200ms | ease-out |
| 下拉菜单展开 | 从上展开 + 100ms 淡入 | 100ms | ease-out |

### 逐区域详述

## S3 — 缩放状态（以 150% 为例）

### 视觉表现

```
┌──────────────────────────────────────────────────────────────┐
│  [Icon]  DSC001.jpg - 图片查看器 (150%)    [_] [□] [X]    │  ← 标题栏追加缩放比例
├──────────────────────────────────────────────────────────────┤
│ [📂打开] │ [-] [+] [150%▼] [1:1] [适应] │ [↺] [↻] │ [⊞全屏] │  │
│          │          指示器=150%         │         │         │  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│            ┌──────────────────────────────────┐              │
│            │                                  │              │
│            │     图片内容 (150% 放大)         │              │
│            │     超出窗口, 可拖拽平移          │              │
│            │                                  │              │
│            │                                  │              │
│            └──────────────────────────────────┘              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ ● 第 1/1 张 │ DSC001.jpg │ 1920×1080 │ 2.3 MB │ JPEG │150%│  │
│                                                     #4A90D9  │  ← accent 色高亮
└──────────────────────────────────────────────────────────────┘

                                    ┌───────────┐
                                    │   150%    │  ← 右下角浮动指示器
                                    │  ████░░   │     背景 rgba(0,0,0,0.75)
                                    └───────────┘     文字 #E8E8E8 16px Semibold
                                                      进度条 accent 色
                                                      圆角 6px, padding 8px 12px
                                                      出现后 1.5s → 300ms 淡出
```

### 逐区域详述

#### 标题栏
- 标题追加缩放比例：`DSC001.jpg - 图片查看器 (150%)`
- 当缩放比例 ≠ 100% 时显示括号内容

#### 工具栏
- "150%▼" 以 `--accent`（#4A90D9）高亮
- 右侧出现 `[1:1]` 和 `[适应]` 按钮正常可点击（缩放状态下这两个是高频入口）

#### 图片显示区
- 图片按 150% 渲染，居中但溢出窗口区域
- 水平/垂直滚动条出现（系统原生滚动条，暗色主题）
- **光标**：图片上悬停 → `grab`（✋ 抓手），拖拽时 → `grabbing`（✊ 捏手）
- 拖拽平滑跟随鼠标，边界停止

#### 状态栏
- 缩放比例 "150%" 使用 `--accent`（#4A90D9），12px Semibold

#### 右下角浮动指示器（Zoom Indicator OSD）
- **触发**：每次缩放比例变化时出现
- **位置**：窗口右下角，距右边缘 16px，距底部（状态栏上边缘）12px
- **视觉**：
  - 背景：`rgba(0, 0, 0, 0.75)`，圆角 6px
  - 内边距：8px 12px
  - 百分比数字：`--text-primary`（#E8E8E8），16px Semibold
  - 下方进度条：宽 40px，高 2px，背景 `rgba(255,255,255,0.2)`，填充 `--accent`
  - 进度条表示当前缩放级别在 10%~3200% 范围中的位置
- **动效**：
  - 出现：200ms fade-in（ease-out）
  - 持续显示：1.5 秒
  - 消失：300ms fade-out（ease-in）
  - 如果在消失过程中再次缩放 → 重置计时器，不闪

### S3 动效（新增）

| 元素 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 缩放指示器 OSD | 淡入 | 200ms | ease-out |
| 缩放指示器 OSD | 淡出 | 300ms | ease-in |
| 光标切换（箭头→抓手） | 即时 | 0ms | — |

## S4 — 旋转状态（以顺时针 90° 为例）

### 视觉表现

```
┌──────────────────────────────────────────────────────────────┐
│  [Icon]  DSC001.jpg - 图片查看器            [_] [□] [X]    │
├──────────────────────────────────────────────────────────────┤
│ [📂打开] │ [-] [+] [100%▼] [1:1] [适应] │ [↺] [↻] │ [⊞全屏] │  │
│          │                             │      ↻高亮│         │  │
│          │                             │ 选中态蓝底 │         │  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                    ┌──────────────┐                          │
│                    │              │                          │
│                    │  图片内容    │                          │
│                    │  (旋转90°)  │                          │
│                    │              │                          │
│                    │              │                          │
│                    └──────────────┘                          │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ ● 第 1/1 张 │ DSC001.jpg │ 1920×1080 │ 2.3 MB │ JPEG │ ↻90°│  │
│                                                    #D4A040   │  ← warning 色高亮
└──────────────────────────────────────────────────────────────┘
```

### 逐区域详述

#### 工具栏 — 旋转按钮
- 被激活的旋转方向按钮进入**选中态**：
  - 背景：`--surface-selected`（`rgba(74,144,217,0.18)`）
  - 图标颜色：`--accent`（#4A90D9）
  - 这是一个持续性状态（非悬停），表示"当前图片正在旋转显示"
- 另一个方向按钮保持默认态，但点击会取消旋转（回到 0°）

#### 图片显示区
- 图片顺时针旋转 90° 渲染
- 适应窗口规则仍然生效：旋转后的图片仍然等比缩放至窗口内完整可见
- 若旋转后图片宽度超出窗口→可拖拽平移

#### 状态栏
- 旋转指示 "↻90°" 使用 `--warning`（#D4A040），12px Regular
- 位置：放在格式信息之后（最右侧）
- ⚠️ **视觉设计强调**：使用暖橙色而非红色——旋转不是错误，是用户主动操作，只需温和提醒

#### ⚠️ 设计标注：关于"关闭时询问保存"

> **UI 标注**：UX 文档 S4 线框和 02 文档中描述了"关闭时弹出保存询问对话框"。**需求文档 v1.0 §4.3 明确规定旋转为「仅视图旋转，关闭即丢失」。** 本 UI 文档以此为准，**不设计保存询问对话框**。此不一致将在 `03_与UX对照检查.md` 中正式记录，待阶段 D 评审裁决。

### S4 动效

无新增。旋转是即时渲染变化，不添加过渡动画（旋转动画会让用户产生"这需要等待"的错觉）。

```
│                    ┌─────────────────┐                       │
│                    │                 │                       │
│                    │                 │                       │
│                    │   图片内容      │                       │
│                    │   (适应屏幕)    │                       │

  图片居中，等比缩放至适应屏幕
  所有 UI chrome 隐藏（标题栏、工具栏、状态栏）
  光标：默认隐藏（3秒无操作后自动隐藏）
```

### 逐区域详述

- **背景**：纯黑 `#000000`（全屏下不需要"画框"效果，#1A1A1A 和 #000 的区别在全屏下不可感知，直接用纯黑性能最优）
- **图片**：等比缩放至适应屏幕（不是窗口），居中
- **光标**：
  - 鼠标移动时显示（箭头或抓手，取决于是否放大）
  - 3 秒无操作 → 光标隐藏（OS 级全屏标准行为）
- **工具栏**：完全隐藏（位于屏幕底部下方，等待 S6 触发）
- **退出方式**：`Esc` / `F11` / `F` / 双击图片

### S5 动效

| 元素 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 进入全屏 | 背景从 #1A1A1A → #000 | 150ms | ease-out |
| 退出全屏 | 背景从 #000 → #1A1A1A | 150ms | ease-in |

## S6 — 全屏·工具栏浮现

### 视觉表现

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│                                                              │
│                    ┌─────────────────┐                       │
│                    │                 │                       │
│                    │   图片内容      │                       │
│                    │                 │                       │
│                    │                 │                       │
│                    └─────────────────┘                       │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤  ← 全屏底部工具栏
│  ←  │ 第 3/15 张  │  DSC001.jpg  │  1920×1080  │ [↺][↻] │ ⊟ │    高度 44px
│ P3  │    P1           P1             P1          P3  P3   P3│    背景 rgba(0,0,0,0.85)
└──────────────────────────────────────────────────────────────┘    从底部滑入
```

### 逐区域详述

- **触发**：全屏模式下鼠标移动
- **工具栏位置**：屏幕底部，横贯全宽
- **背景**：`rgba(0, 0, 0, 0.85)` —— 纯黑 85% 不透明度，支撑沉浸感的同时保证文字可读
- **高度**：44px（比默认工具栏高 8px，因为全屏下需要更大的点击目标）
- **内容布局**（从左到右）：
  - `[‹]` 上一张 — 图标 20px
  - `第 N/M 张` — `--text-primary`，12px
  - `│` 分隔符
  - `文件名` — `--text-primary`，12px
  - `│`
  - `分辨率` — `--text-secondary`，11px
  - 弹性空间
  - `[↺]` `[↻]` 旋转 — 图标 20px
  - `[⊟]` 退出全屏 — 图标 20px
- **按钮尺寸**：44×44px 点击区域（全屏下手不易精确操作，给大一点）
- **动效**：
  - 出现：从底部滑入（slide-up），200ms，ease-out
  - 消失：鼠标静止 2 秒后滑出（slide-down），200ms，ease-in
  - 鼠标再次移动 → 立即滑入，重置 2 秒计时器

### S6 动效

| 元素 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 工具栏滑入 | translateY(44px → 0) + opacity(0 → 1) | 200ms | ease-out |
| 工具栏滑出 | translateY(0 → 44px) + opacity(1 → 0) | 200ms | ease-in |
| 光标显示/隐藏 | opacity 切换 | 150ms | — |

## S7 — 错误·不支持格式

### 视觉表现

- **主标题**："无法打开此文件"，`--text-primary`（#E8E8E8），14px Regular，图下 12px
- **副标题**："不支持的图片格式"，`--text-secondary`（#999），12px，主标题下 4px

- **背景**：`--surface-image-area`（#1A1A1A）
- **工具栏**：除"打开"外全部禁用
- **状态栏**：显示"打开失败"，`--text-secondary`

### 视觉要点
- 红色图标足够醒目，但不过度（48px vs 空状态的 64px，因为错误状态不应比正常状态更"大"）
- 文案语气：陈述事实，不道歉（"无法打开"而非"抱歉，无法打开"——工具不需要道歉）

## S8 — 错误·文件损坏

### 视觉表现

```
（布局与 S7 相同，替换内容）

                          ⚠  (48×48 线性图标, #D4A040)       ← 注意：用 warning 橙而非 error 红

                    无法打开此文件
                    文件已损坏或格式异常

                    请检查文件是否完整
                    (灰色小字，11px, #999)
```

### 逐区域详述

- **图标颜色**：`--warning`（#D4A040）而非 `--error`（#E05555）
  - 理由：文件损坏可能是用户拷贝未完成等临时问题，未必是用户选错了文件。橙色比红色少一些"你做错了"的指责感
- **文案**："无法打开此文件 / 文件已损坏或格式异常"
- **建议**："请检查文件是否完整"
- 其余布局与 S7 完全一致

## S9 — 加载中

### 视觉表现

```
┌──────────────────────────────────────────────────────────────┐
│  [Icon]  DSC001.jpg - 图片查看器            [_] [□] [X]    │
├──────────────────────────────────────────────────────────────┤
│ [📂打开] │ [-] [+] [---▼] [1:1] [适应] │ [↺] [↻] │ [⊞全屏] │  │
│  正常         全部禁用                     禁用      禁用      │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                                                              │
│                                                              │
│                    ⏳ 正在加载...                             │
│                    (14px, #999)                               │
│                                                              │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ ● 第 1/1 张 │ DSC001.jpg │ --- │ --.- MB │ 加载中...        │
└──────────────────────────────────────────────────────────────┘
```

### 逐区域详述

- **加载指示**：
  - ⚠️ **不使用旋转动画 spinner**——这是"板正"的底线之一
  - 替代方案：仅显示文字"正在加载..."，`--text-secondary`（#999），14px，居中
  - 如果图片在 200ms 内加载完成 → 跳过加载态，直接渐显图片
  - 如果超过 200ms → 显示"正在加载..."
- **状态栏**：分辨率和文件大小显示 `---` / `--.- MB`（尚不可用），格式位置显示"加载中..."
- **工具栏**：缩放/旋转/全屏按钮禁用（图片未就绪时不可操作）
- **背景**：`--surface-image-area`（#1A1A1A）

### 设计决策：为什么不用 spinner

> Spinner（转圈动画）对用户传达的信息是"请等待，系统正在忙碌"。但对于图片查看器场景，用户等待的是"图片解码→渲染"，通常耗时 < 500ms。在这段时间内，一个转圈的动画比一行静态文字更分散注意力——用户会盯着圈看。一行安静的"正在加载..."告诉用户同样的事情，但不制造焦虑。这是"克制"的体现。

## S10 — 单实例替换

### 视觉表现

此状态为**过渡状态**，持续时间极短（200-500ms），但需要明确设计以避免"闪烁感"。

```
时间线：

t=0      已有窗口显示图片 A（状态 S1/S3/S4 等）
         ↓
t=10ms   用户双击图片 B
         ↓
t=50ms   图片 A 快速淡出（100ms fade-out）
         窗口标题栏更新为图片 B 的文件名
         状态栏信息更新为图片 B 的信息
         工具栏按钮恢复默认态（清除旋转选中、缩放指示器重置）
         ↓
t=150ms  图片 B 开始加载（如需要则进入 S9 加载态）
         ↓
t=???    图片 B 渐显（200ms fade-in）
         ↓
         恢复为 S1 默认浏览状态
```

### 逐区域详述

- **关键设计**：替换过程有短暂交叉淡入淡出（crossfade），避免生硬的"闪一下"
- **图片 A 淡出**：100ms，ease-in
- **UI 信息切换**：标题栏、状态栏信息在图片 A 开始淡出时同步更新（无动画，直接替换文字）
- **工具栏状态重置**：缩放比例回 100%、旋转状态清除、按钮恢复默认
- **图片 B 淡入**：200ms，ease-out（加载完成后）
- **如果图片 B 加载失败**：淡出图片 A 后，进入 S7/S8 错误状态

### S10 动效

| 元素 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 图片 A 淡出 | opacity 1→0 | 100ms | ease-in |
| 图片 B 淡入 | opacity 0→1 | 200ms | ease-out |
| UI 信息切换 | 即时替换 | 0ms | — |
