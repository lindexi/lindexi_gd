# 01 — 视觉风格定义

> **文档定位**：定义图片查看器的视觉设计语言——配色、字体、图标、间距、圆角、阴影、动效原则。作为所有高保真设计稿的"宪法"。  
> **气质关键词（继承 UX）**：板正、克制、专业、简洁（非极简）  
> **产出人**：UI 设计师  
> **基于**：`图片查看器_需求分析文档_v1.0.md` + UX 设计 `01~05`

---

## 一、设计哲学：板正的视觉翻译

UX 总纲将「板正」翻译为 4 条交互原则。UI 视角下，进一步翻译为视觉语言：

| 板正关键词 | UI 翻译 | 视觉约束 |
|-----------|---------|---------|
| **规矩** | 视觉一致 | 颜色、字号、间距、图标风格全局统一。不出现"这里圆角 4px、那里圆角 8px"的随意感 |
| **正统** | 不追潮流 | 不走毛玻璃、新拟态、渐变弥散等流行风格。用经典暗色工具面板——这种风格 20 年前就在用，20 年后还能用 |
| **专业** | 信息清晰 | 文本对比度 ≥ 4.5:1（WCAG AA），状态栏信息无歧义，缩放指示器精确到百分比 |
| **克制** | 存在感极低 | 所有 UI 元素服务于"让图片成为唯一主角"。界面 chrome 的存在感 = 0 |

**核心设计目标：用户回忆使用体验时，记住的是图片，不是界面。**

## 二、配色方案

### 2.1 设计理念

图片查看器的核心任务是展示图片内容——色彩丰富的、黑白灰的、高饱和的、低反差的，什么图片都可能出现。UI chrome 必须做到**在任何图片旁边都不抢戏**。

策略：使用**暖色暗灰**作为基底。纯黑（#000）过于"冷"，冷灰（#2D2D2D）偏向代码编辑器气质。暖灰在暗色环境下更接近人眼舒适区，且与大多数图片的色温兼容。

### 2.2 色板

#### 主色系 — UI Chrome

| Token 名 | 色值 | 用途 |
|----------|------|------|
| `--surface-chrome` | `#2C2C2C` | 工具栏背景、状态栏背景、标题栏背景 |
| `--surface-image-area` | `#1A1A1A` | 图片显示区域背景（比 chrome 更深，形成"画框"效果） |
| `--surface-hover` | `rgba(255,255,255,0.06)` | 按钮悬停态 |
| `--surface-pressed` | `rgba(255,255,255,0.10)` | 按钮按下态 |
| `--surface-selected` | `rgba(74,144,217,0.18)` | 按钮选中/激活态（如旋转按钮激活时） |

> **设计理由**：图片区用更深色（#1A1A1A）是刻意为之。视觉上形成"外框（chrome #2C2C2C）→ 画框（#1A1A1A）→ 内容（图片）"三层嵌套，让图片"浮"在界面上，类似画廊的黑色衬纸效果。

#### 文字色系

| Token 名 | 色值 | WCAG 对比度（vs #2C2C2C） | 用途 |
|----------|------|--------------------------|------|
| `--text-primary` | `#E8E8E8` | 12.6:1 ✅ AAA | 工具栏按钮标签、状态栏信息、标题栏文件名 |
| `--text-secondary` | `#999999` | 4.9:1 ✅ AA | 状态栏辅助信息（文件大小单位）、工具提示 |
| `--text-tertiary` | `#666666` | 2.8:1 ⚠️ | 仅用于极低优先级的辅助文字（如分隔符） |
| `--text-disabled` | `#555555` | 2.2:1 — | 禁用按钮文字/图标 |

#### 语义色

| Token 名 | 色值 | 用途 |
|----------|------|------|
| `--accent` | `#4A90D9` | 主要交互强调色（缩放指示器百分比、激活状态、聚焦环） |
| `--accent-hover` | `#5BA0E9` | 强调色悬停 |
| `--accent-pressed` | `#3A80C9` | 强调色按下 |
| `--warning` | `#D4A040` | 旋转指示器（状态栏"↻90°"高亮）、警告性提示 |
| `--error` | `#E05555` | 错误状态图标/文字（不支持格式、损坏文件提示） |
| `--success` | `#5CB85C` | 保留（当前无使用场景，为后续扩展预留） |

> **设计理由**：`--accent` 选 `#4A90D9` 而非 Windows 蓝（#0078D4）或 macOS 蓝（#007AFF），这是刻意选择。比 Windows 蓝略柔和，比 macOS 蓝略暖——在跨平台场景下不偏向任何一方。饱和度中等，不刺眼，在暗色背景下温和但足够引人注意。

#### 分割线与边框

| Token 名 | 色值 | 用途 |
|----------|------|------|
| `--divider` | `#3A3A3A` | 工具栏与图片区之间的分割线、按钮组之间的分隔 |
| `--border` | `#454545` | 输入框边框、下拉菜单边框 |

### 2.3 配色禁用清单

| 禁用 | 原因 |
|------|------|
| ❌ 渐变色（任何形式） | 违反"板正"——渐变属于装饰，分散注意力 |
| ❌ 纯黑背景 #000000 | 过于冰冷，与大多数图片的暖色调冲突 |
| ❌ 高饱和度强调色（如 #FF4500） | 刺眼，工具不应"喊叫" |
| ❌ 半透明毛玻璃效果（acrylic/blur） | 跨平台实现不一致，且属于流行风格 |
| ❌ 亮色主题（v1.0） | 看图场景暗色更沉浸；亮色主题留给 v2.0 |

## 三、字体体系

### 3.1 字体族

```
--font-family-ui: "Segoe UI", "Noto Sans", "Noto Sans CJK SC", system-ui, sans-serif;
--font-family-mono: "Cascadia Code", "JetBrains Mono", "Noto Sans Mono", monospace;
```

| 平台 | 首选字体 | CJK 回退 | 理由 |
|------|---------|---------|------|
| Windows | Segoe UI | Microsoft YaHei | Windows 原生 UI 字体，用户视觉惯性 |
| Linux | Noto Sans | Noto Sans CJK SC | 跨发行版一致性好，Google 维护 |

> **设计决策**：不引入自定义字体。系统字体零加载成本，且用户最熟悉。这也是"板正"的体现——不做品牌化字体差异。

### 3.2 字号与行高

| Token 名 | 字号 | 行高 | 字重 | 用途 |
|----------|------|------|------|------|
| `--text-xs` | 11px | 16px | Regular (400) | 状态栏辅助文字、工具提示 |
| `--text-sm` | 12px | 18px | Regular (400) | 工具栏按钮标签、状态栏主要信息、标题栏 |
| `--text-md` | 14px | 20px | Regular (400) | 对话框正文、错误提示文案、下拉菜单项 |
| `--text-lg` | 16px | 24px | Semibold (600) | 缩放指示器浮动百分比数字 |
| `--text-xl` | 20px | 28px | Regular (400) | 空状态大图标下方主提示文字 |

> **设计理由**：字号不超 20px，因为 UI 不是主角。所有 UI 文字都偏小——让用户把注意力留给图片。

### 3.3 字体灰度（Font Weight）使用原则

| 字重 | 使用场景 | 原则 |
|------|---------|------|
| Regular (400) | 默认全部 UI 文字 | 克制——不靠粗细制造层级，靠位置和颜色 |
| Semibold (600) | 仅缩放指示器百分比 | 唯一例外：这个数字在 1.5 秒内必须被看清 |

## 四、图标体系

### 4.1 风格规范

| 属性 | 规范 | 理由 |
|------|------|------|
| 风格 | **线性图标（Line）** | 面性图标视觉重量过大，在暗色背景下过于显眼 |
| 描边宽度 | **1.5px** | 2px 笨重，1px 在低分屏上发虚 |
| 尺寸 | **16×16px**（工具栏）、**20×20px**（空状态） | 16px 是桌面端工具栏标准 |
| 颜色 | 继承 `--text-primary`（#E8E8E8） | 与文字统一，保持视觉安静 |
| 圆角 | 图标内部圆角 1px（连接处）、端点为圆头 | 匹配"专业但不过于锋利"的气质 |

### 4.2 各功能图标规格

| 功能 | 图标描述 | 参考（不照抄） |
|------|---------|--------------|
| 打开 | 文件夹 + 向右小箭头（组合） | Phosphor `FolderOpen` |
| 缩小 | 放大镜 + `−` | Phosphor `MagnifyingGlassMinus` |
| 放大 | 放大镜 + `+` | Phosphor `MagnifyingGlassPlus` |
| 1:1 | `1:1` 文字图标（不用图形） | 纯文字在此时更直接 |
| 适应窗口 | 四角外扩箭头 或 矩形框 + 四角箭头 | Phosphor `ArrowsOut` |
| 逆时针旋转 | ↺ 弧形箭头 | Phosphor `ArrowArcLeft` |
| 顺时针旋转 | ↻ 弧形箭头 | Phosphor `ArrowArcRight` |
| 全屏 | ⊞ 四角外扩 | Phosphor `ArrowsOut`（与适应窗口可共用或微调） |
| 退出全屏 | ⊟ 四角内收 | Phosphor `ArrowsIn` |
| 更多 | ⋮ 三点纵排 | 标准"更多"图标 |
| 上一张 | ‹ 左箭头 | Phosphor `CaretLeft` |
| 下一张 | › 右箭头 | Phosphor `CaretRight` |
| 错误 | ⚠ 三角形感叹号 | Phosphor `Warning` |
| 空状态图片 | 山/太阳简化图形 | Phosphor `Image` |

### 4.3 图标设计红线

- ❌ 不填充（保持线性）
- ❌ 不用 emoji（跨平台渲染不一致）
- ❌ 不超 16×16px（工具栏）
- ❌ 不用渐变或阴影

## 五、间距系统

### 5.1 基准网格

基于 **4px 网格系统**。所有间距为 4 的倍数。

| Token 名 | 值 | 用途 |
|----------|-----|------|
| `--space-xs` | 4px | 紧凑间距：按钮组内间隙、图标与文字间距 |
| `--space-sm` | 8px | 标准间距：工具栏按钮组之间、状态栏信息项之间 |
| `--space-md` | 12px | 宽松间距：工具栏左右内边距、状态栏内边距 |
| `--space-lg` | 16px | 大间距：对话框内边距 |
| `--space-xl` | 24px | 超大间距：空状态内容区上下留白 |

### 5.2 各区域具体间距

```
工具栏：
  - 高度：36px
  - 左右内边距：12px (--space-md)
  - 按钮内边距：6px 10px (上下 6px，左右 10px)
  - 按钮组之间分隔线：左右各 8px (--space-sm)
  - 图标与标签文字间距：4px (--space-xs)

图片显示区：
  - 无边距（图片撑满可用空间）
  - 与工具栏之间：1px 分割线

状态栏：
  - 高度：28px
  - 左右内边距：12px (--space-md)
  - 信息项之间：使用 " │ " 分隔符，两侧各 8px

窗口默认大小（建议研发参考）：
  - 宽度：900px
  - 高度：680px
  - 最小窗口：400×300px
```

## 六、圆角规范

### 6.1 原则

暗色 UI 下圆角不易被察觉，但仍需统一规范。原则：**能不圆就不圆，必须圆就用小圆角。**

| Token 名 | 值 | 适用范围 |
|----------|-----|---------|
| `--radius-none` | 0px | 工具栏容器、状态栏容器、图片区——这些都该是直角 |
| `--radius-sm` | 4px | 按钮、输入框、工具栏上的交互元素 |
| `--radius-md` | 6px | 下拉菜单面板、缩放指示器浮层、工具提示 |
| `--radius-lg` | 8px | 对话框、模态面板 |

> **设计理由**：工具栏和状态栏用锐利直角（0px），传递"可靠、稳固"的感觉。按钮用微圆角（4px），纯粹的可用性优化——完全直角的按钮在鼠标悬停高亮时边缘过于锋利，4px 刚好软化到不刺眼的程度。这是 Photoshop 工具栏的做法。

## 七、阴影与层次（Elevation）

### 7.1 原则

暗色 UI 下阴影效果大打折扣（阴影投在暗色背景上不显眼）。因此：**用亮度差替代阴影来表现层次**。

| 层级 | 视觉表现 | 使用场景 |
|------|---------|---------|
| Base (0) | `#2C2C2C` | 工具栏、状态栏 |
| Raised (1) | `#353535` + 1px 边框 `#454545` | 下拉菜单面板、工具提示 |
| Overlay (2) | `#2C2C2C` + 1px 边框 `#555` | 对话框、模态面板 |

### 7.2 阴影（仅对话框）

对话框是唯一使用真正阴影的地方（因为它覆盖在图片上方，需要明确的分层）：

```css
dialog-shadow: 0px 4px 16px rgba(0, 0, 0, 0.5);
```

## 八、动效原则

### 8.1 总纲

**v1.0 动效 = 功能性动效 + 零装饰动效。**

| 允许 | 不允许 |
|------|--------|
| ✅ 全屏工具栏 200ms 滑入/滑出（ease-out / ease-in） | ❌ 弹跳、弹性缓动 |
| ✅ 缩放指示器 200ms 淡入 + 1.5s 后 300ms 淡出 | ❌ 旋转动画 |
| ✅ 按钮悬停 100ms 背景色过渡 | ❌ 页面切换过渡 |
| ✅ 图片加载 200ms 渐显（fade-in） | ❌ 任何形式的 loading spinner 动画 |
| ❌ — | ❌ 窗口打开/关闭动画 |

### 8.2 缓动函数

| 场景 | 缓动 | CSS 等价 |
|------|------|---------|
| 元素出现（工具栏滑入、指示器淡入） | ease-out | `cubic-bezier(0.0, 0.0, 0.2, 1.0)` |
| 元素消失（工具栏滑出、指示器淡出） | ease-in | `cubic-bezier(0.4, 0.0, 1.0, 1.0)` |
| 状态过渡（按钮悬停） | ease | `cubic-bezier(0.4, 0.0, 0.2, 1.0)` |

### 8.3 动效时长

| 场景 | 时长 | 理由 |
|------|------|------|
| 按钮悬停背景色过渡 | 100ms | 几乎不可感知但足够平滑 |
| 全屏工具栏滑入 | 200ms | 快但不突兀 |
| 全屏工具栏滑出 | 200ms | 与滑入对称 |
| 缩放指示器淡入 | 200ms | 让用户注意到它出现了 |
| 缩放指示器淡出 | 300ms | 比淡入更慢——"轻轻地走" |
| 图片加载渐显 | 200ms | 避免闪烁，但不过度 |

## 九、空状态设计

### 9.1 空状态布局

当应用启动未打开任何图片时：
- 中心放置一个 64×64px 的线性图标（山/图片轮廓），颜色 `--text-tertiary`（#666）
- 图标下方 16px 放置引导文字："拖拽图片到此处，或点击 [打开...] 按钮"
- [打开...] 为可点击按钮样式，使用 `--accent` 色文字
- 背景使用 `--surface-image-area`（#1A1A1A）

### 9.2 设计理由

不做华丽的空状态插图。一张简单的线性图标 + 一句话。用户不是来欣赏空状态的。

## 十、与平台视觉惯例的对齐

| 视觉元素 | Windows 10/11 惯例 | Linux (GNOME) 惯例 | 本项目选择 |
|---------|-------------------|-------------------|-----------|
| 标题栏 | 原生系统标题栏 | 原生系统标题栏 | **原生系统标题栏**（不由应用自绘） |
| 工具栏高度 | 36-40px | 32-38px | **36px**（折中） |
| 强调色 | Accent Color API | 无统一规范 | **固定 #4A90D9**（不跟随系统） |
| 暗色/亮色 | 跟随系统 | 跟随系统 | **v1.0 仅暗色**（暗色对看图场景最优） |
| 滚动条 | 系统原生 | 系统原生 | **系统原生**（Avalonia 默认） |

> **关键决策**：v1.0 使用原生系统标题栏而非自绘。理由：①板正——窗口应该像窗口；②跨平台一致性交由 OS 保证；③减少 Avalonia 自绘的兼容风险。

## 十一、视觉风格总结

```
┌─────────────────────────────────────────┐
│         图片查看器 视觉 DNA              │
├─────────────────────────────────────────┤
│  配色：暖暗灰基底 + 克制蓝强调            │
│  字体：系统原生、偏小、Regular 为主       │
│  图标：线性、1.5px 描边、16px            │
│  间距：4px 网格、紧凑但透气              │
│  圆角：能直就直（0px），按钮微圆（4px）   │
│  阴影：几乎不用，靠亮度差分层             │
│  动效：只做功能性，不做装饰性             │
│  气质：像徕卡相机——黑、直、精准、不喧哗  │
└─────────────────────────────────────────┘
```
