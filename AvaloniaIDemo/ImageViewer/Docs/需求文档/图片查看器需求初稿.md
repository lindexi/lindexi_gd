## 一、需求概述

| 维度 | 内容 |
|------|------|
| **需求背景** | 用户作为 OS 方，正在为自有操作系统铺设上层应用生态，图片查看器是其中一环 |
| **落地目标** | 打造一款跨平台桌面端图片查看工具，作为 OS 自带应用提供给终端用户 |
| **业务价值** | 补齐 OS 基础应用矩阵，让用户无需安装第三方工具即可完成每日看图需求 |

---

## 二、需求范围

### ✅ 明确包含

1. 图片查看（打开、浏览、缩放、旋转、翻转）
2. 同目录图片连续浏览（上一张/下一张）
3. 基础文件操作（删除、重命名、复制、在文件夹中显示）
4. 打印图片
5. Windows / Mac / Linux 三个桌面平台

### ❌ 明确不包含（保持板正）

| 排除项 | 原因 |
|--------|------|
| 图片编辑（裁剪、调色、滤镜） | 花里胡哨 |
| 标注/涂鸦 | 花里胡哨 |
| AI 修图/美化 | 花里胡哨 |
| 拼图/相册制作 | 花里胡哨 |
| 图片转 PDF | 花里胡哨 |
| 云同步/账号体系 | OS 自带应用无需账号 |
| 图片管理（标签、分类、搜索） | 那是文件管理器的职责 |

> ⚠️ 以上排除项为推断，请确认。

## 三、目标用户画像

| 维度 | 描述 |
|------|------|
| **用户类型** | 普通终端用户 |

## 五、需要补确认的关键问题

在我继续深挖之前，以下问题直接影响需求边界，请优先回答：

**Q1**：上面 4.1～4.4 的功能清单，哪些要？哪些不要？有没有遗漏？（直接说"全要，再加一个 xxx"也行）

**Q2**：支持的图片格式？最低限度要哪些？
- JPEG / PNG / BMP / GIF / WebP（静态）— 这是基本盘
- SVG — 矢量图，需要吗？
- RAW（CR2/NEF/ARW 等）— 摄影师才会用，普通用户基本不碰，建议不要

**Q3**：GIF 动图怎么处理？
- A) 不区分，打开就显示第一帧
- B) 自动播放动画
- C) 支持播放/暂停控制

**Q4**：多窗口还是单窗口？
- A) 每次双击图片打开一个新窗口
- B) 始终只有一个窗口，新图片替换当前显示
- C) 有设置项让用户自己选

**Q5**：缩略图/胶片条？
- 底部或侧边是否需要一个同目录图片的缩略图导航条？这是很多图片查看器的标配，但会牺牲一点界面简洁度
