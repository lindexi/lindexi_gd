# 交互大屏设备操控

## 概述

此 Skill 允许你连接并操控教室中的交互大屏设备（一体机、电子黑板、交互白板等）。支持的典型操作包括：打开或切换应用、调整系统显示设置、管理文件、播放媒体、诊断故障等。具体能力取决于每台设备的 Agent.md 说明书。

## 前置条件

- 设备屏幕角落显示有连接二维码（常驻）
- 设备已联网
- 用户在设备附近（需要手动在设备屏幕上点击确认）

## MCP 服务

| 项目 | 值 |
|---|---|
| 地址 | `https://mcp.lindexi.site/mcp` |

## 工作流

### Step 1：引导用户扫码

请用户打开摄像头，扫描大屏屏幕角落的连接二维码。

> 二维码约每 60 秒刷新一次，如果扫码后提示过期，请重新扫描。

### Step 2：解析二维码

二维码内容为纯文本，请从中提取以下三个字段：

- **设备ID**：`设备ID：xxx` 后的内容
- **连接码**：`连接码：xxx` 后的内容（GUID 格式）
- **服务器**：`服务器：xxx` 后的内容（备用，通常与上文 MCP 地址一致）

如果无法提取到以上三个字段，说明用户扫描的不是设备连接码，请提示用户确认。

### Step 3：发起连接请求

调用 MCP Tool `request_connection`：

```
request_connection(deviceId, code)
```

**返回状态处理：**

| 返回状态 | 含义 | 你的回应 |
|---|---|---|
| `pending` | 请求已送达，等待用户在设备上确认 | 告诉用户："请在大屏上点击「允许」完成连接授权。" |
| `active` | 设备已确认，连接建立 | 获得 sessionId。告诉用户："已成功连接设备。" 然后进入 Step 4。 |
| `queued` | 前面有其他请求在排队 | 告诉用户："当前有其他连接请求正在处理，请稍候..." |
| `code_expired` | 连接码已过期 | 告诉用户："二维码已过期，请重新扫描大屏上的二维码。" |
| `device_offline` | 设备当前离线 | 告诉用户返回的最后在线时间，建议稍后重试。 |
| `rejected` | 设备拒绝了连接 | 告诉用户："设备拒绝了连接请求，如需连接请重新扫码并在设备上点击允许。" |
| `request_timeout` | 设备确认超时 | 告诉用户："设备端确认超时（2分钟内未点击），如需连接请重新扫码。" |
| `device_busy` | 设备正被其他人操控 | 告诉用户设备当前被谁操控，建议稍后再试。 |
| `too_many_requests` | 请求过于频繁 | 告诉用户稍后再试。 |

### Step 4：读取设备能力说明书

连接成功后，调用 MCP Tool `read_agent_md`：

```
read_agent_md(sessionId)
```

返回该设备的 Agent.md 说明书，描述设备身份、能力列表、每种操作的调用方式、已知限制和异常处理方式。

**仔细阅读 Agent.md**，你将根据其中的能力描述来执行后续操作。

### Step 5：执行操作

根据用户意图和 Agent.md 中的能力描述，选择合适的方式执行：

**方式 A（推荐）：结构化指令**

当 Agent.md 中明确描述了某操作的调用方式时，使用 `execute_command`：

```
execute_command(sessionId, command)
```

其中 `command` 的结构应符合 Agent.md 中的定义。

**方式 B：自然语言转发**

当用户的意图无法精确匹配 Agent.md 中的结构化操作时，使用 `execute_natural_language`：

```
execute_natural_language(sessionId, intent)
```

设备侧会自行理解并处理该自然语言意图（需设备支持 Device Agent 模式）。

**注意：**
- 部分操作可能被标记为"敏感"，执行前设备会弹出二次确认框。告知用户留意设备屏幕。
- 指令执行超时（30 秒）时会返回 `command_timeout`，可建议用户重试。
- 指令执行失败时会返回具体原因（如"权限不足"），请将原因转述给用户。

### Step 6：断开连接

用户操作完毕后，调用 MCP Tool `disconnect`：

```
disconnect(sessionId)
```

会话也会在闲置 10 分钟后自动断开。

## 重要提醒

1. **交互结果的反馈主要来自设备屏幕本身**。比如用户说"播放视频"，用户看的是设备有没有播放，而非等待手机上的反馈。你的角色是指令的中转者，不需要过度描述执行过程。
2. **连接码为一次性使用**，有效期短（90 秒）。如遇过期，引导用户重新扫码即可。
3. **同一设备同时只能有一个活跃会话**。如果返回 `device_busy`，告知用户当前操控者名称。
4. **设备端确认是必须的**。没有用户在设备上的物理确认，连接无法建立。这是安全设计，不是故障。
5. **Agent.md 是动态的**。设备被使用越多，补充的经验越多。每次连接后都建议重新读取 Agent.md 获取最新能力描述。
