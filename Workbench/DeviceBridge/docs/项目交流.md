# 我想做一件事：让 AI 能操控任何设备

## 一句话说清楚

你有没有想过——以后你走进教室，对着手机说一句"上课"，一体机自动打开、课件加载好、音量调到合适。你不用碰设备，不用学菜单，不用找设置。

我想做的就是这件事：**设备和 AI 之间的那一层东西**。让任何 AI App 都能理解设备、操控设备。用户只跟自己的 AI 说话，不需要学设备。

## 为什么是我来做

我们现在手里有一千万台设备。设备上的软件我能控制。这本身就是一张牌——我不需要去求厂商预装，我自己就能把程序推上去。

另一方面，我不会去做手机端的 AI App。那是豆包、DeepSeek、小米、华为的赛道，我没兴趣也没必要去碰。我要做的是让他们来对接我——我提供一个开放的服务，他们接入就能操控设备。

## 大概怎么做

整体上三块东西：

**第一，每台设备上跑一个小的程序程序。** 我叫它设备连接器。它做三件事：在屏幕角落显示一个二维码、接收指令并真正去执行、以及告诉别人"我是谁、我能做什么"。这个东西用 C# 写，编译成单个可执行文件，不依赖运行环境，Windows 和 Linux 都能跑。体积极小。

**第二，云上跑一个服务器。** 设备连上来，手机 Agent 也连上来，服务器在中间做中转。对外提供 MCP 协议接口（现在 AI Agent 生态都在往这个方向靠），同时也提供普通的 HTTP 接口做兜底。

**第三，一份 Skill.md 文件。** 这个文件投放到各个 AI 平台的 Skill 广场——DeepSeek、豆包、Kimi，有哪个投哪个。任何 AI App 加载了这份 Skill，就自动获得了操控设备的能力。Skill 会告诉 Agent：怎么扫码连接设备、怎么读设备说明书、怎么发指令。

## 用户使用起来是什么样

想象你是老师，站在教室里：

1. 大屏角落有个二维码，常驻在那里
2. 你打开手机上的 AI App（你平时用的那个，豆包也好 DeepSeek 也好），扫一下
3. 大屏弹出一个框："李老师想要连接此设备，允许吗？"
4. 你走过去点一下"允许"
5. 然后你就可以对着手机说话了——"帮我把展台打开""屏幕颜色不太对帮我调一下""播放 U 盘里的课件"

你从头到尾没有翻过任何设置菜单，没有找过任何功能入口。你只跟你的 AI 说话，设备就动了。

## 技术上的关键设计

### 连接方式：为什么不走局域网直连？

因为我不想碰手机端。走局域网就意味着我必须在手机上做点什么——至少得有个 SDK 或者 App。这条路我走不通。

所以我的方案是所有流量走中央服务器。设备连服务器，手机连服务器，服务器在中间转发。代价是延迟会高一点，但对于这个场景来说完全够了——用户看的是大屏有没有反应，不是盯着手机等毫秒级响应。

### 二维码里放什么

纯文本，不搞 URL。内容大概长这样：

```
【设备连接码】
请使用AI助手扫描此二维码以连接本设备。
设备ID：SC-2025-001
连接码：a7f3e9c2-4b1d-48e6-9a05-2d8f7c3e1b6a
服务器：https://mcp.lindexi.site
```

AI 扫了能解析（现在的 LLM 解析这种结构化文本毫无压力），普通人扫了也能看明白这是个什么东西。二维码每 60 秒刷新一次，里面的连接码是 GUID，用过就作废。

### 安全怎么做的

没有 API Key，完全开放调用。安全性不靠认证，靠两层物理关卡：

第一层：连接码 90 秒过期，而且是一次性的。扫了不用就作废。

第二层：就算有人扫了码，设备上会弹确认框，必须有人在设备跟前点"允许"。学生调皮扫了码——那他也得敢走过去点教室前面的电子黑板才行。

敏感操作（比如系统重置）还会再加一层：Agent.md 里标记为敏感，执行前设备再弹一次确认。

### 两种执行模式

模式 A：AI App 读完设备的 Agent.md 说明书，自己推理出要执行什么操作，发结构化指令过来，设备端只负责执行。这种模式不烧设备的 Token，推理成本在 AI App 那边。

模式 B：AI App 直接把用户的自然语言转发过来，设备侧自己理解意图、自己执行。这种模式上限高——没有比厂商自己更懂自己产品的了。但每台设备都要烧 Token。

两个模式不是非此即彼，可以共存。先做 A，B 作为可选项。

## 我考虑过的一些麻烦事

说几个你可能也会想到的问题：

**"设备离线了怎么办？"** 设备每 60 秒向服务器报一次到。超过 3 个周期没动静就标记离线。有人扫码连接的时候，服务器会告诉对方"这设备最后在线是 3 分钟前"。设备恢复后自动重新注册，不需要人工干预。

**"万一有人恶意不停扫码骚扰设备？"** 每个设备每 30 秒最多接受 3 个连接请求，超了直接拒绝。设备上也可以开一个"屏蔽连接请求 5 分钟"的开关。

**"千万台设备同时重启，服务器扛得住吗？"** 每台设备的重连加一个随机延迟（0 到 30 秒），不会同时冲过来。服务器跑在 Kubernetes 上，自动扩容。

**"设备系统时间不准怎么办？"** 连接码的过期时间以服务器时间为准，不依赖设备时间。

**"学生扫了码，在手机上能看到设备能做什么，但还没连上——这算不算信息泄露？"** 不存在的。扫码只是拿到了设备 ID 和临时连接码，Agent.md 说明书要等连接建立之后才能读到。
