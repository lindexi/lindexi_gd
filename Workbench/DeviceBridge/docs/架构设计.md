# DeviceBridge 架构设计

## 概述

DeviceBridge 要做的事情很简单：**让用户的 AI App 能操控任何设备**。它是设备与 AI 之间的 "USB"——不做更好的设备，做设备与 AI 之间的通用语言。

用户对着自己的 AI App 说话，AI App 通过 DeviceBridge 操控物理设备。用户不需要学习设备。

## 背景与约束

- **现有资源**：一千万台教育领域交互大屏，设备端软件完全可控
- **不做的事**：不碰手机端 Agent，那是大厂的赛道
- **核心策略**：中央服务器作为唯一中继，设备与手机 Agent 均通过公网服务器通信。不依赖局域网直连
- **分发策略**：提供 Skill.md 文件投放到各 Agent 平台的 Skill 广场，让第三方 Agent（DeepSeek、豆包等）对接

## 角色

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  AI App      │────▶│  中央服务器   │◀────│   设备端      │
│ (第三方Agent) │     │ (DeviceBridge│     │ (交互大屏等)  │
│              │     │  Server)     │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
  用户手机侧           你来搭建和控制        你来控制
```

| 角色 | 说明 | 谁来做 |
|---|---|---|
| AI App | 用户侧 AI 端点。理解用户意图，调用设备能力 | 第三方（DeepSeek、豆包等） |
| DeviceBridge Server | 中央服务器。设备注册、连接配对、指令中转、Agent.md 分发 | 我们 |
| 设备连接器 (Device Connector) | 设备侧运行时。被发现、暴露能力、执行指令 | 我们 |
| 设备 | 物理硬件与其操作系统 | 厂商（已有的） |

## 核心组件

### DeviceBridge Server（中央服务器）

```
┌─────────────────────────────────────────┐
│           DeviceBridge Server            │
│                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ MCP 服务  │  │ HTTP API │  │ Web 页 │ │
│  │ (AI Agent)│  │ (通用)   │  │(浏览器)│ │
│  └────┬─────┘  └────┬─────┘  └───┬────┘ │
│       └──────────────┼─────────────┘      │
│              ┌───────┴───────┐            │
│              │   会话管理    │            │
│              └───────┬───────┘            │
│       ┌──────────────┼──────────────┐     │
│  ┌────┴────┐  ┌──────┴──────┐ ┌────┴───┐ │
│  │设备注册 │  │ Code 映射   │ │Agent.md│ │
│  │& 心跳   │  │ (Redis)    │ │ 存储   │ │
│  └─────────┘  └────────────┘ └────────┘  │
└─────────────────────────────────────────┘
```

- **MCP 服务**：面向 AI Agent 的标准 MCP 协议接口
- **HTTP API**：面向非 MCP Agent 的等价 REST 接口
- **设备管理**：设备注册、心跳、在线状态
- **Code 映射**：维护 (deviceId, code) → 设备 WebSocket 连接的映射（Redis，TTL 90s）
- **Agent.md 存储**：设备上报的能力描述文件，供 Agent 拉取
- **会话管理**：连接配对、会话生命周期、权限

### 设备连接器（Device Connector）

```
┌─────────────────────────────────────────┐
│            Device Connector              │
│           (C# .NET Native AOT)           │
│                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ 二维码    │  │ HTTP     │  │WebSocket│ │
│  │ 生成器    │  │ Refresh  │  │ 客户端   │ │
│  └──────────┘  └──────────┘  └────────┘ │
│                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ 确认 UI  │  │ 执行器    │  │Agent.md│ │
│  │          │  │          │  │ 管理   │ │
│  └──────────┘  └──────────┘  └────────┘ │
│                                          │
│  ┌──────────────────────────────────┐   │
│  │      可选：Device Agent           │   │
│  │   (设备侧自然语言推理，模式B)      │   │
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

- **二维码生成器**：每 60s 生成新二维码，内容为含 deviceId + GUID 连接码 + 服务器地址的纯文本
- **HTTP Refresh**：每 60s 向服务器 POST 刷新连接码，顺风车接收连接请求
- **WebSocket 客户端**：活跃会话期间建立，用于实时指令收发
- **确认 UI**：收到连接请求时弹出确认框（允许/拒绝，120s 超时）
- **执行器**：接收结构化指令，映射为系统调用、进程管理、文件操作等。只执行不推理
- **Agent.md 管理**：启动时上报设备能力描述文件给服务器
- **Device Agent（可选）**：模式 B——接收自然语言意图，设备侧自行推理执行

## 数据流

### 阶段一：设备上线（持续）

```
设备                      服务器
 │                         │
 │── POST /refresh ──────▶ │  每 60s，携带 deviceId + code
 │   { deviceId, code }    │  服务器更新 Redis 映射，TTL=90s
 │                         │
 │◀── 200 OK ──────────── │
 │   { status: "ok" }      │  或 status: "pending_request"（有人请求连接）
 │                         │
```

### 阶段二：连接建立

```
用户手机 (Agent)           服务器                   设备
 │                         │                         │
 │  用户扫码                │                         │
 │  解析二维码文本           │                         │
 │  提取 deviceId + code    │                         │
 │                         │                         │
 │── request_connection ──▶│                         │
 │   {deviceId, code}      │                         │
 │                         │── 校验 code              │
 │                         │── 标记 code 已使用       │
 │                         │                         │
 │◀── status: "pending" ──│                         │
 │                         │                         │
 │                         │  (下一次 refresh 响应)    │
 │                         │── pending_request ─────▶│
 │                         │   {requestId, userName}  │
 │                         │                         │
 │                         │                     设备弹确认框
 │   Agent 提示:            │                     等待用户点击
 │   "请在大屏上点击       │                         │
 │    「允许」完成连接"     │                         │
 │                         │                         │
 │                         │◀── approve ────────────│
 │                         │                         │
 │                         │── 创建 session          │
 │                         │                         │
 │◀── status: "active" ──│                         │
 │   { sessionId }         │                         │
 │                         │                         │
 │                         │       WebSocket 连接     │
 │                         │◀═══════════════════════▶│
 │                         │    (设备进入活跃态)       │
```

### 阶段三：指令执行

```
Agent                    服务器                    设备
 │                         │                         │
 │── execute_command ────▶│                         │
 │   {sessionId, command}  │                         │
 │                         │── WebSocket push ──────▶│
 │                         │   {command}              │
 │                         │                         │
 │                         │                   设备执行指令
 │                         │                   (调系统API等)
 │                         │                         │
 │                         │◀── result ─────────────│
 │                         │                         │
 │◀── result ────────────│                         │
 │   {success, output}     │                         │
 │                         │                         │
```

### 阶段四：断开

```
Agent                    服务器                    设备
 │                         │                         │
 │── disconnect ─────────▶│                         │
 │                         │── close session         │
 │                         │── close WebSocket ─────▶│
 │                         │                         │
 │                         │                   设备回空闲态
 │                         │                   (HTTP refresh)
```

## 技术选型

| 组件 | 技术 | 说明 |
|---|---|---|
| 服务器运行时 | .NET (C#) | Kestrel + 容器化部署 |
| 设备连接器 | .NET Native AOT | 无运行时依赖，跨 Windows/Linux |
| 设备↔服务器（空闲） | HTTP POST | 每 60s 刷新连接码 |
| 设备↔服务器（活跃） | WebSocket | 实时指令收发 |
| Agent↔服务器 | MCP / HTTP | MCP 优先，HTTP 兜底 |
| Code 映射存储 | Redis | key TTL 自动过期 |
| 设备认证 | mTLS | 设备预置客户端证书 |

## 方案名

**DeviceBridge** —— 设备之桥。解决方案 `DeviceBridge.sln`，项目分层：
- `DeviceBridge.Server` —— 中央服务器
- `DeviceBridge.Device` —— 设备端连接器
- `DeviceBridge.Protocol` —— 通信协议定义（共享类型）
