# DeviceBridge 通信协议设计

## 概述

DeviceBridge 涉及两条通信链路：

1. **设备 ↔ 服务器**：设备端连接器与中央服务器之间
2. **Agent ↔ 服务器**：第三方 AI App 与中央服务器之间

## 一、设备 ↔ 服务器

### 1.1 空闲态：HTTP Refresh

设备每 60s 向服务器发送 HTTP POST 请求刷新连接码。服务器在响应中顺风车带回待处理的连接请求。

**请求：**

```
POST /api/device/refresh
Content-Type: application/json

{
    "deviceId": "SC-2025-001",
    "code": "a7f3e9c2-4b1d-48e6-9a05-2d8f7c3e1b6a",
    "timestamp": "2026-07-01T08:30:00Z",
    "version": "1.0.0"
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| deviceId | string | 设备唯一标识 |
| code | string (GUID) | 当前连接码，60s 刷新一次 |
| timestamp | string (ISO 8601) | 设备端时间戳，用于服务器检测时钟偏差 |
| version | string | 设备连接器版本号 |

**响应（无待处理请求）：**

```json
{
    "status": "ok",
    "codeExpiresIn": 90,
    "serverTime": "2026-07-01T08:30:01Z"
}
```

**响应（有待处理连接请求）：**

```json
{
    "status": "pending_request",
    "requestId": "req-a1b2c3d4",
    "userName": "李老师",
    "requestedAt": "2026-07-01T08:29:45Z",
    "message": "有人请求连接此设备",
    "codeExpiresIn": 90,
    "serverTime": "2026-07-01T08:30:01Z"
}
```

设备收到 `pending_request` 后弹出确认框。

**错误响应：**

```json
{
    "status": "error",
    "error": "device_not_registered",
    "message": "设备未注册，请联系管理员"
}
```

### 1.2 确认连接请求

用户在设备确认框点击"允许"或"拒绝"后，设备通过 HTTP POST 回复服务器。

**请求：**

```
POST /api/device/respond
Content-Type: application/json

{
    "deviceId": "SC-2025-001",
    "requestId": "req-a1b2c3d4",
    "approved": true
}
```

**响应：**

```json
{
    "status": "ok",
    "sessionId": "sess-x1y2z3w4",
    "websocketUrl": "wss://mcp.lindexi.site/ws/device?sessionId=sess-x1y2z3w4"
}
```

设备收到响应后，建立 WebSocket 连接进入活跃态。

### 1.3 活跃态：WebSocket

**连接：**

```
wss://mcp.lindexi.site/ws/device?sessionId=sess-x1y2z3w4
```

**消息格式（双向）：**

```json
{
    "type": "message_type",
    "sessionId": "sess-x1y2z3w4",
    "timestamp": "2026-07-01T08:30:15Z",
    "payload": { }
}
```

**服务器 → 设备（执行指令）：**

```json
{
    "type": "execute",
    "sessionId": "sess-x1y2z3w4",
    "commandId": "cmd-001",
    "timestamp": "2026-07-01T08:30:15Z",
    "payload": {
        "action": "open_app",
        "target": "展台",
        "params": {}
    }
}
```

**设备 → 服务器（执行结果）：**

```json
{
    "type": "execute_result",
    "sessionId": "sess-x1y2z3w4",
    "commandId": "cmd-001",
    "timestamp": "2026-07-01T08:30:17Z",
    "payload": {
        "success": true,
        "output": "展台已打开",
        "error": null
    }
}
```

**服务器 → 设备（会话结束）：**

```json
{
    "type": "session_close",
    "sessionId": "sess-x1y2z3w4",
    "timestamp": "2026-07-01T08:35:00Z",
    "payload": {
        "reason": "user_disconnected"
    }
}
```

### 1.4 WebSocket 心跳

```
设备 → 服务器: { "type": "ping" }
服务器 → 设备: { "type": "pong" }
```

每 30s 一次。超过 60s 无心跳视为断开。

### 1.5 设备上报 Agent.md

设备启动时或 Agent.md 变更时上报：

```
POST /api/device/agent-md
Content-Type: application/json

{
    "deviceId": "SC-2025-001",
    "agentMd": "# 设备说明书\n\n## 设备身份\n...",
    "version": 3
}
```

## 二、Agent ↔ 服务器

### 2.1 MCP 接口

MCP 服务地址：`https://mcp.lindexi.site/mcp`

**Tools：**

| Tool | 参数 | 返回 | 说明 |
|---|---|---|---|
| `request_connection` | deviceId, code | sessionId / pending / error | 发起连接请求 |
| `check_connection_status` | sessionId | status | 查询连接状态 |
| `read_agent_md` | sessionId | agentMd 文本 | 读取设备能力描述 |
| `execute_command` | sessionId, command | result | 执行结构化指令 |
| `execute_natural_language` | sessionId, intent | result | 自然语言指令（转发给 Device Agent） |
| `disconnect` | sessionId | — | 断开连接 |

### 2.2 HTTP API（非 MCP 降级）

功能等价，REST 风格：

```
POST /api/v1/connect          # request_connection
GET  /api/v1/session/{id}     # check_connection_status
GET  /api/v1/session/{id}/agent-md  # read_agent_md
POST /api/v1/session/{id}/execute    # execute_command
POST /api/v1/session/{id}/nl-execute # execute_natural_language
DELETE /api/v1/session/{id}   # disconnect
```

### 2.3 request_connection 响应状态

| status | 含义 | Agent 行为 |
|---|---|---|
| `pending` | 请求已送达设备，等待用户确认 | 提示"请在设备上点击「允许」" |
| `active` | 设备已确认，连接建立 | 获得 sessionId，可开始操作 |
| `queued` | 设备正处理其他请求，当前请求排队 | 提示"前面还有 N 个请求，请稍候" |
| `code_expired` | 连接码已过期 | 提示"二维码已过期，请重新扫描" |
| `device_not_found` | 设备不存在 | 提示"未找到该设备" |
| `device_offline` | 设备当前离线 | 提示"设备当前离线，最后在线于 xx 分钟前" |
| `device_busy` | 设备正被其他人操控 | 提示"设备正被 xx 操控中" |
| `rejected` | 设备拒绝了连接 | 提示"设备拒绝了连接请求" |
| `request_timeout` | 等待确认超时（120s） | 提示"设备端确认超时，请重新扫码" |
| `too_many_requests` | 请求过于频繁 | 提示"请求过于频繁，请稍后再试" |

## 三、二维码格式

二维码编码为纯文本，格式如下：

```
【设备连接码】
请使用AI助手扫描此二维码以连接本设备。
设备ID：SC-2025-001
连接码：a7f3e9c2-4b1d-48e6-9a05-2d8f7c3e1b6a
服务器：https://mcp.lindexi.site
操作说明：将设备ID和连接码发送到上述服务器，即可建立连接。
```

Agent 解析时按关键词定位：`设备ID`、`连接码`、`服务器`。

## 四、安全

| 链路 | 方案 |
|---|---|
| 设备 → 服务器（HTTP） | mTLS，设备预置客户端证书 |
| 设备 → 服务器（WebSocket） | TLS，基于已有 session 认证 |
| Agent → 服务器 | TLS，完全开放，无需 API Key |
| 指令安全 | Agent.md 中标记敏感操作，执行前设备端二次弹窗确认 |
| 防重放 | code 一次性使用 + TTL 90s |
| 防枚举 | 无设备列表接口，找不到设备时统一返回 `device_not_found` |
