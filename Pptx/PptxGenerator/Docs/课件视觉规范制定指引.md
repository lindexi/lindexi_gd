# 课件视觉规范制定指引

## 你的角色

你要做的是：通读课件 Markdown 内容 → 理解课件结构 → 从零定义视觉规范 → 输出页面类型骨架。

样本中已有的字号、字体、颜色、布局**不需要遵守**。你的工作是立法，不是考古。

---

## 第一步：通读内容，理清结构

### 1.1 确认画布尺寸

从 Markdown 页面元数据中读取页面尺寸（宽 × 高，px）。这是唯一必须从样本中提取的参数。如果样本中未写明，要求提供方补充。

### 1.2 通读全部页面的内容

只看内容本身，不看原来怎么排的。回答三个问题：

**Q1：这个课件里出现了哪几种内容形态？**

例如：大段叙述、对话句型、单词列表、练习题、歌曲/韵文、图文配对……

**Q2：哪些页面内容过多，一页放不下？**

判断标准：在基准正文字号下，内容超过画布可排区域 80% → 需要拆分。

**Q3：哪些页面内容太少，一页撑不起来？**

判断标准：在基准正文字号下，内容不足画布可排区域 30% → 需要合并或丰富。

### 1.3 重新组织页面

**不要只提问题，直接动手改。** 因为后续没有交互环节，等反馈等于没反馈。

对**内容过多**的页面：
- 找到自然的分割点（话题切换、角色切换、步骤切换）。
- 拆成多页，每页一个主题。
- 页面类型可能随之变化——原来混在一起的内容，拆开后各自归入不同类型。

对**内容过少**的页面：
- 合并到相邻页面；或者
- 如果有配图空间，标注「需要补充插图」，页面类型改为图文页。

对**类型混乱**的页面：
- 一个页面里如果塞了「标题 + 正文 + 练习 + 词汇」四种内容，拆成多页。
- 每种内容形态对应一种页面类型。

### 1.4 输出页面类型清单

经过重组后，确定最终的页面类型清单。常见的页面类型包括（按需增减）：

- 封面页
- 内容页（单栏）
- 内容页（双栏对话）
- 练习/选择题页
- 总结/复习页
- ……

每种类型附一句话说明其用途。这份清单就是后续骨架定义的目录。

---

## 第二步：从零定义视觉规范

以下所有决策，**不从样本继承**。样本只告诉你「有哪些内容需求」，不告诉你「应该长什么样」。

### 2.1 定义字号阶梯

以正文为基准，记为 L3 = 1x。

**选定基准正文大小：**

在基准画布下，正文（L3）的 px 值。经验范围为画布宽度的 3.5% ~ 4.5%。例如 1280 宽的画布，正文约 45~58px。

**定义五个等级：**

```
L1 — 封面主标题       = L3 × 1.5 ~ 2.0
L2 — 页面标题          = L3 × 1.1 ~ 1.3
L3 — 正文 / 句型       = 1x（基准）
L4 — 辅助文字 / 词汇   = L3 × 0.8 ~ 0.9
L5 — 微文字 / 页码     = L3 × 0.6 ~ 0.75
```

**每个等级必须写明用途，不允许出现「视情况而定」。** 例如：

```
L1：仅封面页主标题
L2：内容页顶部标题、章节过渡页标题
L3：正文、句型、对话、列表内容
L4：词汇条、图片说明、注释
L5：页码、版权信息
```

**为什么用等级而不是 px：**

一、换画布尺寸时等比缩放。规范只写 L1~L5，渲染端负责将等级换算为当前画布的实际 px。

二、同一等级在不同页面类型里保持一致。页面标题无论在内容页还是练习页都用 L2，不会出现这页比那页大一号。

三、限制执行者自由发挥。执行者只能在 L1~L5 里选字号，不能自己造。

### 2.2 定义安全区

安全区 = 页面内部四条不可见的边界线。所有可读内容（文字、图片主体）必须在安全区内。背景色块、装饰元素可以超出。

**为什么需要：** 不同执行者做不同页面，如果没有统一边距，翻页时内容会上下左右跳动。

定义上下左右四条边界，给出基于基准画布的 px 值。经验值：上边距 40~80px，下边距 30~60px，左右边距 60~100px。

输出示例：

```
基准画布 1280×720
安全区：上 60、下 40、左 80、右 80
```

### 2.3 定义字体栈

字体由你自行选择，不从样本继承。

**要求：**

- 区分标题字体、正文字体、代码字体（如课件不含代码，代码字体可省略）。
- 每种字体必须给出 fallback 序列。
- 英文字体在前，中文字体在后，以通用字体族（sans-serif / serif / monospace）收底。
- 选字体时考虑：是否在多数设备上可用、是否适合目标年龄段。

输出示例：

```
标题：Arial Rounded MT Bold, PingFang SC, Microsoft YaHei, sans-serif
正文：Arial, PingFang SC, Microsoft YaHei, sans-serif
代码：JetBrains Mono, Consolas, monospace
```

### 2.4 定义配色

配色由你自行选择，不从样本继承。

**必须覆盖以下色域：**

**① 主色（1 个）** — 页面标题、重点强调。

**② 辅助色（1 个）** — 次要强调、图表第二色。

**③ 强调色（1 个）** — 极少使用，仅用于最关键的高亮（如正确答案）。

**④ 中性色阶梯：**

| 用途 | 说明 |
|------|------|
| 最深文字 | 标题用 |
| 正文文字 | 内容、句型、对话 |
| 辅助文字 | 注释、页码 |
| 分割线 / 弱边框 | 分隔元素 |
| 浅底色 | 卡片背景、词卡底板 |
| 页面底色 | 画布背景 |

**⑤ 语义色**（有练习/测验页时必须有）：

| 用途 | 说明 |
|------|------|
| 正确 | 正确答案、通过 |
| 错误 | 错误答案、不通过 |

**每个颜色输出时，必须包含：** hex 值 + 用途说明 + 不得用于什么场景。例如：

```
主色：#4A7BF7
用途：页面标题、重点强调
禁用：不得用于正文、不得大面积为背景
```

## 第三步：为每种页面类型写 SlideML 骨架

对第一步确定的每一种页面类型，输出一个完整但内容留空的 SlideML 模板。

模板中使用以下标签：

**`<Page>`** — 一个页面，最外层容器。`Background` 属性指定页面底色。

**`<Panel>`** — 一个矩形区域，用于划分页面区块。可嵌套。支持两种定位方式：
- 绝对定位：写 `X`、`Y` 精确控制位置。
- 流式布局：`Layout="Horizontal"` 或 `Layout="Vertical"`，子元素自动排列，无需手动算坐标。

**优先使用流式布局。** 因为执行者填入的内容长短不一，流式布局会自动适应，不容易出错。

**`<TextElement>`** — 文本。核心属性：`Text`（内容，骨架中写注释占位）、`FontSize`（写 L1~L5）、`FontName`（写完整字体栈）、`Foreground`（写 hex）、`TextAlignment`。

**`<Rect>`** — 矩形色块，用于卡片底板、词卡背景等。

**`<Image>`** — 图片。`Source` 写资源 ID。

**模板要求：**

- `Id` 属性使用 PascalCase（如 `SlotTitle`、`SlotContentLeft`、`SlotVocabBar`）。
- `FontSize` 写等级（L1~L5），不写 px。
- `Foreground`、`Fill`、`Background` 写 hex 值。
- `FontName` 写完整字体栈。
- 内容用 XML 注释 `<!-- 此处填入 XXX -->` 占位。
- 所有标签名和属性名使用 PascalCase。

**示例——Summary（总结/复习页）的骨架：**

```xml
<!-- 页面类型：Summary（总结页） -->
<!-- 结构：顶栏 + 双栏主体 + 底栏词卡 -->
<Page Background="#FFFFFF">

  <!-- === 顶部标题栏 === -->
  <Panel Id="SlotTopBar" X="0" Y="0" Width="960" Height="80"
         Background="#F5F5F5">
    <Image Id="SlotLogoLeft" X="24" Y="4" Width="87" Height="77"
           Source="<!-- 此处填入 logo 资源 ID -->" />
    <TextElement Id="SlotTitle" X="91" Y="10" Width="778"
                 Text="<!-- 此处填入标题文本 -->"
                 FontName="Arial Rounded MT Bold, PingFang SC, Microsoft YaHei, sans-serif"
                 FontSize="L2"
                 IsBold="True"
                 Foreground="#1A1A1A"
                 TextAlignment="Center" />
    <Image Id="SlotLogoRight" X="877" Y="10" Width="82" Height="35"
           Source="<!-- 此处填入 logo 资源 ID -->" />
  </Panel>

  <!-- === 双栏主体 === -->
  <Panel Id="SlotContent" Layout="Horizontal" Gap="80"
         X="80" Y="100" Width="800" Height="380">
    <Panel Id="SlotContentLeft" Layout="Vertical" Gap="16"
           Width="310" VerticalAlignment="Center">
      <TextElement Text="<!-- 此处填入左栏句型 -->"
                   FontName="Arial, PingFang SC, Microsoft YaHei, sans-serif"
                   FontSize="L3"
                   Foreground="#333333" />
    </Panel>
    <Panel Id="SlotContentRight" Layout="Vertical" Gap="16"
           Width="280" VerticalAlignment="Center">
      <TextElement Text="<!-- 此处填入右栏句型 -->"
                   FontName="Arial, PingFang SC, Microsoft YaHei, sans-serif"
                   FontSize="L3"
                   Foreground="#333333" />
    </Panel>
  </Panel>

  <!-- === 底部词卡栏 === -->
  <Panel Id="SlotVocabBar" X="140" Y="500" Width="680" Height="40"
         HorizontalAlignment="Center">
    <Rect Fill="#E8F0FE" CornerRadius="8"
          Width="680" Height="40" />
    <TextElement Text="<!-- 此处填入词汇，空格分隔 -->"
                 FontName="Arial, PingFang SC, Microsoft YaHei, sans-serif"
                 FontSize="L4"
                 Foreground="#333333"
                 TextAlignment="Center" />
  </Panel>

</Page>
```

**目标：执行者拿到模板，把注释替换成实际内容，就能直接渲染。** 不需要他们自己决定字号、颜色、间距、对齐。

### （可选）补充元素规格

如果某种视觉元素在多个页面类型中反复出现（≥3 次），单独定义其样式参数。例如：

```
元素名称：词卡底框
类型：Rect
Fill：#E8F0FE
CornerRadius：8
Stroke：无
Shadow：无
```

## 第四步：汇总输出

### 规范文档结构

将以上内容整理为一份规范文档，结构如下：

```
1. 基础参数
   - 基准画布尺寸
   - 安全区（上 下 左 右）
   - 字号等级表（L1~L5 比例 + 基准 px + 用途）
   - 字体栈
   - 主色 hex + 正文色 hex

2. 配色完整方案
   - 主色 / 辅助色 / 强调色
   - 中性色阶梯
   - 语义色

3. 页面类型骨架（每种类型一節）
   - 类型名称 + 适用场景
   - 完整 SlideML 模板
```

### 自查清单

文档输出前，逐项确认：

```
□ 字号是否全部用 L1~L5，没有写死 px？
□ 字体栈是否有 fallback 序列？
□ 配色是否覆盖了主色 + 中性色阶梯 + 语义色？是否每项都有 hex？
□ 安全区是否定义？
□ 每种页面类型是否有 SlideML 骨架？
□ 骨架里的内容是否用 XML 注释占位？
□ 内容过多的页面是否已拆分？内容过少的页面是否已处理？
□ 是否有页面类型被遗漏？
□ 规范文档是否一打开就能看到基础参数？
```
