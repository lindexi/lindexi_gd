# 引入 AI 驱动的中立评估者 + 完整项目重构 —— 实施计划

## 1. 理解

当前 `PptxGenerator` 项目通过 `SlideChatManager` 编排主模型生成 SlideML，但没有质量评估环节。本次任务引入一个 **AI 驱动的中立评估者**，用来：

1. 评估主模型生成的 SlideML 质量（XML 结构、排版合理性）
2. 评估 SlideML 渲染后的视觉效果（通过截图反馈给评估模型）
3. 评估 `BuildSystemPrompt` 和 `BuildInitialUserPrompt` 的提示词质量并给出改进建议
4. 支持自动触发（生成后自动评估）和手动触发两种模式
5. 对整个项目进行重构以支撑上述能力

---

## 2. 假设

- 评估者复用同一个 `CopilotChatManager` + 相同或不同的 AI 模型（通过 `AgentApiEndpointManager` 选择模型）
- 评估者是另一个独立的 AI 调用（独立的 SystemPrompt），作为「第三方裁判」
- 评估反馈形式：**结构化评分（1-10 分多维度） + 自然语言改进建议**
- 提示词的优化建议直接输出到聊天消息中，供开发者查看
- 项目以 Avalonia UI + AgentLib 为基础，重构围绕抽取 Pipeline 模式进行
- `CopilotChatManager` API 稳定，不做修改
- 目标框架 `net10.0`，Nullable 启用

---

## 3. 目标架构

```
User Input
    │
    ▼
┌──────────────────────────────────────────────────┐
│            SlideGenerationPipeline                │
│                                                  │
│  1. IPromptProvider → 构建 System/User Prompt     │
│     │                                             │
│  2. CopilotChatManager → 主模型生成 SlideML       │
│     │  (含 render_slide / get_render_preview)     │
│     │                                             │
│  3. SlideRenderer → 渲染 Bitmap                   │
│     │                                             │
│  4. ISlideEvaluator → 评估 SlideML + 渲染结果     │
│     │  (AI 驱动，多维度评分 + 改进建议)            │
│     │                                             │
│  5. IPromptEvaluator → 评估提示词质量              │
│     │  (AI 驱动，元评估 + 优化建议)                │
│     │                                             │
│  ▼                                                │
│  Output (SlideML + Bitmap + EvaluationResult)     │
└──────────────────────────────────────────────────┘
```

## 6. 评估维度设计

### SlideML 评估维度（1-10 分）

| 维度 | 说明 |
|------|------|
| **XmlWellFormedness** | XML 语法正确性、标签合规性、属性完整性 |
| **LayoutStructure** | 层级结构清晰度、Panel 嵌套合理性 |
| **VisualBalance** | 元素分布均衡性、留白是否充足 |
| **ConstraintAdherence** | 是否遵守 1280x720 画布、颜色格式等约束 |
| **SemanticAlignment** | 内容是否与用户需求对齐 |
| **AestheticQuality** | 配色、字体大小层级、整体美观度 |

### 提示词评估维度（1-10 分）

| 维度 | 说明 |
|------|------|
| **Clarity** | 指令清晰度、无歧义 |
| **Completeness** | 是否覆盖所有必要规则 |
| **ConstraintQuality** | 约束是否有效、无矛盾 |
| **Actionability** | 是否可被模型准确执行 |
| **Redundancy** | 是否存在冗余/重复内容 |

## 7. 实施步骤

### 步骤 1：创建 Evaluation 结果模型
- 新建 `Core/SlideMl/Evaluation/SlideEvaluationResult.cs`
- 新建 `Core/SlideMl/Evaluation/PromptEvaluationResult.cs`
- 使用 `record` 类型，含多维度评分、自然语言建议、时间戳

### 步骤 2：创建 IPromptProvider + SlideMlPromptProvider
- 新建 `Core/SlideMl/Prompt/IPromptProvider.cs`
- 新建 `Core/SlideMl/Prompt/SlideMlPromptProvider.cs`
- 将 `SlideChatManager.BuildSystemPrompt()` 和 `BuildInitialUserPrompt()` 迁移至此

### 步骤 3：创建 ISlideEvaluator + AiSlideEvaluator
- 新建接口和实现
- 构建评估者 SystemPrompt（裁判角色）
- 输入：用户需求 + SlideML XML + 渲染警告 + 渲染截图
- 输出：`SlideEvaluationResult` JSON
- 复用 `CopilotChatManager.SendMessage()` 进行 AI 调用

### 步骤 4：创建 IPromptEvaluator + AiPromptEvaluator
- 新建接口和实现
- 构建元评估 SystemPrompt
- 输入：SystemPrompt + UserPrompt 模板
- 输出：`PromptEvaluationResult`

### 步骤 5：创建 SlideGenerationPipeline 管道编排器
- 新建 `Core/SlideMl/Pipeline/SlideGenerationPipeline.cs`
- 新建 `Core/SlideMl/Pipeline/PipelineContext.cs`
- 管理完整生命周期：生成 → 渲染 → 评估 → 提示词评估
- 事件驱动：`SlideGenerated`、`EvaluationCompleted`、`PromptEvaluationCompleted`

### 步骤 6：重构 SlideChatManager 为管道适配层
- 保留现有公开 API（`SendMessageAsync`、属性等）
- 内部委托给 `SlideGenerationPipeline`
- 新增 `LastEvaluationResult`、`LastPromptEvaluationResult` 属性
- 旧方法标记 `[Obsolete]` 引导迁移

### 步骤 7：重构 MainWindowViewModel
- 适配新管道事件
- 暴露评估结果供 UI 绑定
- 新增「手动触发评估」命令
- 状态栏显示评分摘要

### 步骤 8：重构 SlideCliRunner + Program.cs
- CLI 管道集成，输出评估结果
- DI 装配链更新

### 步骤 9：编译验证
- `dotnet build` 确保零编译错误
- 修复所有编译问题

## 8. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 评估者 AI 调用增加 Token 消耗 | 评估者可用更便宜的模型（如 qwen 系列小模型） |
| 评估 SystemPrompt 初期不够准确 | 先出 MVP，随使用迭代优化评估提示词 |
| 元评估（评估提示词）可靠性 | 元评估建议仅作参考，不自动修改提示词 |
| 重构范围大、可能引入回归 | 保留旧 `SlideChatManager` 公开 API 作为兼容层 |
