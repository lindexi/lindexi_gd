# 流式布局中主轴坐标显式设置的行为设计

## 背景

SlideML V2 规范中，Panel 支持流式布局模式（`Layout="Horizontal"` 或 `Layout="Vertical"`）。当前规范指出"子元素在排列轴方向上的 X 或 Y 被忽略，由引擎自动计算"。但这也引出一个问题：如果用户希望在流式布局中**显式控制某个元素的位置**，应该怎么做？

## 当前实现分析

在 `LayoutFlowPanel` 方法（`SlideMlLayoutEngine.cs`）中，流式布局的处理逻辑如下：

| 轴 | 行为 |
|---|---|
| **主轴**（Horizontal 的 X / Vertical 的 Y） | 始终由引擎计算，忽略元素声明的 X/Y |
| **跨轴**（Horizontal 的 Y / Vertical 的 X） | 通过 `ResolveOrigin` 处理 — 有显式 X/Y 则用之，否则用 `VerticalAlignment`/`HorizontalAlignment` |

当前已支持跨轴的显式坐标，但主轴坐标在流式模式下被完全忽略。

### 回填现状

布局引擎计算出的位置写入 `LayoutBounds`，但 X/Y 属性是 `init` 只读的，XML 回填仅添加 `ActualWidth`/`ActualHeight`/`ActualLineCount`，不更新 X/Y。`LayoutBounds` 是渲染端的权威位置来源。

## 策略对比

### 策略一：脱离流（CSS `position: absolute` 语义）⭐ 推荐

**规则**：流式 Panel 中，如果子元素在**主轴**上设置了显式坐标（Horizontal 的 `X` / Vertical 的 `Y`），则该元素：

1. 使用显式坐标定位（相对于 Panel 内容区原点）
2. **不推进流式游标**（脱离流，不占据流式空间）
3. **不参与 Panel 自动尺寸计算**
4. 跨轴行为保持不变

```xml
<!-- 示例：r2 设置了 X=300，脱离水平流 -->
<Panel Layout="Horizontal" Gap="8" Padding="16">
  <Rect Id="r1" Width="100" Height="50" />           <!-- 流式：X=16 -->
  <Rect Id="r2" X="300" Width="80" Height="50" />     <!-- 绝对：X=300，不推进游标 -->
  <Rect Id="r3" Width="100" Height="50" />            <!-- 流式：X=16+100+8=124 -->
</Panel>
```

**优点**：
- 与 CSS Flexbox 的 `position: absolute` 语义一致，开发者直觉上容易理解
- 不破坏现有流式布局 — 当前主轴 X/Y 本来就被忽略，新增此行为是纯增量
- 可预测：无 X/Y 的元素始终流式排列，有 X/Y 的元素固定定位

**缺点**：
- 脱离流的元素不参与 Panel 尺寸计算，可能需要 Warning 提示

### 策略二：覆盖位置但不脱离流

**规则**：元素有主轴 X/Y 时，使用该坐标定位，但**仍然推进流式游标**（仍占据流式空间）。

**优点**：实现简单
**缺点**：会导致元素重叠，行为不够直观

### 策略三：同时设置 X 和 Y 才脱离流

**规则**：流式 Panel 中，如果子元素**同时**设置了 X 和 Y，则完全使用绝对定位，不参与流式布局。

**优点**：条件更严格，避免误触发
**缺点**：跨轴坐标当前已被支持（如 Horizontal 中设置 Y 做垂直对齐），要求同时设置 X 和 Y 才脱离流会导致语义混乱

## 决策

采用 **策略一（CSS `position: absolute` 语义）**，理由如下：

1. **最小破坏性**：当前主轴 X/Y 被忽略，新行为是纯增量，不影响现有 SlideML 文档
2. **直觉一致**：与 Web 开发者熟悉的 CSS Flexbox 语义对齐
3. **与规范文档兼容**：规范文档中"流式布局中排列轴方向的坐标被忽略"改为"流式布局中排列轴方向的坐标若设置则该元素脱离流式布局"
4. **跨轴行为不变**：已有跨轴 X/Y 支持保持不动

### 实现要点

修改 `LayoutFlowPanel` 方法中的循环逻辑：

```
遍历子元素 → 判断主轴是否有显式 X/Y
  ├─ 是 → 绝对定位：使用显式坐标，不推进 flowPosition，不参与 crossAxisSize 计算
  └─ 否 → 流式定位：使用 flowPosition，推进 flowPosition，参与 crossAxisSize 计算
        ↓
  设置 LayoutBounds → 递归处理子 Panel → 继续下一个元素
```

### 规范文档修改

Panel 流式布局规则章节中"流式布局中排列轴方向的坐标被忽略"修改为：

> - 子元素在排列轴方向上的 X 或 Y 被**忽略**：若不设置 X/Y，元素沿主轴依次排列
> - 子元素在排列轴方向上的 X 或 Y 被**启用**：若设置了 X/Y，该元素**脱离流式布局**，使用显式坐标定位，不占用流式空间，不影响其他流式元素的位置
