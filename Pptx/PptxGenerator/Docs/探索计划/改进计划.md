# SlideML 改进计划：融合 CopilotChatManager

## 目标

将 `SlideGenerationService` 的手动迭代闭环迁移到 `CopilotChatManager` 的聊天流驱动模式，把 SlideML 渲染封装为 AI Tool，UI 改为聊天气泡布局，CLI 一并重构。

---

## 决策回顾

根据澄清问答，已确定以下设计方向：

1. **迭代闭环方式**：改为通过 `CopilotChatManager` 的聊天流来驱动，将 `SlideRenderer.RenderAsync` 做成 AI Tool，让模型在流式对话中自行调用渲染并基于反馈修正。

2. **UI 布局**：改成聊天气泡方式，渲染图在聊天界面中呈现。布局调整为左右分栏：**左侧 = 页面预览 + XML 展示 + 迭代记录**，**右侧 = 聊天气泡列表 + 底部输入框**。去掉顶部的"输入自然语言需求……"面板，用户直接在聊天框中输入需求。

3. **系统提示词**：注入到 `CopilotChatManager`。如果现有 AgentLib 不支持，则修改 AgentLib 库添加通用系统提示词能力。由于 AgentLib 尚未正式发布，不考虑 API 兼容性。

4. **命令行路径**：`SlideCliRunner` 一并重构为使用 `CopilotChatManager`，以方便通过 `dotnet run` 进行实验测试。

5. **工具集**：默认工具（WorkspaceTools + SubAgentTools）保持附加，SlideML 项目特定的工具（渲染页面、后续的测量文本等）额外加上。

---

## 架构调整概要

```
重构前：
  User Prompt → SlideGenerationService（手动迭代最多 4 轮）
                 ├── IChatClient.GetResponseAsync（手动构建 ChatMessage 列表）
                 ├── SlideXmlUtilities.ExtractXml
                 ├── SlideRenderer.RenderAsync
                 └── BuildFeedbackPrompt → 反馈注入下一轮

  UI: MainWindowViewModel → GenerateAsync / ContinueConversationAsync
  CLI: SlideCliRunner → SlideGenerationService

重构后：
  User Prompt → CopilotChatManager（流式聊天 + 工具驱动）
                 ├── 系统提示词（SlideML 排版引擎指令，注入 ChatOptions.Instructions）
                 ├── 默认工具（WorkspaceTools + SubAgentTools）
                 ├── 扩展工具（SlideRenderTool / 文本测量等）
                 └── AI 自行决定是否调用渲染工具，根据结果迭代

  UI: MainWindowViewModel → 绑定 CopilotChatManager 消息集合
  CLI: SlideCliRunner → CopilotChatManager（保留文件输出逻辑）
```

### 步骤 3：PptxGenerator — 创建 SlideRenderTool（AI Tool）

- **文件**：`Code/PptxGenerator/Core/SlideMl/SlideRenderTool.cs`（新建）
- **内容**：
  - 定义 `[AITool]` 静态方法或 `AIFunctionFactory.Create` 委托

### 步骤 4：PptxGenerator — 创建 SlideChatManager

- **文件**：`Code/PptxGenerator/Core/SlideMl/SlideChatManager.cs`（新建）
- **内容**：
  - 内部持有 `CopilotChatManager` 实例（组合模式）

    - 当前预览 Bitmap（由 SlideRenderTool 缓存）
    - 当前 SlideML XML

  - 提供 `SendSlideRequestAsync(string userPrompt)` 便捷方法，内部组装初始提示词后调用 `CopilotChatManager.SendMessageAsync`

  - 将其 `Build系统提示词`、`BuildInitialUserPrompt` 等静态方法保留为 `SlideChatManager` 的内部辅助方法（或公开静态方法）

## 风险与注意事项

1. **线程调度**：`SlideRenderer.RenderAsync` 内部使用 `Dispatcher.UIThread.InvokeAsync`。在 AI Tool 回调中，调用线程不一定是 Avalonia UI 线程。需要在 `SlideRenderTool` 中判断当前线程，若非 UI 线程则通过 `Dispatcher.UIThread.InvokeAsync` 调度执行。

2. **模型工具调用行为**：模型是否会在聊天中主动调用渲染工具，取决于 System Prompt 的引导和初始提示词的设计。需要在 System Prompt 中明确指示"生成 SlideML 后必须调用 render_slide 工具验证"。

3. **迭代收敛**：旧的"最多 4 轮"限制在新模式下不再适用。模型可能自己决定调用多次渲染工具来修正问题。需要通过 System Prompt 引导适可而止。

4. 无

5. **AgentLib 改动**：修改的 CopilotChatManager 等需保持通用性，不能引入 SlideML 特定代码。
