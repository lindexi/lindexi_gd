## 二、当前有效待办

### 2.1 课件资源到本地 WPF 渲染的闭环

当前已具备：

- `CoursewareFolderLoader` 加载 `ResourceId`、资源类型、导出文件和解析后的本地路径。
- 页面生成 Prompt 只允许模型引用已登记的逻辑 `ResourceId`。
- `SlideMlPipelineContext` 提供 `MaterialResourceManager`。

仍需完成：

1. 建立课件级资源解析与注册边界，将允许使用的图片资源注册到每页渲染上下文。
2. 让 WPF SlideML 渲染器能够按逻辑 `ResourceId` 稳定取得对应素材，而不是只把 `Image.Source` 当作文件路径或候选文件名。
3. 保持路径约束，不向模型或输出文件暴露不必要的本地绝对路径。
4. 对缺失、越界、类型不支持和解码失败提供结构化诊断。
5. 增加资源解析、真实图片渲染和非法资源拒绝测试。

完成标准：使用课件中已登记的图片 `ResourceId` 生成 SlideML 后，本地 WPF 预览能够稳定显示真实图片，并且不存在任意文件读取能力。

### 2.2 全课件逐页执行与批处理

当前工作台支持用户逐页操作，也能汇总各页状态，但没有全课件串行执行编排。

仍需完成：

1. 增加串行生成全部页面或指定页面范围的命令。
2. 提供整体进度、当前页、成功、失败、取消和剩余数量状态。
3. 明确单页失败后的继续、停止和重试策略。
4. 支持取消批处理而不清除已完成页面结果。
5. 避免一次性初始化全部页面模型；继续使用页面级懒运行时。
6. 增加部分失败、取消、重试和顺序稳定性测试。

完成标准：用户可以可靠地串行处理整份课件，失败可定位、已完成结果可保留，应用保持可响应。

### 2.3 页面结果保存、恢复与增量更新

主题分析快照只保存原课件输入和主题分析结果，不保存页面生成后的 SlideML、聊天、预览和渲染状态。项目已有 `CoursewareOutputExportFormat.md`，但尚无对应保存服务和恢复入口。

仍需完成：

1. 按 `CoursewareOutputExportFormat.md` 实现版本化输出模型和 JSON 序列化元数据。
2. 保存主题结果、逐页 SlideML、预览图、生成记录、渲染日志、页面结果索引和总摘要。
3. 所有索引使用相对路径，避免写入敏感配置和不必要的绝对路径。
4. 支持逐页增量保存和安全覆盖，避免中途中断产生可误识别的完整结果。
5. 增加重新打开输出目录并恢复页面可编辑状态的入口。
6. 明确聊天历史、手工未保存编辑和 MCP 状态是否进入持久化契约。
7. 增加往返、版本不支持、文件损坏、部分结果和中断恢复测试。

完成标准：应用中断后可以从输出目录恢复已生成页面并继续编辑或重试，不需要重新执行已经成功保存的页面。

### 2.4 可执行 `CoursewareDesignSystem`（已完成）

当前活动结果仍是小型 `CoursewareTheme` 主题建议。Core 中存在部分设计系统 DTO 和 JSON 元数据，不代表设计系统能力已经完成。

仍需完成：

1. 定义并冻结版本化 `CoursewareDesignSystem` 执行契约。
2. 提供完整设计令牌，包括颜色、字体栈、字号、间距、网格、圆角、描边、阴影和密度规则。
3. 定义可复用组件、素材策略、无障碍规则和跨页一致性规则。
4. 根据课件实际内容形成动态页面类型，而不是只保留固定的 Cover、Section、Content、Ending 自然语言建议。
5. 为每个 `SlideId` 建立唯一页面类型或页面计划分配，并验证覆盖完整性。
6. 保存设计证据、假设、验证报告和明确的能力状态。
7. 使用适合大型对象的有状态、多工具草稿协议，避免把 DTO 存在或单次大对象提交误判为能力完成。
8. 让页面生成消费冻结后的强类型设计系统和页面计划，而不是仅消费主题摘要。

完成标准：设计系统可以生成、验证、冻结、持久化，并作为所有目标页面的真实执行输入。

### 2.5 SlideML 模板编译与真实内容压力测试（已完成）

当前页面类型只有自然语言建议，没有可编译模板目录和模板验证链路。

仍需完成：

1. 定义模板槽位、占位符、设计 Token 和运行时 SlideML 属性之间的转换规则。
2. 为动态页面类型提供可编译的 SlideML 模板或等价的强类型布局执行契约。
3. 使用真实 SlideML 解析器验证标签、属性、引用和根元素。
4. 使用短、中、长内容、特殊字符、图片缺失和多画布场景执行压力测试。
5. 验证安全区、溢出、裁切、最小字号和素材 fallback。
6. 将模板编译和压力测试结果纳入设计系统能力状态，失败时不得标记为 `Passed`。

完成标准：模板不只是模型返回的 XML 字符串，而是经过当前运行时真实编译、渲染和压力测试的可执行资产。

### 2.6 多模态视觉分析与结构化事实（已完成）

当前主题分析以完整 Markdown 和逻辑资源清单为主要输入，模型没有获得整课件视觉证据，也缺少结构化内容块和视觉事实账本。

仍需完成：

1. 扩展或新增可验证的内容块、页面几何、样式和素材事实协议。
2. 建立确定性的字体、字号、颜色、密度、布局簇和资源引用统计。
3. 选择全页面缩略图总览、高清代表页和关键素材样本，并记录选择原因。
4. 在模型支持时执行图像辅助的视觉分析；模型不支持时明确报告，不得伪造视觉结论。
5. 形成可追溯的视觉延续策略、全局结构计划和逐页页面计划。
6. 保留现有完整输入、路径隐私、不可静默截断和预算预检边界。

完成标准：设计决策能够追溯到结构化事实、页面、素材或视觉证据，而不是只由模型从长 Markdown 自行归纳。

### 2.7 页面级与课件级质量门

当前可以生成和渲染单页，但尚未形成产品化质量门。

仍需完成：

1. 页面级检查内容覆盖、SlideML 合法性、渲染错误、溢出、最小字号、对比度、安全区和资源可用性。
2. 课件级检查主题、字体、颜色、页码、固定素材和章节节奏一致性。
3. 将 Warning 分级为可接受、需要人工确认和阻断提交。
4. 支持失败页面定向返工，避免整课件无差别重做。
5. 保存质量报告，并让最终提交只消费通过质量门的结果。
6. 增加确定性规则测试、视觉回归策略和失败恢复测试。

完成标准：页面生成成功不再自动等同于产品结果通过；质量状态具有明确证据和可执行后续动作。

### 2.8 事务式提交与最终课件产物

当前没有全课件结果事务、回滚和最终 `.pptx` 组装闭环。

仍需完成：

1. 定义暂存、验证、提交和回滚流程，避免部分结果被误认为完整产物。
2. 支持页面替换、顺序调整、必要的拆分或合并结果进入最终课件结构。
3. 明确母版、字体、图片、音视频和其他资源的打包规则。
4. 生成最终 `.pptx`，并执行重新打开、页数、尺寸、资源和基本渲染验证。
5. 保存构建清单、来源映射和失败诊断。

完成标准：可以从已通过质量门的页面结果生成可重新打开和继续处理的完整 `.pptx`，失败时不破坏上一次有效产物。

## 三、建议实施顺序

1. 完成资源 ID 到本地 WPF 渲染闭环。
2. 完成页面结果保存、恢复和增量更新。
3. 完成全课件串行批处理与失败恢复。
4. 完成可执行 `CoursewareDesignSystem` 及页面覆盖契约。
5. 完成 SlideML 模板编译和真实内容压力测试。
6. 完成多模态视觉分析与结构化事实扩展。
7. 完成页面级和课件级质量门。
8. 完成事务式提交和最终 `.pptx` 产物。

资源、持久化和批处理优先，是因为它们能先把当前已经可用的真实单页工作台扩展为可恢复的全课件执行流程；设计系统、模板、视觉分析和质量门随后替换当前主题建议的过渡执行契约。
