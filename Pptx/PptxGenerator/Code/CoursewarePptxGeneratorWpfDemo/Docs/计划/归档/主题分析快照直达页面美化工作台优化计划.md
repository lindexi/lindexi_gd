# 🎯 主题分析快照直达页面美化工作台优化计划

> 状态：已完成并归档。当前实现已覆盖快照保存、恢复校验、损坏拒绝、跳过分析和直达等价页面美化工作台，并具有对应单元测试。

## Understanding
目标是在主题分析成功后自动生成可独立打开的本地快照，并让首页现有“打开课件文件夹”入口自动识别普通课件目录与主题分析快照目录。打开快照时跳过主题分析，复用正常流程的会话、工作台创建和激活代码，直接进入页面美化工作台，并通过单元测试证明两条路径的状态和页面生成上下文等价。

## Assumptions
- 生产环境以 `Directory.GetCurrentDirectory()` 作为快照输出根目录；测试通过注入临时输出根目录避免修改进程工作目录。
- 每次主题分析或重新分析成功后创建一个独立时间戳目录，例如 `CoursewareThemeAnalysis_yyyyMMdd_HHmmss_fff`；发生同毫秒冲突时追加序号，不覆盖历史结果。
- 快照必须自包含页面美化所需的全部输入：规范化后的 `Courseware.json`、页面 Markdown、已有页面截图、`Resources.json` 和已有资源文件。
- 快照根目录增加固定标记文件 `CoursewareThemeAnalysis.json`；该文件存在时优先按快照解析，解析失败不得静默退回普通课件分析。
- 快照保存失败属于主题分析流程失败：保留已加载课件和错误信息，但不发布主题结果、不创建工作台、不进入工作台。
- 普通课件分析成功后仍停留在主题分析结果页；只有打开已保存快照时自动进入页面美化工作台。
- 本期只持久化原课件输入与 `CoursewareThemeAnalysisResult`，不持久化页面生成后的 SlideML、聊天和渲染状态。

## Approach
在 WPF Demo 层新增版本化的主题分析快照协议。`CoursewareThemeAnalysis.json` 保存协议版本、创建时间、课件名称、页面数量、原始事实指纹和完整 `CoursewareThemeAnalysisResult`；同目录按现有课件导出协议重建一份规范化输入副本，使现有 [CoursewareFolderLoader.cs](Pptx/PptxGenerator/Code/CoursewarePptxGenerator.Core/Analysis/CoursewareFolderLoader.cs) 可以直接加载快照目录。保存过程先写入当前工作目录下的临时兄弟目录，完成文件复制、JSON 写入和回读校验后再重命名发布，取消或失败时清理临时目录，避免出现可被误识别的半成品。

新增统一的文件夹打开协调器，对用户选择的目录只做一次分类和加载：普通目录返回待分析的 `CoursewareInputPackage`，快照目录返回已加载的输入包和已验证主题结果。随后重构 [CoursewareWorkspaceViewModel.cs](Pptx/PptxGenerator/Code/CoursewarePptxGeneratorWpfDemo/ViewModels/CoursewareWorkspaceViewModel.cs)，让正常分析与快照恢复都调用同一个“提交主题结果并创建工作台”方法；正常路径先保存再提交，快照路径直接提交并调用现有工作台激活入口。快照加载时重新计算 `CoursewareAnalysisSourceSnapshotBuilder` 指纹、校验页面数/课件名/参考画布/主题 Schema，并复用 `CoursewareThemeValidator`，确保结果没有与复制后的输入脱节。

## Risks & Open Questions
- 时间戳快照会持续占用磁盘；按当前要求保留历史，本期不增加自动清理策略。
- 当前工作目录可能不可写；该情况将按已确认规则使主题分析进入失败状态，并且不会创建工作台。
- 复制范围只包含当前加载器和美化工作台真实消费的清单文件、Markdown、截图及已登记资源，不递归复制未知附加文件，避免把无关或敏感内容带入快照。
- 第一版只接受当前快照 Schema 和当前 `CoursewareTheme` Schema；版本不支持、JSON 损坏或输入指纹不一致均明确失败，不自动迁移或重新分析。
- 缺失截图或资源继续沿用现有非阻塞 Warning 语义；复制后重新加载应产生与正常路径一致的可用状态和警告。
