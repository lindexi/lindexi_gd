## 一、目标与完成定义

本期把当前“小型 `CoursewareTheme` 建议 + 完整 Markdown 输入”升级为可执行、可验证、可追溯的设计系统分析闭环：

1. 主题分析输出版本化、可冻结的 `CoursewareDesignSystem`，包含完整设计令牌、组件、素材策略、动态页面类型和全页面覆盖映射。
2. 每个动态页面类型具有可编译的 SlideML 模板或等价强类型布局契约；本期要求模板实际通过当前 SlideML 解析器和 WPF 渲染流水线验证。
3. 从完整 Markdown 确定性提取页面几何、字体、字号、颜色、密度、布局簇和资源引用事实。
4. 在模型明确支持图片附件时，选择受控截图样本并执行图像辅助分析；不支持时明确报告 `NotSupported`，不得伪造视觉结论。
5. 页面生成消费冻结的强类型设计系统、当前页页面类型和模板契约，不再仅依赖 `GenerationPromptSummary`。

完成后，三项能力必须分别具有独立状态和验证证据：

- `DesignSystem = Passed`：设计系统已冻结，引用、证据边界和 SlideId 覆盖完整。
- `TemplateValidation = Passed`：模板已用真实解析器和 WPF 渲染管线完成静态编译及压力测试，没有阻断错误。
- `VisualAnalysis = Passed | NotSupported | NotRequested | Failed`：只有实际发送并分析图像时才允许为 `Passed`。

## 四、结构化事实协议

### 4.1 页面事实

每页事实至少包含：

- `SlideId`、页码、画布尺寸和宽高比；
- 元素总数及文本、图片、表格、形状、组合、音视频分类数量；
- 可解析元素矩形、归一化矩形、内容包围盒和安全区留白候选；
- 文本字符数、字体分布、字号分布和常见字号组合；
- Markdown 中可确定的背景颜色、背景资源和普通资源引用；
- 越界、重叠和高密度风险候选；
- 确定性布局签名和启发式布局簇；
- 无法解析字段和诊断。

事实来源必须标记为：

- `SlideMarkdownFact`：Markdown 直接事实；
- `DeterministicStatistic`：本地确定性聚合；
- `HeuristicCandidate`：阈值或聚类候选；
- `VisualObservation`：实际图像分析结果；
- `DesignInference`：模型设计推断。

### 4.2 全局事实

全局事实至少包含：

- 页面尺寸分布和主画布；
- 元素类型、文本长度、字体、字号和背景分布；
- ResourceId 引用次数、引用页和未使用资源；
- 密度分布、布局簇、代表页和离群页；
- 可解析页数、诊断数量和统计版本；
- 全部事实来源的输入指纹。

### 4.3 视觉样本清单

视觉样本采用最小必要原则，默认最多选择：

1. 首个有截图页面，作为课件入口样本；
2. 每个主要布局簇的代表页；
3. 一个高密度离群页；
4. 一个图片资源引用较多的页面；
5. 首尾页面中尚未覆盖的页面。

每个样本记录 `SlideId`、页码、样本角色、选择原因和问题描述，不把本地路径写入模型 Prompt 或结果。

## 五、可执行设计系统契约

### 5.1 冻结条件

设计系统冻结前必须满足：

- Schema 版本正确；
- `DesignSystemId`、设计意图和主画布有效；
- 颜色、字体、字号、间距、网格、圆角、描边、阴影和密度规则完整；
- 必需语义颜色和文本层级存在；
- 组件只引用已定义令牌；
- 素材策略只引用输入资源目录中的 `ResourceId`；
- 动态页面类型具有证据、密度范围、槽位、组件、硬约束、允许调整和溢出策略；
- 每个输入 `SlideId` 恰好分配一个主要页面类型；
- 所有页面类型引用均可解析；
- 证据、推断、未知项和视觉观察没有混淆；
- 未执行图像分析时不存在 `VisualObservation`；
- 不包含本地绝对路径。

### 5.2 有状态多工具草稿

工具协议采用以下分区：

1. `begin_courseware_design_system`；
2. `set_canvas_and_grid`；
3. `set_typography_tokens`；
4. `set_color_tokens`；
5. `set_spacing_and_effect_tokens`；
6. `upsert_design_component`；
7. `set_asset_policy`；
8. `upsert_page_type`；
9. `assign_slides_to_page_type`；
10. `upsert_page_template`；
11. `set_accessibility_and_consistency_rules`；
12. `set_design_evidence_and_assumptions`；
13. `complete_courseware_design_system`。

所有写操作携带 `DraftId + Revision`：

- Revision 不匹配时拒绝写入；
- `set_*` 原子替换一个分区；
- `upsert_*` 按稳定 ID 幂等更新；
- 局部失败不污染已通过分区；
- 完成后草稿只读；
- 完整验证失败时返回字段路径级错误和允许的下一步工具。

## 六、SlideML 模板编译与压力测试

### 6.1 模板语法边界

模板使用当前 SlideML 标签和属性，不扩展 SlideML 解析器语法。业务令牌和槽位由编译器在进入解析器前解析：

- 设计令牌引用解析为最终数值或颜色；
- 文本槽位执行 XML 转义；
- 图片槽位只能绑定逻辑 `ResourceId`；
- 可选槽位为空时移除对应模板节点或使用声明的 fallback；
- 编译结果必须具有唯一 `<Page>` 根节点；
- 未知标签、未知属性、未知令牌、未知槽位和非法资源引用为阻断错误。

### 6.2 压力样本

每个页面类型至少生成以下样本：

- 最小样本：只填充必需槽位；
- 短内容样本；
- 中等内容样本；
- 长内容样本；
- 特殊字符样本：XML 保留字符、中英文混排、换行和长英文单词；
- 图片缺失样本：验证 fallback 或明确失败；
- 每个支持画布的样本。

优先从真实 Markdown 提取短、中、长文本；无法获得时使用明确标记的确定性合成边界样本，不伪装为原课件事实。

### 6.3 质量门

`TemplateValidation = Passed` 必须同时满足：

- 全部模板的页面类型、槽位、令牌、组件和 fallback 引用有效；
- 真实 `SlideMlParser` 编译无错误；
- 真实 WPF 布局和渲染无错误；
- 没有元素越出画布；
- 必需槽位未缺失；
- 字号不低于设计系统最小字号；
- 安全区规则满足；
- 长内容失败时按声明的溢出策略给出明确诊断，不得静默裁切；
- 图片缺失不会触发任意文件读取。

## 七、多模态视觉分析

### 7.1 能力门控

运行前检查模型能力：

- `Capabilities.Attachment == true`；
- `Capabilities.Input.Image == true`；
- 存在可用截图；
- 视觉样本清单非空。

状态规则：

- 用户或调用方未请求视觉分析：`NotRequested`；
- 已请求但模型不支持附件或图片输入：`NotSupported`；
- 已发送图片并获得通过验证的视觉观察：`Passed`；
- 已发送图片但调用、解析或验证失败：`Failed`。

### 7.2 视觉观察边界

视觉观察只允许描述实际附件可见内容，并必须记录：

- 观察 ID；
- 对应 `SlideId` 和页码；
- 观察类别；
- 观察内容；
- 置信度；
- 支持该观察的视觉样本角色。

视觉观察不能覆盖 Markdown 明确事实；发生冲突时记录冲突诊断。没有发送图片时，任何 `VisualObservation` 都是阻断错误。

## 八、下游页面生成消费

页面生成 Prompt 的 `designContext` 改为设计系统执行视图，至少包含：

- `DesignSystemId` 和 Schema 版本；
- 当前画布匹配的画布、网格和安全区；
- 可用颜色、字体、字号、间距和效果令牌；
- 当前页分配的 `PageTypeId`；
- 页面类型槽位、组件、硬约束、允许调整和溢出策略；
- 对应模板及模板验证状态；
- 当前页允许使用的素材策略；
- 无障碍和跨页一致性规则；
- 与当前页相关的事实、视觉观察和未知项。

过渡期 UI 可以继续显示主题摘要，但页面生成执行约束以冻结设计系统为准。
