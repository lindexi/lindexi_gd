# CoursewarePptxGeneratorWpfDemo 全课件美化双页界面原型实施方案

```text
状态：界面原型实施阶段。

本文仅用于实现和验证“双页外壳 + 全课件分析页”的界面与交互。
所有课件、缩略图、分析过程、主题结果和状态变化均使用演示数据，不接入真实课件加载、AI 主题分析、聊天会话或单页生成流程。

完整产品方案仍以《全课件美化双页交互改造方案.md》为准；实现当前界面原型时只需阅读本文。
```

---

## 一、本轮目标

本轮先做可运行的 WPF 界面原型，用于验证以下问题：

- 应用从现有固定三栏工作台改为双页后，整体导航是否自然。
- 第一页能否清楚表达“已查看整份课件并形成统一主题”。
- 缩略图概览、主题方案、分析过程和主要操作之间的视觉层级是否合理。
- Welcome、加载、分析、完成和失败等状态的切换是否容易理解。
- 从第一页进入现有单页工作台，再返回第一页时，操作是否符合预期。
- 界面在 `1720 × 1000` 和 `1360 × 820` 窗口尺寸下是否可用。

本轮以快速验证界面为第一优先级。原型确认前，不为尚未稳定的交互提前建设真实业务链路。

---

## 二、实施范围

### 2.1 本轮实现

本轮只实现：

1. 将主窗口调整为双页内容宿主。
2. 新增全课件分析页。
3. 使用演示数据呈现第一页的主要状态。
4. 使用演示命令切换状态，便于检查不同画面。
5. 将现有三栏工作台整体作为第二页使用。
6. 实现第一页与第二页之间的演示导航。
7. 为第一页准备设计时数据，支持在 XAML 设计器中直接预览主要画面。
8. 补充必要的样式、键盘焦点和可访问名称。

### 2.2 第二页范围

第二页不重新设计现有工作台，只做以下调整：

- 将现有左侧页面列表、中间工作区和右侧 Copilot 的三栏组合放入独立页面容器。
- 由双页外壳提供“返回主题概览”导航，不在工作台内部增加新的主题功能。
- 保持现有三栏尺寸、控件和视觉结构不变。
- 只保留选择、焦点、滚动、展开和折叠等纯界面交互。
- 原型模式下，任何可能触发课件加载、聊天、AI、生成、渲染、保存或持久化的命令必须禁用或替换为演示命令。

### 2.3 本轮明确不实现

本轮不涉及：

- 真实文件夹选择和课件目录读取。
- `Courseware.json` 解析和输入校验。
- 真实页面截图、Markdown 或资源加载。
- AI 模型调用、流式输出和结构化主题生成。
- `CopilotChatManager`、`SlideChatManager` 或任何会话工厂调整。
- 页面会话延迟创建、后台预创建或性能测量。
- 真实取消、重试、错误恢复和重新打开课件流程。
- 主题结果保存、版本管理或注入单页提示词。
- SlideML 生成、渲染、评估和输出流程改造。
- 未保存结果确认和课件会话生命周期管理。
- 服务层、输入模型、领域模型或持久化设计。

界面中的“打开课件”“取消分析”“重新分析”“打开其他课件”等操作只驱动演示状态，不调用真实业务服务。

## 三、原型页面结构

### 3.1 页面枚举

原型只需要两个页面状态：

```text
CoursewareAnalysis
SlideWorkspace
```

含义：

- `CoursewareAnalysis`：全课件分析页，是应用启动后的默认页面。
- `SlideWorkspace`：现有三栏单页工作台页面。

页面切换只改变当前显示内容，不重新创建主窗口。

### 3.2 第一页演示状态

第一页使用强类型演示状态：

```text
Welcome
LoadingCourseware
AnalyzingTheme
AnalysisReady
AnalysisFailed
Canceled
```

这些状态只服务于界面展示，不代表真实业务状态机已经实现。

### 3.3 原型导航

```text
应用启动
  -> CoursewareAnalysis / Welcome
  -> 点击“打开课件文件夹”
  -> LoadingCourseware
  -> 演示延迟或开发状态切换
  -> AnalyzingTheme
  -> 演示延迟或开发状态切换
  -> AnalysisReady
  -> 点击“进入单页美化工作台”
  -> SlideWorkspace
  -> 点击“返回主题概览”
  -> CoursewareAnalysis / AnalysisReady
```

为了便于反复检查界面，可以提供仅在 Debug 构建或原型模式下可见的状态切换入口。该入口不属于最终产品界面，不能与正式主操作混在一起。

## 四、演示数据约定

### 4.1 示例课件

原型统一使用以下演示课件：

- 课件名称：`初中数学八年级上册`
- 页面数量：`24`
- 分析状态：分析完成
- 输入状态：完整，无缺失截图和资源警告
- 主题关键词：`现代`、`清晰`、`理性`、`轻量科技`
- 视觉目标：适合数学概念讲解、公式推导和例题分析

各区域必须使用同一份课件数据，不能出现不同页数或互相矛盾的分析结论。

### 4.2 缩略图数据

准备 24 个演示缩略图项：

- 每项具有稳定索引。
- 使用内置演示图片、现有可复用图片或统一占位图。
- 图片区域保持幻灯片宽高比。
- 分析完成主画面不显示页码、页面名称和页面 ID。
- Welcome 状态不创建或不展示缩略图区域。

不得在演示数据中引用本地绝对路径。

### 4.3 分析过程数据

分析完成主画面展示一组紧凑的只读阶段摘要：

1. 已读取课件基本信息。
2. 已识别封面、章节页和内容页结构。
3. 已归纳数学课件的内容层级与讲解节奏。
4. 已分析页面比例、图文分布和视觉重点。
5. 已形成配色、字体和版式建议。
6. 已完成主题结构校验。

分析中状态可以让最后一项保持进行中，但不展示虚假的 Token、评分、百分比或模型内部推理。

### 4.4 主题结果数据

#### 主题摘要

建议示例文案：

> 以清晰推导和理性表达为核心，通过稳定的蓝色体系、明确的标题层级和规整的图文对齐，建立适合初中数学教学的现代课堂视觉风格。

#### 色彩系统

| 用途 | 示例颜色 |
| --- | --- |
| 主色 | `#2563EB` |
| 强调色 | `#F59E0B` |
| 页面背景 | `#F8FAFC` |
| 主文字 | `#0F172A` |
| 次文字 | `#475569` |

颜色必须同时显示用途和十六进制值。

#### 字体层级

- 标题字体：微软雅黑或可用的等效无衬线字体。
- 正文字体：微软雅黑或可用的等效无衬线字体。
- 一级标题：突出章节或核心结论。
- 二级标题：组织页面内知识层级。
- 正文：保证投影环境下的阅读清晰度。
- 辅助文字：仅用于注释、来源或次要说明。

原型重点验证信息层级，不需要检测本机是否安装推荐字体。

#### 版式原则

- 标题区保持稳定高度和对齐起点。
- 每页只保留一个主要视觉焦点。
- 公式、图形和说明文字遵守统一对齐线。
- 例题、步骤和结论使用一致的视觉节奏。
- 章节页与普通内容页形成明确变化。

#### 补充信息

安全区和页面类型建议作为次级内容展示：

- 左右安全边距：`0.6 in`
- 上下安全边距：`0.4 in`
- 封面页：突出课题名称和学科识别。
- 章节页：使用简洁的大标题与章节编号。
- 内容页：优先保证公式、图形和步骤的清晰关系。
- 结尾页：收束知识点并保留复习提示。

## 五、双页外壳

### 5.1 主窗口职责

主窗口只承担：

- 保持现有窗口尺寸、位置和标题栏。
- 根据当前页面状态显示第一页或第二页。
- 提供页面切换所需的根级 DataContext。

主窗口不继续直接定义固定三栏布局。三栏布局属于第二页内部结构。

### 5.2 页面切换

首轮可以直接切换内容，不要求先实现复杂转场。若增加动画，应满足：

- 不阻塞导航。
- 不使用快速闪烁。
- 页面隐藏后不保留键盘焦点。
- 进入新页面后将焦点移动到页面标题或主要操作。
- 动画关闭时布局和交互仍完整可用。

### 5.3 状态保留

从第二页返回第一页时，应继续展示 `AnalysisReady` 的演示画面。再次进入第二页时，不应因为导航而重建现有工作台控件或清空其当前 UI 状态。

本轮只验证视图实例和界面状态保留，不设计真实课件会话对象。

## 六、第一页布局

### 6.1 视觉层级

分析完成主画面的视觉优先级依次为：

1. 主题方案。
2. 分析完成状态与“进入单页美化工作台”。
3. 分析过程。
4. 顶部缩略图概览。

缩略图虽然位于内容顶部，但不能成为页面主体。

### 6.2 页面头部

页面头部保持简洁，展示：

- 产品名称。
- 当前课件名称。
- `24 页`。
- 分析完成状态。
- “打开其他课件”次要入口。

头部不显示：

- 本地绝对路径。
- 模型选择。
- MCP 设置。
- Token 统计。
- 页面级生成控制。

### 6.3 顶部缩略图概览

缩略图区域位于页面主要内容顶部：

- 使用 `WrapPanel` 从左到右排列。
- 当前行空间不足时自动换行。
- 不提供横向滚动。
- 页面整体或内容区域只允许纵向滚动。
- 每项只显示原始页面缩略图。
- 默认主画面不显示页码、名称、ID 和逐页操作。
- 不承担选择、排序、编辑或批量处理。

缩略图尺寸通过样式或资源统一控制，不在单个项中散落固定尺寸。

### 6.4 主题方案主体

主题方案是页面最大的视觉区域，内部应形成主次关系，而不是把所有字段做成等权卡片。

建议层级：

1. 主题摘要和风格关键词。
2. 核心配色。
3. 字体层级。
4. 关键版式原则。
5. 折叠或弱化展示的安全区与页面类型建议。

布局可以根据实际视觉效果选择两列或不对称栅格，但必须保证用户先读到整体方向，再读具体规范。

### 6.5 分析过程

分析过程使用紧凑的只读阶段列表或时间线：

- 不显示输入框。
- 不显示用户消息气泡。
- 不显示附件和设置。
- 不展示工具参数和内部推理。
- 每个阶段通过图标、文字和状态共同表达，不能只依赖颜色。
- 分析完成时整体保持辅助信息权重。

### 6.6 操作区

分析完成状态提供：

- `进入单页美化工作台`：主操作。
- `重新分析`：次操作，仅切换演示状态。
- `打开其他课件`：次操作，仅返回 Welcome 或 Loading 演示状态。

主要操作必须清晰可发现，但不要求固定在窗口底部。具体位置以整体视觉层级为准。

## 七、第一页各演示状态

### 7.1 Welcome

目标：应用启动时只突出打开课件入口，不出现空白三栏编辑器。

展示：

- 产品名称和一句用途说明。
- `打开课件文件夹` 主按钮。
- 适量辅助视觉元素。

不展示：

- 缩略图。
- 主题结果。
- 分析时间线。
- 聊天输入。
- 模型、附件、MCP 或添加空页面入口。

点击主按钮后直接进入演示加载状态，不打开真实目录选择器。

### 7.2 LoadingCourseware

目标：验证从空状态进入有内容页面时是否平稳。

展示：

- 演示课件名称。
- “正在读取课件”等加载说明。
- 明确的进行中状态。
- 可选的内容骨架或占位区域。

不展示容易被误认为已经有效的完整主题结果。

### 7.3 AnalyzingTheme

目标：验证缩略图、分析过程和逐步形成的主题区域能否同时存在。

展示：

- 24 页缩略图概览。
- 只读分析过程，最后一个阶段处于进行中。
- 主题区域骨架、部分内容或渐进出现状态。
- `取消分析` 次要操作，仅切换到 `Canceled`。

不得为了动画人为延长原型状态切换。

### 7.4 AnalysisReady

这是本轮最重要、默认用于高保真检查的状态。

展示完整的：

- 课件上下文。
- 顶部缩略图概览。
- 主题摘要和关键词。
- 色彩、字体和版式规范。
- 弱化的安全区与页面类型建议。
- 已完成的分析过程。
- 进入工作台主操作。

用户应能在一个屏幕内快速回答：

- 这是什么课件？
- 推荐采用什么视觉风格？
- 核心颜色是什么？
- 标题和正文如何分级？
- 页面布局遵循什么原则？
- 分析是否已经完成？

### 7.5 AnalysisFailed

目标：检查失败信息是否清楚，但不实现真实错误处理。

展示：

- 简洁错误标题。
- 演示错误原因。
- `重试主题分析` 主操作。
- `打开其他课件` 次操作。
- 可展开或复制的演示技术详情。

如果该状态模拟“课件已加载但分析失败”，可以保留缩略图和课件名称；不显示可进入工作台的有效主题。

### 7.6 Canceled

展示：

- 已加载的课件名称和缩略图。
- 已产生的分析阶段摘要。
- 分析已取消的明确状态。
- `重新分析` 和 `打开其他课件`。

不允许进入第二页。

## 八、第二页复用方式

### 8.1 页面容器

将当前三栏整体放入 `SlideWorkspaceView` 或等效页面级视图：

```text
双页外壳返回导航
  +
左侧页面列表 | 中间工作区 | 右侧 Copilot
```

迁移目标是让主窗口能够承载双页，不借此调整现有工作台内部职责。

### 8.2 返回导航

“返回主题概览”属于双页外壳导航，不属于工作台新增功能。除该导航外，第二页不增加课件名称、主题状态、主题摘要入口或演示浮层。

### 8.3 状态保留检查

在第二页切换选择、焦点、滚动或展开折叠等纯 UI 状态后返回第一页，再次进入第二页时，该状态应尽量保留。原型阶段重点检查页面容器是否被重复创建，不扩展到真实页面聊天和生成结果生命周期。

现有工作台中可能调用真实服务的操作在原型模式下不可执行，不以“保持状态”为由创建聊天会话或保留生成结果。

## 九、建议 View 与 ViewModel 拆分

以下拆分仅服务于界面原型，可根据现有项目结构适当合并，但不要把所有状态继续堆入 `MainWindow.xaml.cs`。

### 9.1 View

```text
Views/
  CoursewareAnalysisView.xaml
  CoursewareThumbnailOverview.xaml
  CoursewareThemeSummaryView.xaml
  CoursewareAnalysisTimelineView.xaml
  SlideWorkspaceView.xaml
```

职责：

- `CoursewareAnalysisView`：第一页整体布局和状态画面。
- `CoursewareThumbnailOverview`：顶部自动换行缩略图。
- `CoursewareThemeSummaryView`：主题结果主体。
- `CoursewareAnalysisTimelineView`：只读阶段摘要。
- `SlideWorkspaceView`：现有三栏工作台页面容器。

如果拆分后只产生很薄且无法复用的控件，可以先保留在 `CoursewareAnalysisView` 内，避免为了目录形式过度拆分。

### 9.2 ViewModel

建议最小模型：

```text
MainWindowViewModel
  - CurrentPage
  - Analysis
  - NavigateToWorkspaceCommand
  - NavigateToAnalysisCommand

CoursewareAnalysisDesignViewModel
  - Stage
  - CoursewareName
  - SlideCount
  - Thumbnails
  - TimelineItems
  - ThemeSummary
  - OpenDemoCommand
  - CancelDemoCommand
  - RetryDemoCommand
```

命名可以根据项目现有约定调整。演示 ViewModel 不得伪装成已经完成的真实课件分析服务。

### 9.3 演示状态切换

演示命令可以使用短暂、可取消的 UI 延迟模拟状态变化，也可以立即切换。若使用异步延迟：

- 方法名以 `Async` 结尾。
- 使用 `CancellationToken`。
- 不使用 fire-and-forget。
- 页面关闭或重新开始演示时取消旧延迟。
- 延迟仅用于观察状态，不模拟真实处理耗时。

## 十、样式与资源

界面美化应放入独立资源文件，避免在单个 XAML 中重复散落颜色、圆角、间距和字号。

建议至少统一：

- 页面背景和内容表面色。
- 主按钮、次按钮和文字按钮。
- 状态徽标。
- 主题关键词标签。
- 色彩样本。
- 缩略图卡片。
- 分析阶段项。
- 标题、正文和辅助文字层级。
- 内容区间距和最大宽度。

所有涉及的主要控件应有明确样式。颜色资源使用语义化名称，不以具体颜色名称代替用途。

## 十一、设计时支持

第一页必须提供设计时数据，至少可在设计器中预览：

- `Welcome`。
- `AnalyzingTheme`。
- `AnalysisReady`。
- `AnalysisFailed`。

其中 `AnalysisReady` 是默认设计画面，包含 24 个缩略图、完整主题结果和分析阶段。

设计时数据要求：

- 不读取磁盘文件。
- 不访问网络。
- 不创建聊天管理器或渲染器。
- 不依赖本地绝对路径。
- 与运行时演示数据使用相同的展示模型，避免两套 UI 结构。

---

## 十二、无障碍与键盘交互

原型阶段至少满足：

- 所有主要按钮支持 Tab 导航和 Enter/Space 激活。
- 页面切换后焦点进入新页面标题或主要操作。
- 状态由图标和文字共同表达，不只依赖颜色。
- 色块旁显示用途和十六进制值。
- 缩略图提供可访问名称，例如“课件缩略图，第 3 页”。
- 分析阶段项提供阶段名称和完成状态。
- 错误详情文字可选择和复制。
- 动画不快速闪烁，关闭动画后仍可完整操作。

## 十四、实施顺序

### 阶段一：双页外壳

1. 将现有三栏组合迁入第二页容器。
2. 将主窗口改为页面宿主。
3. 增加页面枚举和演示导航命令。
4. 确保启动时显示第一页 Welcome。
5. 验证往返导航不会重复创建第二页。

### 阶段二：第一页完成态

1. 先实现 `AnalysisReady` 主画面。
2. 加入顶部 24 页缩略图概览。
3. 实现主题摘要、关键词、色彩、字体和版式区域。
4. 实现只读分析阶段区域。
5. 加入进入工作台主操作。
6. 在目标窗口尺寸下调整视觉层级。

先完成最能代表产品价值的画面，再补其他状态，避免在空状态和加载动画上过早投入。

### 阶段三：其他演示状态

1. 实现 Welcome。
2. 实现 LoadingCourseware。
3. 实现 AnalyzingTheme。
4. 实现 AnalysisFailed。
5. 实现 Canceled。
6. 增加 Debug 或原型模式状态切换入口。

### 阶段四：体验收尾

1. 补充设计时数据。
2. 统一资源和控件样式。
3. 调整键盘焦点与可访问名称。
4. 检查动画降级。
5. 检查两个目标窗口尺寸。
6. 复核第二页纯 UI 交互可用且真实业务命令不可执行。

## 十五、验收标准

### 15.1 双页结构

- 应用启动时不显示现有三栏工作台。
- Welcome 只有一个清晰的主要操作。
- 可以从分析完成态进入现有工作台。
- 可以从工作台返回主题概览。
- 往返导航不重新创建主窗口。
- 第二页现有三栏布局及选择、焦点、滚动、展开和折叠等纯 UI 交互保持可用。
- 第二页不会因为界面原型操作而调用真实课件、聊天、生成、渲染或保存服务。

### 15.2 分析完成主画面

- 顶部可看到 24 页缩略图，空间不足时自动换行。
- 页面没有横向滚动条。
- 缩略图只表达“已查看整份课件”，不抢占主题主体的视觉注意力。
- 用户能快速理解主题摘要、关键词、核心颜色、字体层级和版式原则。
- 安全区与页面类型建议明显弱于核心主题信息。
- 分析过程紧凑、只读且不出现聊天输入。
- “进入单页美化工作台”清晰可发现。

### 15.3 演示状态

- Welcome、加载、分析、完成、失败和取消状态均可独立预览。
- 状态之间的数据保持一致，不出现课件名称和页数冲突。
- 失败和取消状态不能进入第二页。
- 重新分析和打开其他课件只切换演示状态，不访问真实业务服务。
- Debug 状态切换入口不会被误认为正式产品功能。

### 15.4 尺寸与可用性

- `1720 × 1000` 下信息层级完整且留白合理。
- `1360 × 820` 下主要操作、主题摘要和核心规范仍易于发现。
- 内容超出高度时只使用纵向滚动。
- 键盘可以到达所有主要操作。
- 页面切换后的焦点位置正确。
- 状态和颜色信息满足基本无障碍要求。

## 十六、原型评审重点

完成界面后，优先评审以下问题：

1. 用户是否一眼理解第一页是在分析整份课件，而不是编辑单页？
2. 缩略图放在顶部是否既能表达分析覆盖，又不会喧宾夺主？
3. 主题结果是否像一套完整方案，而不是若干无关联指标卡片？
4. 用户是否无需阅读分析过程就能理解推荐方向？
5. 分析过程是否足以建立信任，又不会像聊天记录一样占据主体？
6. 进入工作台的操作是否在正确时机出现且足够清楚？
7. 从工作台返回主题概览是否自然？
8. 在较小窗口下，信息是否仍有清晰主次？
9. 哪些区域需要合并、弱化、折叠或重新排序？
10. 当前交互是否值得进入真实流程对接阶段？

评审结论确认前，不开始课件加载、AI 分析和页面生成上下文的真实接入。
