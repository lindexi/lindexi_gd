### Step 1: 新建 AttachedImageItem 数据模型
- 创建 `Code\PptxGenerator\Core\Ui\AttachedImageItem.cs`
- 属性：`string FilePath`（用于缩略图显示文件路径/名称）、`byte[] ImageData`（二进制图片数据）、`Avalonia.Media.Imaging.Bitmap? Thumbnail`（用于 UI 缩略图绑定）

### Step 2: 新增 SlideChatManager.SendMessageAsync 重载
- 在 `SlideChatManager.cs` 新增方法：
  ```csharp

      string userMessage, bool isFirstMessage, bool attachPreview,
      IReadOnlyList<DataContent> attachedImages,
      CancellationToken cancellationToken = default)
  ```
- 在 `contents` 列表中，先添加 `TextContent`，再添加外部传入的图片 `DataContent`，最后按需附加预览图
- 保留原方法作为兼容入口，原方法内部调用新重载（传入空列表）

### Step 3: 扩展 MainWindowViewModel
- 新增 `public ObservableCollection<AttachedImageItem> AttachedImages { get; }`
- 新增 `ICommand AttachImageCommand` → 调用 `OpenFileDialog` 选择图片文件，读取为 `byte[]`，创建 `AttachedImageItem` 并加入集合

- 修改 `RunSendMessageAsync()`：将 `AttachedImages` 中的图片转为 `DataContent` 列表，传入 `SlideChatManager.SendMessageAsync()` 新重载；发送成功后清空 `AttachedImages`

## 风险与注意事项

- **CharUserControl 的 x:DataType**：当前 `MainWindow.axaml` 中的 `<vm:CharUserControl>` 直接放在 Grid 里，DataContext 会从父级 Window 继承（即 `MainWindowViewModel`），绑定可工作。但如果未来要独立编译绑定，需要在 CharUserControl 内部声明 `xmlns:vm="using:PptxGenerator"` 和 `x:DataType="vm:MainWindowViewModel"`。
- **大图片性能**：目前不压缩，如果用户选择数 MB 级别的大图，内存和发送速度可能成为问题。后续可加限制。
- **AttachPreview 与 AttachedImages 共存**：`RunSendMessageAsync` 中需处理好两者同时存在的逻辑（先加用户图片，再根据 `_attachPreview` 追加预览图）。
- **API 兼容性**：`SlideChatManager.SendMessageAsync` 现有签名保持 `[Obsolete]` 标记或直接委托给新重载。
