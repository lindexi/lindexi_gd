## 详细步骤

### Step 1: 修改 SlideChatManager.SendMessageAsync 签名
- **文件**：`Core\SlideMl\SlideChatManager.cs`
- 在现有 `SendMessageAsync(string userMessage, bool isFirstMessage, bool attachPreview, CancellationToken cancellationToken = default)` 上增加可选参数：
  ```csharp
  IReadOnlyList<string>? attachedImageFiles = null
  ```
- 方法体 L94 之前：若 `attachedImageFiles` 不为空，遍历调用 `DataContent.LoadFromAsync(filePath)` 并将结果加入 `contents` 列表
- 调用方无需修改（可选参数默认 `null`），向后兼容
- 同时 `SendSlideRequestAsync`（L69）无需变更

### Step 2: 扩展 MainWindowViewModel
- **文件**：`Core\Ui\MainWindowViewModel.cs`
- 新增 `public ObservableCollection<FileInfo> AttachedImageFiles { get; }`（集合变更时需要通知 `_sendMessageCommand.RaiseCanExecuteChanged` 不是必须的——发送按钮仅依赖 InputText 非空和 IsBusy；但附件栏显隐可绑 `AttachedImageFiles.Count > 0`）
- 新增 `ICommand AttachImageCommand`：
  - 使用 `OpenFileDialog`（Avalonia 的 `StorageProvider.OpenFilePickerAsync`）选择图片文件
  - 将选中的 `FileInfo` 加入 `AttachedImageFiles`
  - 由于 `OpenFileDialog` 需要 `TopLevel` 引用，命令执行时可从 CharUserControl 传入 `Visual` 或使用其他方式获取 `TopLevel`
- 新增 `ICommand RemoveImageCommand`：接受 `FileInfo` 参数，从 `AttachedImageFiles` 中移除
- 修改 `RunSendMessageAsync()`：
  - L122：在调用 `SlideChatManager.SendMessageAsync(...)` 时，传入 `AttachedImageFiles.Select(f => f.FullName).ToList()`
  - 发送成功后清空 `AttachedImageFiles`
- 新增 `private readonly DelegateCommand _attachImageCommand` 和 `_removeImageCommand`（或直接在属性 getter 中返回）

### Step 3: 修改 CharUserControl.axaml XAML 布局
- **文件**：`Core\Ui\Views\CharUserControl.axaml`
- 将最外层 `Grid` 从 `RowDefinitions="Auto,*,Auto"` 改为 `RowDefinitions="Auto,*,Auto,Auto"`
  - 原 Row 2（底部输入栏）→ 新 Row 3
  - 新 Row 2 用于图片附件横排栏
- 在 Row 2 新增附件栏：
  ```xml
  <Border Grid.Row="2" ... IsVisible="{Binding AttachedImageFiles.Count, Converter=...}">
    <DockPanel>
      <!-- 横排图片缩略图列表，填充左侧 -->
      <ItemsControl ItemsSource="{Binding AttachedImageFiles}" DockPanel.Dock="Left">
        <ItemsControl.ItemsPanel>
          <ItemsPanelTemplate><WrapPanel Orientation="Horizontal" /></ItemsPanelTemplate>
        </ItemsControl.ItemsPanel>
        <ItemsControl.ItemTemplate>
          <DataTemplate>
            <Border CornerRadius="8" ... Width="72" Height="72">
              <Grid>
                <Image Source="{Binding FullName}" ... />   <!-- 缩略图 -->
                <Button Content="×" ... />                 <!-- 左上角关闭按钮 -->
              </Grid>
            </Border>
          </DataTemplate>
        </ItemsControl.ItemTemplate>
      </ItemsControl>
      <!-- 右侧附加图片按钮 -->
      <Button Content="+ 图片" Command="{Binding AttachImageCommand}"
              CommandParameter="{Binding $parent[UserControl]}" ...
              DockPanel.Dock="Right" />
    </DockPanel>
  </Border>
  ```
- 底部输入栏 Grid.Row 从 `"2"` 改为 `"3"`，内容不变（发送、调试按钮保留）
- 附件栏显隐：当 `AttachedImageFiles.Count > 0` 时显示（或有默认显示逻辑）

### Step 4: 完善 CharUserControl.axaml.cs
- **文件**：`Core\Ui\Views\CharUserControl.axaml.cs`
- 保留 `DebugButton_OnClick`（调试功能无需改动）
- 对于 `AttachImageCommand` 需要 `TopLevel`（`OpenFilePickerAsync` 需要窗口引用），有两种方案：
  - **方案 A**：在 `CharUserControl.axaml.cs` 中通过事件处理（如 Button.Click）获取 `TopLevel.GetTopLevel(this)` 然后调用 `StorageProvider`，不通过 ICommand
  - **方案 B**：在 ViewModel 的 `AttachImageCommand` 中通过 `CommandParameter` 传入 `UserControl`，在命令内部获取 `TopLevel`
  - 建议方案 A（更简单直接），需要修改 `CharUserControl.axaml.cs` 新增 `AttachImageButton_OnClick` 事件处理
- 图片关闭按钮同样建议用事件处理或 `Command="{Binding $parent[UserControl].DataContext.RemoveImageCommand}"`

### Step 5: 主线验证
- `dotnet build` 编译检查
- `dotnet run` 运行验证：
  - 点击 `+图片` → 文件选择对话框
  - 选中图片后附件栏显示缩略图横排 + 左上角 × 关闭按钮
  - 输入文本 + 点击发送
  - 确认 `SlideChatManager` 收到含图片的 AIContent

## 风险与注意事项

- **TopLevel 依赖**：`OpenFilePickerAsync` 需要在 UI 线程且有 `TopLevel`（Window）上下文。ViewModel 层不持有 UI 引用，需要从 View 层传入。建议在 `CharUserControl.axaml.cs` 的事件处理中完成文件选择。
- **AttachPreview 与 AttachedImageFiles 共存**：`RunSendMessageAsync` 中先处理用户选择的图片文件路径，再根据 `_attachPreview` 追加渲染预览图。两者不冲突。
- **API 兼容性**：`SlideChatManager.SendMessageAsync` 仅新增可选参数，所有现有调用点无需修改。
- **x:DataType 编译绑定**：CharUserControl 内部已声明 `x:DataType="vm:MainWindowViewModel"`，新增绑定可直接使用编译绑定。
