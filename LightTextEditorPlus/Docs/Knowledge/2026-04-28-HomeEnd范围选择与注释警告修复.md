# HomeEnd范围选择与注释警告修复

## 背景与目标

本次任务是在现有键盘选择能力基础上，补齐 `Shift + Home/End` 与 `Ctrl + Shift + Home/End` 的快捷键行为，同时补充更细粒度的跨段、反向收缩选择测试，并清理构建中的 `CS1573` XML 注释警告。

## 关键文件与模块

- `LightTextEditorPlus.Wpf/Editing/Handlers_/KeyboardHandler.cs`
- `LightTextEditorPlus.Avalonia/Editing/Handlers_/KeyboardHandler.ava.cs`
- `LightTextEditorPlus.Avalonia/Editing/Handlers_/TextEditorHandler.ava.cs`
- `LightTextEditorPlus.Core/Carets/KeyboardCaretNavigation_/KeyboardCaretNavigationHelper.cs`
- `Tests/LightTextEditorPlus.Core.Tests/API/TextEditorCaretTest.cs`

## 主要决策与原因

1. 平台层仅补快捷键绑定，不改 Core 的选择锚点语义。
   - 原因：`TextEditorCore.Select(SelectionType)` 已经统一负责锚点与终点计算，平台只需把按键翻译成对应的 `SelectionType`。
2. `Shift + Home/End` 复用 `SelectionType.LineStart/LineEnd`，`Ctrl + Shift + Home/End` 复用 `SelectionType.DocumentStart/DocumentEnd`。
   - 原因：避免新增重复枚举和重复导航逻辑，保持选择行为与现有 `Select` 映射一致。
3. 更细粒度测试放在 Core 层完成。
   - 原因：跨段选择、反向收缩和行首行尾/文档首尾终点计算都属于核心行为，Core 测试更稳定，也更能保证跨平台一致。
4. `CS1573` 通过补全 XML `param` 标签修复。
   - 原因：这是最小改动方案，不影响运行时行为，只清理构建噪声。

## 修改点摘要

- WPF 新增：
  - `Shift + Home` -> `SelectionType.LineStart`
  - `Shift + End` -> `SelectionType.LineEnd`
  - `Ctrl + Shift + Home` -> `SelectionType.DocumentStart`
  - `Ctrl + Shift + End` -> `SelectionType.DocumentEnd`
- Avalonia 新增同等快捷键绑定，并在按键分支中补齐 `Home/End` 的带修饰键处理。
- Core 测试补充：
  - `ShiftHome` 选择到当前行行首
  - `ShiftEnd` 选择到当前行行尾
  - `CtrlShiftHome` 跨段选择到文档开始
  - `CtrlShiftEnd` 跨段选择到文档结束
  - 已有跨段反向选择时继续 `ShiftRight` 的跨段收缩
  - 已有跨段反向选择时继续 `ShiftEnd` 的反向收缩
- 为 `KeyboardCaretNavigationHelper` 中多个私有方法补齐缺失的 XML 参数注释，消除 `CS1573`。

## 验证方式（构建/测试/手工验证）

- Visual Studio 工作区构建成功。
- 运行：
  - `dotnet test Tests\LightTextEditorPlus.Core.Tests\LightTextEditorPlus.Core.Tests.csproj -c Release --filter TextEditorCaretTest`
  - `dotnet test Tests\LightTextEditorPlus.Tests\LightTextEditorPlus.Tests.csproj -c Release --filter TextEditorHandlerTest`
  - `dotnet test Tests\LightTextEditorPlus.Avalonia.Tests\LightTextEditorPlus.Avalonia.Tests.csproj -c Release --filter TextEditorHandlerTest`
- 重新检查生成输出，确认本次定位到的 `KeyboardCaretNavigationHelper` 上的 `CS1573` 警告已消失。

## 后续建议

1. 如果后续继续补键盘行为，优先检查 `SelectionType` 与 `CaretMoveType` 的复用空间，避免平台层直接分叉出独立逻辑。
2. 可继续补充“已有正向选择后再用 `Shift + Home/End` 收缩”的对称测试，覆盖更多边界。
3. 如果 WPF 侧后续需要更精确验证键位绑定，建议在平台测试中增加 `Home/End` 组合键处理器测试。
