# 段落水平对齐便捷 API 与测试补齐

## 背景与目标

在已补齐段落便捷 API 后，遗漏了 `ParagraphProperty.HorizontalTextAlignment` 的便捷设置入口。
本次目标是补齐水平对齐设置 API（含多入口）以及对应双平台单元测试。

## 关键文件与模块

- `Build/Shared/API/TextEditor.Edit.Shared.cs`
- `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`

## 主要决策与原因

1. 新增统一入口 `SetHorizontalTextAlignment`，提供三种重载：
   - 当前光标段落
   - 指定 `CaretOffset` 段落
   - 指定 `ParagraphIndex` 段落
2. 功能开关检查采用“总开关 + 分项开关”策略：
   - 总开关：`TextFeatures.HorizontalAlign`
   - 分项开关：`AlignHorizontalLeft` / `AlignHorizontalCenter` / `AlignHorizontalRight` / `AlignJustify`
3. 测试沿用既有段落便捷 API 用例结构，在同一场景覆盖水平对齐赋值和开关拦截。

## 修改点摘要

### 1) 共享 API

在 `TextEditor.Edit.Shared.cs` 新增：

- `SetHorizontalTextAlignment(HorizontalTextAlignment horizontalTextAlignment)`
- `SetHorizontalTextAlignment(in CaretOffset caretOffset, HorizontalTextAlignment horizontalTextAlignment)`
- `SetHorizontalTextAlignment(ParagraphIndex index, HorizontalTextAlignment horizontalTextAlignment)`

并新增内部判断方法：

- `CheckHorizontalAlignFeatureDisableWithLog(HorizontalTextAlignment horizontalTextAlignment)`

### 2) WPF 测试

在 `ParagraphStyleApis` 中新增：

- `Left`（当前光标）
- `Center`（指定 `CaretOffset`）
- `Justify`（指定 `ParagraphIndex`）

并验证 `AlignHorizontalRight` 被禁用时设置 `Right` 不生效。

### 3) Avalonia 测试

与 WPF 同步，新增同等水平对齐设置及开关拦截断言。

## 验证方式（构建/测试/手工验证）

- 执行工作区构建：`run_build`
- 结果：构建成功
