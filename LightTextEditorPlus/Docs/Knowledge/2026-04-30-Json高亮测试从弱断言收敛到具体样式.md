# Json 高亮测试从弱断言收敛到具体样式

## 背景与目标

在已有 Json 独立高亮能力之后，`OtherCodeDocumentHighlighterTests` 中多行 Json 测试仍然使用 `AssertTokenUsesNonPlainTextColor` 这一弱断言。

这类断言只能说明 token 不是纯文本色，无法说明它是否被着成了正确的语义样式，因此即便 key、字符串值、数字、常量被错误着色，测试也可能继续通过。

本次任务目标：

1. 将多行 Json 相关单元测试从“非纯文本颜色”弱断言改成“具体 scope 样式”断言；
2. 为 Markdown 文档中的多行 Json 代码块补充同样粒度的样式断言测试；
3. 保持现有实现不变，仅修正测试约束能力。

## 关键文件与模块

- `LightTextEditorPlus.Highlighters.Avalonia.UnitTests/OtherCodeDocumentHighlighterTests.cs`
- `LightTextEditorPlus.Highlighters.Avalonia.UnitTests/MarkdownCodeBlockHighlightingTests.cs`
- `LightTextEditorPlus.Highlighters.Avalonia.UnitTests/DocumentHighlighterTestHelper.cs`

## 主要决策与原因

1. 多行 Json 测试改用 `AssertScopeColor`
   - 对 key 明确要求 `ScopeType.ClassMember`；
   - 对字符串值明确要求 `ScopeType.String`；
   - 对数字明确要求 `ScopeType.Number`；
   - 对 `true/false/null` 明确要求 `ScopeType.DeclarationTypeSyntax`；
   - 这样才能真正约束 Json 语义高亮是否正确。

2. 为 `OtherCodeDocumentHighlighterTests` 增加私有 Json 断言辅助方法
   - 避免在多个 Json 测试里重复写相同的 token 断言循环；
   - 保持测试表达清晰，同时便于后续继续扩展 token 覆盖。

3. 宽松 Json 测试额外验证注释保持纯文本
   - 宽松 Json 解析当前配置为 `CommentHandling.Skip`；
   - 被跳过的注释不应被当成 Json token 着色，因此应明确断言为纯文本色。

4. Markdown 多行 Json 单独补充测试，而不是只复用“与独立高亮一致”
   - 仓库里已经有“Markdown 代码块结果与独立高亮一致”的测试；
   - 但用户明确指出 Json 多行测试缺少具体样式识别，因此需要补一条面向 Json 语义的 Markdown 测试，直接验证 key、值、数字、常量的 scope。

## 修改点摘要

- `OtherCodeDocumentHighlighterTests.cs`
  - 将 `ApplyHighlight_MultiLineLongJson_HighlightsKeysStringsNumbersAndConstants` 中的弱断言替换为具体 scope 断言；
  - 将 `ApplyHighlight_LooseMultiLineLongJson_HighlightsWhenJsonParsingSucceeds` 中的弱断言替换为具体 scope 断言；
  - 新增对宽松 Json 注释 `// dashboard definition` 保持纯文本色的验证；
  - 新增私有辅助方法 `AssertJsonHighlight(...)`，统一做 Json token 的样式断言。

- `MarkdownCodeBlockHighlightingTests.cs`
  - 新增 `ApplyHighlight_CodeBlockWithMultiLineJson_HighlightsJsonScopes`；
  - 使用多行 Json 代码块验证 Markdown 场景下的 key、字符串值、数字、常量 scope；
  - 新增同名私有辅助方法 `AssertJsonHighlight(...)` 用于测试内部复用。

## 后续建议

- 对 Json 相关测试，优先使用 scope 级别断言，不再引入“非纯文本即可”的弱断言模式。
- 如果未来 Markdown 里继续引入 YAML、TOML 等结构化语言，也建议直接补“具体语义 token -> 具体样式”的测试，而不是仅验证存在高亮。
- 当前两个测试文件中各自定义了一个 `AssertJsonHighlight` 私有辅助方法；若后续 Json 测试继续增加，可考虑提取到 `DocumentHighlighterTestHelper` 中统一复用。
