# 指导布局能力实现与测试文档补充

## 背景与目标

为 `LightTextEditorPlus.Core` 增加“指导布局”能力。该能力在文本完成布局后导出一份平台无关的布局快照，供上层文本框或其他模块获取段落、行和字符数量等结果。

本次目标包括：

- 在核心层提供指导布局数据结构与获取入口
- 在布局完成事件中附带指导布局快照
- 补充核心层、WPF、Avalonia 三份测试
- 新增独立文档，并更新现有文档导航与说明

## 主要决策

1. 指导布局不新增第二套布局算法
   - 直接复用现有布局完成后的 `RenderInfoProvider`、段落与行数据生成快照
   - 原因是可以避免核心层出现两套逻辑不一致的问题

2. 指导布局模型按文档、段落、行三级输出
   - 文档级：`GuidingLayoutInfo`
   - 段落级：`GuidingParagraphLayoutInfo`
   - 行级：`GuidingLineLayoutInfo`
   - 原因是既满足“几行、每行多少字符”的最小需求，也保留足够的边界和起始点信息供 UI 层使用

3. 布局完成事件直接携带当次快照
   - 在 `LayoutCompletedEventArgs` 中加入 `GuidingLayoutInfo`
   - 原因是事件订阅方无需再次遍历渲染信息拼装布局结果

## 修改摘要

- 新增指导布局数据结构
- 新增 `TextEditorCore.GetCurrentGuidingLayoutInfo()`
- 新增 `RenderInfoProvider.GetCurrentGuidingLayoutInfo()`
- 修改 `LayoutCompletedEventArgs`，附带 `GuidingLayoutInfo`
- 在 `TextEditorCore` 布局完成时创建并分发指导布局快照
- 增加核心层测试，覆盖：
  - 空文本
  - 空段落
  - 多段中含空段
  - 一段多行
  - 多段多行
  - 各种符号
  - 宽度大于/小于文档宽度
  - 居左、居中、居右
  - 横排、竖排、蒙文竖排
- 增加 WPF 与 Avalonia 测试，验证 UI 层可以读取指导布局结果
- 新增 `Docs/指导布局.md`
- 更新 README 和使用、行为、实现、术语文档

## 验证方式（构建/测试/手工验证）

计划执行以下测试：

- `dotnet test -c Release Tests/LightTextEditorPlus.Core.Tests/LightTextEditorPlus.Core.Tests.csproj`
- `dotnet test -c Release Tests/LightTextEditorPlus.Tests/LightTextEditorPlus.Tests.csproj`
- `dotnet test -c Release Tests/LightTextEditorPlus.Avalonia.Tests/LightTextEditorPlus.Avalonia.Tests.csproj`

同时通过单元测试断言指导布局的段数、行数、字符数、对齐偏移和横竖排方向结果。

## 后续建议

- 如果未来需要“外部输入指导布局并强制应用”，建议新增独立输入模型，不要复用当前只读快照类型
- 可继续补充指导布局与命中测试、光标、项目符号之间的组合测试
- 可考虑后续为平台层公开更直接的指导布局便捷 API
