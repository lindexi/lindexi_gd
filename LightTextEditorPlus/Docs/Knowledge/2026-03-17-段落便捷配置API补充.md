# 段落便捷配置 API 补充

## 背景与目标

现有段落属性公开设置主要依赖 `SetParagraphProperty` 与 `ConfigCurrentCaretOffsetParagraphProperty`，调用侧需要手动构造 `ParagraphProperty`，使用成本偏高。
本次目标是补充直观的段落便捷 API，并接入对应的 `TextFeatures` 功能开关。

## 关键文件与模块

- `Build/Shared/API/TextEditor.Edit.Shared.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`

## 主要决策与原因

1. 便捷 API 放在共享编辑层，确保多平台一致能力。
2. API 直接复用 `ConfigCurrentCaretOffsetParagraphProperty`，减少重复逻辑与变更面。
3. 每个便捷 API 在入口处做功能开关判定，与现有样式 API 行为保持一致。

## 修改点摘要

在 `TextEditor` 共享编辑 API 新增：

- `IncreaseIndentation(double value = 1)`
- `DecreaseIndentation(double value = 1)`
- `SetIndentation(double indent)`
- `SetParagraphSpaceBefore(double paragraphBefore)`
- `SetParagraphSpaceAfter(double paragraphAfter)`
- `SetLineSpacing(ITextLineSpacing lineSpacing)`

并分别接入对应 `TextFeatures` 检查：

- `IncreaseIndentation` / `SetIndentation`
- `DecreaseIndentation` / `SetIndentation`
- `SetParagraphSpaceBefore`
- `SetParagraphSpaceAfter`
- `SetLineSpacing`

测试侧在 Avalonia 样式测试新增：

- 便捷 API 设置生效验证（段前间距 + 缩进增减）
- 功能开关禁用后 API 不生效验证

## 验证方式（构建/测试/手工验证）

- 工作区构建：`run_build`，结果成功。
