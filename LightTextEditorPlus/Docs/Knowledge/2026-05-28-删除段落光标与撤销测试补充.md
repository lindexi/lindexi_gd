# 删除段落光标与撤销测试补充

## 背景与目标

本轮目标是继续补全 `RemoveParagraph` 的测试覆盖，重点验证删除段落时不同光标位置的落点，以及删除段落后的撤销重做在单段、末段、空文本、空段等场景下的文本与光标状态。

## 关键文件与模块

- `Tests/LightTextEditorPlus.Core.Tests/API/TextEditorEditTest.cs`
- `Tests/LightTextEditorPlus.Core.Tests/API/TextEditorUndoRedoTest.cs`
- `LightTextEditorPlus.Core/Document/DocumentManagers_/DocumentManager_/DocumentManager.cs`
- `LightTextEditorPlus.Core/Document/UndoRedo/TextChangeOperation.cs`

## 主要决策与原因

1. 先补测试，再用定向测试确认实际行为，避免在未验证现状前直接修改核心逻辑。
2. 删除段落后的光标行为继续以当前 `RemoveParagraph -> RemoveInner -> EditAndReplaceRunInner` 的统一删除链路为准；对“删除后位于后续段的光标”当前实现会落到删除范围起点，因此测试按现状收敛。
3. 撤销重做继续以 `TextChangeOperation` 的现有语义为准：撤销/重做后光标按替换链路落到恢复内容之后，而不是额外记录并恢复“操作前光标”。
4. 空文本删除唯一空段时，由于删除选择为空，不会生成撤销项，因此测试显式验证该行为，避免误判为缺陷。

## 修改点摘要

### 1) 删除段落光标测试补充

在 `TextEditorEditTest.RemoveParagraphTest` 中新增：

- 光标位于被删除中间段内时的落点断言；
- 光标位于被删除中间段末尾时的落点断言；
- 光标位于被删除段之后时的落点断言。

### 2) 删除段落撤销重做测试补充

在 `TextEditorUndoRedoTest.UndoRedo` 中新增：

- 删除仅有一段文本后的撤销重做；
- 删除末段后的撤销重做；
- 删除空文本唯一段落时的行为；
- 删除中间空段且光标位于后续段时的撤销重做。

这些测试均补充了删除后、撤销后、重做后的光标状态断言。

### 3) 结果说明

本次未修改核心实现。通过新增测试与定向验证确认：当前删除段落与撤销重做的光标行为已经由既有删除/替换基础设施统一决定，新增测试需要对齐该既有语义。

## 后续建议

- 如果后续产品语义希望“撤销段落删除后精确恢复到操作前光标位置”，需要单独扩展 `TextChangeOperation` 或新增专门的撤销数据，而不是只改测试。
- 可继续补一组针对公开 `TextEditor` 入口的集成测试，确保 Core 与共享 API 在删除段落场景下保持一致。
