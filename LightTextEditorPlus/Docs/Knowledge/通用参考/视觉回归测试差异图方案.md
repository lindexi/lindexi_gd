# 视觉回归测试差异图方案

> 从任务日志 `2026-05-29-Avalonia集成测试差异图绘制` 提炼。

---

## 场景

视觉回归测试比对失败时，仅提供"基准图路径"和"当前截图路径"不够直观。需要在失败时自动生成一张**差异总览图**，方便人工快速定位问题。

---

## 方案：Skia 2x2 四宫格

使用 SkiaSharp 将四张图按固定布局拼接：

```
+------------------+------------------+
|   基线图          |   当前截图        |
+------------------+------------------+
| 基线图+红框标注   | 当前截图+红框标注 |
+------------------+------------------+
```

### 布局规则

- 画布宽高：`max(width) * 2` x `max(height) * 2`
- 四张图按左上、右上、左下、右下排列
- 不引入额外缩放，保留原始像素信息

### 红框标注

- 从视觉比对结果的 `VisionCompareRect` 获取差异区域坐标
- 在下排两张图上以红色描边矩形叠加差异区域

---

## 核心代码要点

```csharp
using var baseline = SKBitmap.Decode(baselinePath);
using var current = SKBitmap.Decode(currentPath);
int cellW = Math.Max(baseline.Width, current.Width);
int cellH = Math.Max(baseline.Height, current.Height);

using var canvas = new SKBitmap(cellW * 2, cellH * 2);
using var g = new SKCanvas(canvas);
g.Clear(SKColors.White);

// 上排：原图
g.DrawBitmap(baseline, 0, 0);
g.DrawBitmap(current, cellW, 0);

// 下排：叠加红框
DrawWithRectOverlay(g, baseline, 0, cellH, rects, SKColors.Red);
DrawWithRectOverlay(g, current, cellW, cellH, rects, SKColors.Red);

using var output = canvas.Encode(SKEncodedImageFormat.Png, 100);
using var fs = File.OpenWrite(diffPath);
output.SaveTo(fs);
```

## 后续可扩展

- 四宫格上补充标题文字（"基线图""当前图"等）
- 多分辨率截图场景补充单元格对齐策略
- WPF 集成测试复用同方案
- 差异区域过多时，红框区域的个数和位置可作为统计指标
