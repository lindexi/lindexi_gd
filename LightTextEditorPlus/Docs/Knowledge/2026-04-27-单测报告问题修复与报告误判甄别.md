# 单测报告问题修复与报告误判甄别

## 背景与目标

本次根据单元测试智能体生成的分析报告，核对 LightTextEditorPlus.Core 与 LightTextEditorPlus.Skia 的相关实现，修复其中已经确认的生产代码问题，同时甄别报告中与仓库现有语义或测试前置条件不一致的误判项，避免错误扩大修改范围。

## 主要决策与原因

### 1. 负长度 Selection 需要修复 Front/Behind 排序

`Selection(CaretOffset, int)` 原先直接把 `FrontOffset` 设为起点、`BehindOffset` 设为终点，负长度下会破坏 `FrontOffset <= BehindOffset` 约束。该项确认为真实生产问题，已按现有双 offset 构造函数的语义修复。

### 2. 控制字符问题只放宽调试检查，不直接过滤输入

报告建议包含“输入过滤”“状态修正”“放宽检查”三类方向。结合现有行为与最小改动原则，本次仅调整 `LayoutChecker` 的调试校验，让控制字符在无效运行属性下不再被错误判定为异常，避免扩大到输入管线语义变更。

### 3. `EditAndReplaceRun` 与渲染入口补参数校验

`SkiaTextEditor.EditAndReplaceRun` 的公开 API 传入 null 时应明确抛出 `ArgumentNullException`。`RenderManager.BuildCaretAndSelectionRender` 也应对 `renderInfoProvider` 做空检查，避免隐式空引用异常。

### 4. 波浪线渲染需要对零尺寸边界提前返回

`WaveLineTextEditorDecoration` 在高度为 0 时会把波长算成 0，后续导致除零和异常增长。这里采用提前返回与内部防守双保险，属于典型边界值修复。

### 5. `CharRenderInfoSetter` 的报告部分正确，但测试也需要同步修正

生产代码中的 `Debug.Assert(glyphCluster <= totalUtf16Length)` 与后续 clamp 行为冲突，删除断言是合理的。但原测试对 `in CharDataInfo` 的 Moq 验证方式不正确，需要改为 `It.Ref<CharDataInfo>.IsAny`，否则即便生产代码行为正确也会误报失败。

### 6. `GradientSkiaTextBrushRelativePoint` 报告存在语义误判

报告认为 Absolute 应接受任意数值、Relative 才限制在 `[0,1]`。但仓库内大量既有测试明确要求：

- `Absolute` 与未定义枚举值走 `[0,1]` 范围校验
- `Relative` 不做构造期限制，只在 `ToSKPoint` 中直接参与计算

因此本次**没有**采纳报告中的条件反转建议，只补上 NaN 检查，让现有 Absolute 语义下的 NaN 验证成立。

### 7. 负长度选择替换测试属于前置条件不完整

`Selection` 修复后，`new Selection(new CaretOffset(5), -3)` 已能正确归一化为 `[2,5]`。但若文档为空，替换范围本来就越界，所以测试必须先准备足够文本，不能把越界场景误判为 Selection 构造 bug。

## 修改点摘要

- 修复 `Selection` 在负长度输入下的前后边界归一化。
- 放宽 `LayoutChecker` 对控制字符的调试校验。
- 为 `SkiaTextEditor.EditAndReplaceRun` 增加 `ArgumentNullException.ThrowIfNull`。
- 为 `RenderManager.BuildCaretAndSelectionRender` 增加空参数校验。
- 移除 `RenderManager.GetCurrentTextRender` 中不成立的首次脏状态断言。
- 为 `WaveLineTextEditorDecoration` 增加零宽高提前返回与零波长保护。
- 删除 `CharRenderInfoSetter` 中与 clamp 逻辑冲突的断言。
- 修复 `SolidColorSkiaTextBrush` 仅在有限且合法的透明度范围内调整 Alpha。
- 保留 `GradientSkiaTextBrushRelativePoint` 现有 Absolute 校验语义，仅补充 NaN 校验。
- 向 `LightTextEditorPlus.Skia.UnitTests` 开放 Core/Skia 内部可见性。
- 恢复并修正多项被标记为 `ProductionBugSuspected` 的测试。

### 说明

`RenderManagerExtensionTests` 中的两条跳过测试本次已补充内部可见性基础，但未继续强行构造复杂内部对象，避免引入超出当前问题范围的测试基建变更。

## 后续建议

1. 为 `RenderManagerExtensionTests` 提供稳定的测试构造辅助方法，减少直接依赖复杂内部渲染对象的成本。
2. 对单测智能体报告增加“与现有仓库测试语义冲突”检查，减少把测试前置条件问题误判为生产 bug。
3. 若后续明确希望统一渐变点坐标语义，再集中梳理 `GradientSkiaTextBrushRelativePoint` 的构造校验与所有相关测试，而不要在单个缺陷修复中局部反转行为。
