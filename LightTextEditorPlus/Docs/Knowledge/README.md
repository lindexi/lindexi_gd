## 使用说明

### 新任务后添加知识总结

1. 在 `任务日志/` 下新建文件，命名：`YYYY-MM-DD-任务简述.md`
2. 内容至少包含：背景与目标、关键文件、主要决策、修改摘要、验证方式、后续建议
3. 若是纯通用技术知识（不绑定具体任务），放入 `通用参考/`
4. 更新本 README 索引表

### 提炼知识

- 当某个主题的 `任务日志/` 累计较多时，将共性规律提炼到顶层提炼文档
- 提炼文档保持精炼，详细过程指向 `任务日志/` 中的原始文件

### 归档规则

- 如果任务日志中描述的实现已被完全替换或删除，移入 `_archive/`
- 归档时不删除文件，便于追溯历史

### 任务日志模板

```markdown
# <任务标题>

## 背景与目标

## 关键文件

- `路径A`
- `路径B`

## 主要决策

## 修改摘要

## 验证记录

## 风险与后续建议
```
