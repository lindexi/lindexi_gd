# Json 中文高亮测试拆分与字节偏移修复

## 背景与目标

本次任务聚焦 Json 高亮在中文字符串场景下的单元测试补充与实现修复。

目标有三项：

1. 为独立 Json 高亮补充包含中文字符串的测试用例；
2. 将原本混在 `OtherCodeDocumentHighlighterTests` 里的 Json 相关测试拆到独立测试类，避免 Other 场景继续承载已独立实现的 Json 逻辑；
3. 按测试驱动方式验证实现，如果新增测试失败，则调查并修复 Json 着色实现。

用户给出的关键用例为：

```json
 {   "Key1": "中文",   "Key2": 5 }
```

同时还要求 Markdown fenced code block 中的 Json 也覆盖中文字符串场景，保证 Markdown 内嵌 Json 与独立 Json 高亮行为一致。

## 关键文件与模块

- `LightTextEditorPlus.Highlighters.Avalonia.UnitTests/OtherCodeDocumentHighlighterTests.cs`
- `LightTextEditorPlus.Highlighters.Avalonia.UnitTests/JsonCodeDocumentHighlighterTests.cs`
- `LightTextEditorPlus.Highlighters.Avalonia.UnitTests/MarkdownDocumentHighlighterTests.cs`
- `LightTextEditorPlus.Highlighters/CodeHighlighters/JsonCodeHighlighter.cs`
- `LightTextEditorPlus.Highlighters/OtherCodeDocumentHighlighter.cs`
- `LightTextEditorPlus.Highlighters/MarkdownDocumentHighlighter.cs`

## 主要决策与原因

1. 将 Json 测试从 Other 测试类中完全拆出
   - Json 已经不是单纯依赖 ColorCode 的“其他语言”场景，而是单独实现了语义高亮；
   - 继续放在 `OtherCodeDocumentHighlighterTests` 中，会让测试职责不清晰，也不利于后续补充 Json 专项覆盖。

2. 新增独立测试类 `JsonCodeDocumentHighlighterTests`
   - 这样可以集中覆盖单行、多行、宽松 Json、非法 Json 回退、重复调用、中文字符串等场景；
   - 同时保留通过 `OtherCodeDocumentHighlighter(..., "json")` 驱动实际入口，确保测试覆盖的是实际对外使用路径。

3. Markdown 中单独补一个 Json 中文代码块测试
   - Markdown 代码块的 Json 走独立 Json 高亮路径；
   - 如果只测独立 Json 而不测 Markdown，会遗漏“内嵌代码块偏移量”相关问题。

4. 修复方向选择为“字节偏移转字符偏移”
   - `Utf8JsonReader.TokenStartIndex` 返回的是 UTF-8 字节偏移，不是 .NET 字符串索引；
   - 英文场景下字节数与字符数一致，所以原实现不会暴露问题；
   - 中文在 UTF-8 下通常占 3 个字节，直接拿字节偏移当字符偏移会导致高亮区间错位；
   - 因此修复的核心不是改测试，而是把 Json token 的 UTF-8 字节区间正确换算成字符串字符区间。

## 修改点摘要

- `OtherCodeDocumentHighlighterTests.cs`
  - 删除原有全部 Json 相关测试；
  - 保留 JavaScript、Python、XML、CSS、SQL、PHP、TypeScript、C++、Java 等真正的 Other 场景测试；
  - 移除仅供 Json 测试使用的 `AssertJsonHighlight` 辅助方法。

- `JsonCodeDocumentHighlighterTests.cs`
  - 新增独立 Json 测试类；
  - 迁移原有 Json 测试：
    - 单行 Json key/value/basic type 着色；
    - 多行长 Json；
    - 宽松 Json；
    - 非法 Json 回退到 ColorCode；
    - 重复调用稳定性；
    - RenderBackground / RenderForeground 无异常；
  - 新增中文字符串测试：`{"Key1": "中文", "Key2": 5}`，断言 key、中文字符串值、数字都能按语义着色。

- `MarkdownDocumentHighlighterTests.cs`
  - 新增 `ApplyHighlight_CodeBlockWithJsonChineseString_HighlightsAsJson`；
  - 覆盖 Markdown fenced code block 中 Json 中文字符串的着色行为；
  - 断言 `"Key1"` 为 `ClassMember`，`"中文"` 为 `String`，`5` 为 `Number`。

- `JsonCodeHighlighter.cs`
  - 增加 `using System.Text;`；
  - 将 `TryApplyHighlight` 中的 token 处理从基于 `string code` 改为基于 `byte[] utf8Bytes`；
  - `TryGetTokenSpan` 不再直接把 `TokenStartIndex` 当作字符索引，而是：
    - 先基于 UTF-8 字节数组确定 token 的字节起始与字节长度；
    - 再使用 `Encoding.UTF8.GetCharCount(...)` 将字节区间换算为字符串字符区间；
  - 字符串和属性名的长度计算也改为基于 UTF-8 字节扫描，避免多字节字符时长度错位。

## 后续建议

- 当前 Json 高亮已经验证了中文字符串值场景，后续可继续补：
  - 中文属性名；
  - 转义中文字符串；
  - Emoji 或代理对字符；
  - 中文与英文混合、多层嵌套场景。

- 由于 `Utf8JsonReader` 暴露的是字节偏移，凡是未来继续基于它做定位的实现，都应优先警惕“字节偏移 != 字符偏移”问题；不要再默认按 ASCII 假设处理。

- Markdown 代码块与独立 Json 高亮已经有共同覆盖，后续若修改 Json token 映射规则，建议两边测试一起补，避免入口行为再次分叉。
