# 公开 API 功能开关补齐与测试文档补充

## 背景与目标

用户反馈平台层公开 API 对 `TextFeatures` 功能开关使用不完整，典型是 `SetFontSize` 未在公开入口做功能禁用判定。目标是：

- 在公开 API 层补齐功能开关判断；
- 补充对应单元测试；
- 在使用文档中增加功能开关用法示例。

## 关键文件

- `LightTextEditorPlus.Wpf/API/TextEditor.Edit.cs`
- `LightTextEditorPlus.Avalonia/API/TextEditor.Edit.ava.cs`
- `Build/Shared/API/TextEditor.Edit.Shared.cs`
- `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`
- `Docs/使用说明文档.md`

## 主要决策

1. **样式功能开关集中在平台层 `SetRunProperty` 做映射校验**：
   - 按 `PropertyType` 映射到 `TextFeatures`（字号/字体名/前景色/斜体/加粗）；
   - 这样可覆盖大部分公开样式 API，减少重复判断。
2. **上下标单独校验**：
   - `PropertyType.FontVariants` 粒度不足以区分上标/下标；
   - 在 `ToggleSuperscript`、`ToggleSubscript`、`SetFontVariants` 中增加细粒度判断。
3. **段落属性按“属性差异”映射功能开关**：
   - 在共享公开 API `SetParagraphProperty`/`SetStyleParagraphProperty` 中，先比较旧值与新值；
   - 仅在实际变更对应属性时检查并拦截对应功能（行距、缩进、水平对齐、段前/段后）。

## 修改摘要

- WPF/Avalonia：
  - 新增 `CheckStyleFeatureDisableWithLog(PropertyType)`；
  - 在 `SetRunProperty` 入口提前返回；
  - 上下标公开 API 增加 `SetFontSuperscript`/`SetFontSubscript` 判断。
- 共享段落 API：
  - `SetParagraphProperty(in CaretOffset, ...)`、`SetParagraphProperty(ParagraphIndex, ...)`、`SetStyleParagraphProperty(...)` 接入功能开关；
  - 新增 `CheckParagraphPropertyFeatureDisableWithLog` 与 `CheckHorizontalAlignFeatureDisableWithLog`。
- 单元测试：
  - WPF 测试新增功能开关用例（禁用字号、禁用段前间距）；
  - Avalonia 测试新增对应用例。
- 文档：
  - 在 `Docs/使用说明文档.md` 的“功能开关”章节补充禁用/启用/查询与示例代码。

## 风险与后续建议

- 当前功能开关粒度由 `TextFeatures` 决定，部分公开 API（如通用 `SetRunProperty`）无法精确推断调用方具体意图；如果后续需要更细控制，建议扩展 `TextFeatures` 粒度并在公开 API 做更明确映射。
- 可考虑补充针对禁用日志（`TextFeaturesBeDisabledLogInfo`）的断言测试，提高行为可观测性。
