## 1. 文本编辑器领域概念

### 1.1 布局管线（Layout Pipeline）

文本从段落模型到屏幕像素经过三个有序阶段：

```
Paragraph → 测量(Measurement) → 断行(Line Breaking) → 渲染(Rendering) → 屏幕
```

- **测量**：对每个字符调用字体引擎，获取其宽度和高度。涉及字体回退（当前字体不支持某字符时切换备用字体）。
- **断行**：根据测量结果和可用宽度，决定每行包含哪些字符。支持 CJK 禁则、词边界断行。
- **渲染**：将断行结果交给平台特定渲染器（WPF GlyphRun / Avalonia FormattedText / Skia Canvas）。

### 1.2 指导布局（Guide Layout）

- 一种特殊的布局模式，通过预定义的"指导线"影响文本排布。
- 常用于需要精确控制字符位置的场景（如注释标注对齐）。

### 1.3 竖排（Vertical Layout）

- 支持两种竖排模式：
  - **竖向（Vertical）**：从上到下，列从右到左（中文/日文传统）。
  - **蒙古文竖排（Mongolian）**：从上到下，列从左到右。
- 竖排时字符度量尺寸的宽高互换。
- 断行方向从水平变为垂直。

### 1.4 RunProperty 装饰集合

- `RunProperty` 不仅包含粗体/斜体/字号/颜色等基础属性，还有**装饰集合（Decorations）**。
- 装饰包括：下划线（直线/波浪线）、删除线（直线/波浪线）、上划线等。
- 装饰是独立可叠加的，一个文本段可以同时有下划线和删除线。
- 装饰的渲染位置和样式由平台渲染器决定。

### 1.5 字体回退（Font Fallback）

- 当当前字体不包含某个字符的字形时，按回退链依次尝试备用字体。
- 回退链在不同平台上行为不同（WPF 有内置回退，Avalonia/Skia 需要显式配置）。
- 字体回退结果影响测量宽度，必须在测量阶段确定。

## 2. 高亮/着色架构

### 2.1 高亮器设计

- 每个语言一个 Highlighter 类，返回一个 `HighlightSpan` 列表。
- `HighlightSpan` 包含：起始偏移、长度、样式类别（关键字/字符串/注释/数字等）。
- 高亮器**不依赖** Core 或任何平台，是纯文本处理。
- 平台绑定层将样式类别映射到实际的 Brush、FontWeight、TextDecoration。

### 2.2 已支持语言

- Markdown
- JSON
- XML
- OtherCode（通用代码，关键字匹配）

## 3. UTF-16 偏移处理

### 3.1 问题

- C# 的 `string` 使用 UTF-16 编码，代理对（Surrogate Pairs）占用两个 `char`。
- 某些 Unicode 字符（如 Emoji、罕见汉字）在 C# 中 `Length` 为 2。
- 如果偏移计算不考虑代理对，会导致光标位置错位、选区异常。

### 3.2 解决方案

- `TextOffsetHelper` 工具类统一处理代理对。
- 内部偏移使用"字符索引"（逻辑字符数），在需要 `string` 索引时通过 `TextOffsetHelper` 转换。
- 新增功能时需考虑代理对测试用例。

## 4. .NET 多语言最佳实践

### 4.1 方案

- 使用 `.resx` 文件存储本地化字符串。
- 通过 `ResourceManager` 在运行时加载。
- 每种语言生成卫星程序集（Satellite Assembly），放在对应语言子目录下（如 `zh-CN/TextEditor.resources.dll`）。

### 4.2 本项目决策

- 采用 resx + 卫星程序集方案，而非 JSON 或 YAML 资源文件。
- 原因：与 .NET 生态原生集成，工具链支持好（VS/Rider 设计器），运行时无需额外解析。

## 5. NativeAOT PInvoke 限制

### 5.1 问题

- `Skia.AotLayer` 项目使用 NativeAOT 编译，需要通过 PInvoke 调用 Skia 原生库。
- NativeAOT 的 `[UnmanagedCallersOnly]` 方法**不能**使用 `out` 或 `ref` 参数。

### 5.2 应对

- 需要传出多个值的函数改用结构体指针或返回值元组。
- PInvoke 签名在 AOT 层和非 AOT 层可能不同，通过条件编译处理。

## 6. 常见陷阱速查

| 陷阱 | 说明 |
|---|---|
| UTF-16 代理对 | `string.Length` ≠ 逻辑字符数，需 `TextOffsetHelper` 转换 |
| `\r\n` vs `\n` | 内部统一 `\n`，入口归一化，出口按需转换 |
| NativeAOT `out/ref` | `UnmanagedCallersOnly` 不支持，改用指针或结构体 |
| 事件泄漏 | 不使用 `event`，使用构造注入回调接口 |
| 功能开关放错位置 | 在公开 API 入口判断，不在内部通用方法拦截 |
| 测试不对称 | WPF 和 Avalonia 测试必须对称对齐 |
| 字体回退未配置 | Avalonia/Skia 需要显式配置回退链，不像 WPF 有内置回退 |
