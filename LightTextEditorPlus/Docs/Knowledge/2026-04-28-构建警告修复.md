## 主要决策与原因

1. `CS0168` 直接删除未使用的异常变量名，保持原异常处理流程不变。
2. `CS8625` 优先修改测试构造方式，而不是放宽生产代码空引用约束，避免为了测试污染正式 API。
3. `MSTEST0006` 与 `MSTEST0039` 统一改为 `Assert.ThrowsExactly`，以满足当前 MSTest 分析器建议，并让异常断言更精确。
4. `WaveLineTextEditorDecorationTests` 中不再向 `List<CharData>` 填充 `default`，改为构造真实 `CharData`，避免空引用字面量触发 nullable 警告。

## 修改点摘要

- 清理 Core 布局代码中的未使用异常变量。
- 将部分依赖 `Mock<SkiaTextEditor>(null)` 的测试改为使用真实 `SkiaTextEditor` 实例。
- 将多个测试从 `Assert.ThrowsException` 或 `[ExpectedException]` 改为 `Assert.ThrowsExactly`。
- 为波浪线装饰测试补充合法 `CharData` 测试数据构造方式。

## 验证方式（构建/测试/手工验证）

- Visual Studio 工作区重新生成，确认此前收集到的构建警告不再出现。
- 运行：`dotnet test "Tests/LightTextEditorPlus.Skia.UnitTests/LightTextEditorPlus.Skia.UnitTests.csproj" -c Release`
- 结果：总计 833，成功 816，跳过 17，失败 0。

## 后续建议

1. 若后续继续补测试，优先避免使用 `[ExpectedException]`，统一采用 `Assert.ThrowsExactly` / `Assert.ThrowsExactlyAsync`。
2. 对需要占位 `CharData` 的测试，优先复用真实最小对象构造，避免使用 `default` 引入 nullable 警告。
3. 若要持续压缩构建噪声，可定期检查测试项目中的分析器警告并批量统一断言风格。
