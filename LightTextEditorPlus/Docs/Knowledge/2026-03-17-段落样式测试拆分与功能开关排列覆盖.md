# 背景与目标

在 `TextEditorStyleTest` 中，`ParagraphStyleApis`/`ParagraphStyleApisShouldApply` 将多个段落属性方法集中在单个测试模块中，不利于定位失败原因与维护。
本次目标是在不修改原有测试的前提下，新增更细粒度的段落样式测试：

- 将“按段落设置”与“按光标设置”拆分为两个独立测试模块；
- 补齐段落相关便捷 API 的逐项覆盖；
- 增加“多次设置-读取最后值”验证；
- 增加“功能开关动态启用/禁用排列”验证。

# 关键文件与模块

- `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`
- 段落相关 API 定义参考：`Build/Shared/API/TextEditor.Edit.Shared.cs`

# 主要决策与原因

1. **保留原有测试不变**：避免影响既有回归覆盖和历史语义。
2. **新增而非重构**：通过新增测试模块满足最佳实践要求，降低改动风险。
3. **按入口拆分模块**：
   - `ParagraphIndex` 入口覆盖按段落设置；
   - `CaretOffset`/当前光标入口覆盖按光标设置。
4. **加入状态序列验证**：功能开关不仅验证静态禁用，还验证“禁用→启用→再禁用”动态排列行为。

# 修改点摘要

## WPF 测试项目

在 `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs` 新增：

- `ParagraphStyleApisByParagraphIndex`
  - 逐项覆盖：`SetIndentation`、`IncreaseIndentation`、`DecreaseIndentation`、`SetParagraphSpaceBefore`、`SetParagraphSpaceAfter`、`SetLineSpacing`、`SetHorizontalTextAlignment`（`ParagraphIndex` 重载）；
  - 增加多次设置读取最后值验证；
  - 增加功能开关动态启用禁用排列验证。
- `ParagraphStyleApisByCaretOffset`
  - 逐项覆盖：`SetIndentation`、`IncreaseIndentation`、`DecreaseIndentation`、`SetParagraphSpaceBefore`、`SetParagraphSpaceAfter`、`SetLineSpacing`、`SetHorizontalTextAlignment`（`CaretOffset` 与当前光标重载）；
  - 增加多次设置读取最后值验证；
  - 增加功能开关动态启用禁用排列验证。

## Avalonia 测试项目

在 `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs` 新增：

- `ParagraphStyleApisByParagraphIndexShouldApply`
- `ParagraphStyleApisByCaretOffsetShouldApply`
- `ParagraphStyleApisRepeatedSetAndReadShouldUseLatestValue`
- `ParagraphStyleApisFeatureSwitchDynamicPermutationShouldApply`

覆盖内容与 WPF 侧保持一致：入口拆分、重复设置读取、动态功能开关排列。

# 验证方式（构建/测试/手工验证）

- 构建验证：执行工作区构建，结果“生成成功”。
- 单测验证：新增测试位于 WPF/Avalonia 对应测试集，可在后续 CI 或本地 `dotnet test` 中单独回归。

# 后续建议

1. 后续新增段落 API 时，保持“按段落入口/按光标入口”双模块同步扩展。
2. 可逐步将历史“多行为合并测试”迁移为“单行为测试”，提高失败定位效率。
3. 可考虑为段落 API 的功能开关组合引入参数化测试模板，减少样板代码。
