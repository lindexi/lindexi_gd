# 异常消息多语言资源框架

## 背景与目标

当前文本库的异常消息长期以中文硬编码方式分散在 `Core`、`Wpf`、`Skia`、`Avalonia` 以及共享 API 文件中。这会影响非中文开发者接入，也不利于后续逐语种翻译。

本次工作的目标是：

1. 先搭建 .NET 类库的异常本地化框架。
2. 将当前异常消息统一收敛到资源文件。
3. 通过卫星程序集提供 `zh-Hans` 中文资源，并让主程序集保留完整英文中性资源，形成可继续扩展到更多语种的基础。

## 主要决策与原因

### 1. 主程序集使用英文中性资源，中文放入 `zh-Hans` 卫星程序集

这是 .NET 类库本地化更稳妥的做法：

- 主程序集保留完整 fallback 资源，避免资源缺失导致异常链路再次失败。
- 英文作为中性语言，便于国际开发者理解。
- 中文通过 `zh-Hans` 卫星程序集承载，保留当前中文体验。
- 后续新增 `ja`、`fr`、`de` 等语种时，只需增加新的 `ExceptionMessages.<culture>.resx`。

### 2. 每个程序集独立维护一份 `ExceptionMessages`

没有把全部异常资源强行放到单独资源程序集里，而是按程序集各自维护。原因：

- 异常最终是在各自程序集内部抛出。
- .NET `ResourceManager` 会按当前程序集自然生成对应卫星程序集。
- 避免跨程序集取资源时引入额外耦合。
- `Build/Shared/API` 虽是共享源码，但最终编译进 `Wpf`、`Skia`、`Avalonia` 各自程序集，因此仍然应依赖各自程序集内的资源文件。

### 3. 资源读取统一走 `ExceptionMessages.Get/Format`

封装统一入口的原因：

- 避免业务代码直接拼 `ResourceManager`。
- 统一使用 `CurrentUICulture` 选资源、`CurrentCulture` 做格式化。
- 资源缺失时尽早抛出 `MissingManifestResourceException`，便于发现漏项。

## 修改点摘要

1. 为 `Core`、`Wpf`、`Skia`、`Avalonia` 新增异常资源访问器与资源文件。
2. 为上述程序集添加 `NeutralResourcesLanguage("en")`。
3. 将当前库内活跃异常抛出文本从硬编码迁移到资源文件。
4. 将多个 `Message` / `ToString()` override 也改为资源驱动，避免只处理直接 `throw` 而遗漏自定义异常消息。
5. 为共享 API 增加资源化接入，使 `Build/Shared/API` 内的异常消息在 `Wpf` / `Skia` / `Avalonia` 中各自通过本程序集资源解析。
6. 新增测试覆盖：
   - 英文中性资源回退。
   - `zh-Hans` 卫星程序集命中。

## 验证方式（构建/测试/手工验证）

### 自动验证

计划执行：

- `dotnet test -c Release`（按仓库要求使用 Release）

同时增加以下自动化断言：

- `Core`：`fr-FR` 回退到英文中性资源；`zh-Hans` 命中中文卫星资源。
- `Wpf`：共享 API 异常在 `fr-FR` / `zh-Hans` 下分别取到英文与中文。
- `Avalonia`：顶层 API 与 `Skia` 共享 API 在 `fr-FR` / `zh-Hans` 下分别取到英文与中文。

### 手工验证建议

可在宿主应用中临时切换：

- `CultureInfo.CurrentUICulture = CultureInfo.GetCultureInfo("fr-FR")`
- `CultureInfo.CurrentUICulture = CultureInfo.GetCultureInfo("zh-Hans")`

再触发任一异常路径，确认：

- `fr-FR` 下显示英文中性消息。
- `zh-Hans` 下显示中文消息。

## 后续建议

1. 继续把示例工程（`Demos`）中的异常/提示文本与库本体分离处理，避免示例项目影响库国际化判断。
2. 下一阶段可按优先级追加更多资源文件：
   - `ExceptionMessages.ja.resx`
   - `ExceptionMessages.fr.resx`
   - `ExceptionMessages.de.resx`
3. 后续新增异常时，优先先加英文中性资源键，再补 `zh-Hans` 翻译，避免重新出现硬编码。
4. 可以考虑后续补一个分析器或 CI 检查：扫描库项目中 `throw new ...("...")` 的硬编码字符串，防止回归。
