# 功能开关入口迁移与样式测试扩展

## 背景与目标

在前一版实现中，平台层将样式功能开关判断统一放在 `SetRunProperty` 内部。用户要求调整为：

- 具体公开样式 API（如 `SetFontSize`、`SetFontName`）在方法入口自行判断；
- 通用 API `SetRunProperty` 不受功能开关拦截，直接允许业务层调用。

同时要求提升 WPF/Avalonia 样式功能开关测试覆盖率。

## 关键文件

- `LightTextEditorPlus.Avalonia/API/TextEditor.Edit.ava.cs`
- `LightTextEditorPlus.Wpf/API/TextEditor.Edit.cs`
- `Build/Shared/API/TextEditor.Edit.Shared.cs`
- `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`

## 主要决策

1. **入口前移**：将字体名、字号、前景色、粗体、斜体、下划线、删除线等判断放到公开方法入口。
2. **通用直通**：删除 `SetRunProperty` 内的统一样式开关判断与辅助方法，确保其可直接执行。
3. **保持段落开关逻辑**：恢复共享段落 API 的开关判断（行距、缩进、水平对齐、段前/段后），避免行为回归。
4. **测试覆盖扩展**：WPF 与 Avalonia 均增加多功能点禁用测试，并补充“禁用后 `SetRunProperty` 仍生效”的关键断言。

## 修改摘要

- Avalonia/WPF 平台编辑 API：
  - 新增方法入口判断：`SetFontName`、`SetFontSize`、`SetForeground`、`SetFontStyle`、`SetFontWeight`、`ToggleUnderline`、`ToggleStrikethrough`。
  - 保留上下标入口判断（之前已加）。
  - 删除 `SetRunProperty` 的统一开关拦截逻辑。

- 共享段落 API：
  - 恢复 `SetParagraphProperty`、`SetStyleParagraphProperty` 的功能开关判断。

- 测试扩展：
  - WPF：新增 `FeaturesSwitchStyleApis`，覆盖字体名、颜色、粗斜体、上下标、下划线、删除线、`SetRunProperty` 直通。
  - Avalonia：新增 `FeatureSwitchStyleApisShouldNotApply`，覆盖同类场景。

## 风险与后续建议

- 目前样式功能点测试已覆盖主要公开入口，后续可继续补充对日志 `TextFeaturesBeDisabledLogInfo` 的断言。
- 若后续扩展更多样式 API，建议在新增公开入口时同步添加对应功能开关用例，避免再次回归到统一拦截模式。
