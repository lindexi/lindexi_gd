# 背景与目标

补齐 `TextEditor` 公开 API 中 `IncreaseFontSize` 与 `DecreaseFontSize` 的实现，并在 Avalonia 与 WPF 两套样式测试中增加对应单元测试。

# 关键文件与模块

- `Build/Shared/API/TextEditor.Edit.Shared.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`
- `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs`

# 主要决策与原因

- 采用现有 `SetRunProperty(..., PropertyType.FontSize, selection)` 路径实现字号增减。
- 该路径已统一处理选择范围解析、样式变更事件触发、跨平台行为一致性，因此改动最小且风险最低。

# 修改点摘要

1. 在共享编辑 API 中实现：
   - `IncreaseFontSize`：`FontSize = FontSize + 1`
   - `DecreaseFontSize`：`FontSize = FontSize - 1`
2. 在 Avalonia 测试中新增字号增减测试：
   - 先设置字号为 20，再验证增加后为 21、减少后回到 20。
3. 在 WPF 测试中新增同等行为测试：
   - 同样验证 `+1` 与 `-1` 结果。

# 验证方式（构建/测试/手工验证）

- 执行工作区构建：`run_build`
- 构建结果：成功。

# 后续建议

- 如需约束字号下限（例如不小于 1），可在 `DecreaseFontSize` 增加边界策略并同步补充边界测试。
- 可后续增加“多种不同字号混排选区”的增减测试，验证每段字号均按相对值变化。
