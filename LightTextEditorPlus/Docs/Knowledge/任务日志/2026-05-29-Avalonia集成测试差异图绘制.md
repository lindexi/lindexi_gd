# Avalonia集成测试差异图绘制

## 背景与目标

Avalonia 集成测试在视觉比对失败时，原先只输出“基准图路径”和“当前截图路径”，缺少一张便于人工快速定位问题的差异总览图。

本次目标是在 `LightTextEditorPlus.Avalonia.Tests` 的集成测试里补充差异图片生成能力：使用 Skia 将四张图按两行两列拼接输出，第一行为基准图和当前截图，第二行为分别叠加红色差异框后的基准图和当前截图，并保存到 `Difference` 目录。

## 关键文件

- `Tests/LightTextEditorPlus.Avalonia.Tests/IntegrationTests/IntegrationTest.cs`
- `Tests/TextVisionComparer/VisionCompareRect.cs`

## 主要决策与原因

### 1. 直接在现有集成测试文件内补充最小辅助方法

没有额外新建类型，而是在 `IntegrationTest.cs` 中新增 `SaveDifferenceImage` 私有方法。

原因：

- 当前能力只在视觉比对失败路径使用
- 逻辑与集成测试上下文强相关
- 可以保持修改面最小，避免过早抽象

### 2. 使用 `VisionCompareRect` 的 `X/Y/Width/Height`

绘制红框时直接复用 `VisionCompareRect` 已提供的坐标和尺寸语义，不重新计算边界。

原因：

- 避免重复推导 `Right/Bottom` 到宽高的换算逻辑
- 与现有视觉对比结果对象保持一致

### 3. 采用 2x2 固定拼图布局，并使用较大宽高作为单元格尺寸

最终画布宽高为 `max(width) * 2` 与 `max(height) * 2`，四张图按左上、右上、左下、右下排列。

原因：

- 两张输入图理论上通常同尺寸，但使用最大值更稳妥
- 布局规则简单直接，便于调试时快速对照
- 不引入额外缩放，保留原始像素信息

## 修改点摘要

- 在 `IntegrationTest.cs` 中引入 `SkiaSharp`
- 在视觉比对失败分支中，调用 `SaveDifferenceImage`
- 新增差异图保存逻辑：
  - 读取基准图与当前截图
  - 创建 2x2 白底画布
  - 绘制原图与当前图
  - 在下排两张图上根据 `VisionCompareRect` 叠加红色描边矩形
  - 以 PNG 格式保存到 `diffImageFile`

验证结果：

- 解决方案构建成功

手工验证建议：

- 人为制造一组视觉差异，确认 `Difference` 目录下生成的 PNG 是否符合 2x2 排版预期
- 确认红框位置与 `VisionCompareRect` 输出范围一致

## 后续建议

- 如后续需要更强可读性，可在四宫格上补充标题文字，如“基准图”“当前图”“基准图差异标注”“当前图差异标注”
- 如后续存在多分辨率截图场景，可考虑补充单元格对齐策略或边距样式
- 若 WPF 集成测试也需要相同诊断体验，可复用同样的差异图生成思路补齐
