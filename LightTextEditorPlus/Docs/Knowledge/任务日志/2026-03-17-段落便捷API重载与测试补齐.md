# 段落便捷 API 重载与测试补齐

## 背景与目标

上一轮补充了段落便捷方法，但仅覆盖“当前光标所在段落”。本轮目标是：

- 为新增段落便捷方法补齐 `CaretOffset` 与 `ParagraphIndex` 入口；
- 在 WPF 与 Avalonia 两套测试中覆盖所有新增方法（含 `SetIndentation`）；
- 保持与 `TextFeatures` 功能开关一致的拦截行为。

## 关键文件与模块

- `Build/Shared/API/TextEditor.Edit.Shared.cs`
- `Tests/LightTextEditorPlus.Tests/TextEditorStyleTest.cs`
- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`

## 主要决策与原因

1. 现有“当前光标”方法改为转发到 `CaretOffset` 重载，减少重复逻辑。
2. 每个便捷方法提供三种入口：
   - 当前光标段落；
   - 指定 `CaretOffset` 段落；
   - 指定 `ParagraphIndex` 段落。
3. 功能开关检查继续保留在公开 API 入口，确保行为一致性。

## 修改点摘要

### 1) 共享 API 增强

为以下方法均补齐了 `in CaretOffset` 与 `ParagraphIndex` 重载：

- `IncreaseIndentation`
- `DecreaseIndentation`
- `SetIndentation`
- `SetParagraphSpaceBefore`
- `SetParagraphSpaceAfter`
- `SetLineSpacing`

并保持对应 `TextFeatures` 判定：

- `IncreaseIndentation` + `SetIndentation`
- `DecreaseIndentation` + `SetIndentation`
- `SetIndentation`
- `SetParagraphSpaceBefore`
- `SetParagraphSpaceAfter`
- `SetLineSpacing`

### 2) WPF 测试补齐

在 `TextEditorStyleTest` 新增 `ParagraphStyleApis`，覆盖：

- 当前光标入口：`SetIndentation`
- 指定光标入口：`IncreaseIndentation` / `SetParagraphSpaceBefore` / `SetLineSpacing`
- 指定段落入口：`DecreaseIndentation` / `SetParagraphSpaceAfter`
- 功能开关禁用后不生效

### 3) Avalonia 测试补齐

扩充既有段落便捷 API 测试，覆盖与 WPF 对齐的全部新增方法及重载入口，并验证功能开关拦截。

- 结果：构建成功
