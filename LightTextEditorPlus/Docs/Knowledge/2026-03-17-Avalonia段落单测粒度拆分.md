# 背景与目标

用户反馈 `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs` 中新增段落测试仍存在“一个测试方法覆盖多个段落设置行为”的问题。
目标是对齐 WPF 侧风格：保证一个单元测试模块只测试一个小点，并保留原有历史测试不改。

# 关键文件与模块

- `Tests/LightTextEditorPlus.Avalonia.Tests/TextEditorStyleTest.cs`

# 主要决策与原因

1. 将聚合测试方法拆分为多个细粒度 `[TestMethod]`，每个方法仅验证一个 API 行为。
2. 保留原有历史测试（如 `ParagraphStyleApisShouldApply`）不改，避免引入回归风险。
3. 新增测试继续覆盖四类需求：
   - `ParagraphIndex` 入口；
   - `CaretOffset`/当前光标入口；
   - 多次设置读取最后值；
   - 功能开关动态禁用/启用排列。

# 修改点摘要

将以下聚合方法拆分为单行为测试：

- 原：`ParagraphStyleApisByParagraphIndexShouldApply`
- 原：`ParagraphStyleApisByCaretOffsetShouldApply`
- 原：`ParagraphStyleApisRepeatedSetAndReadShouldUseLatestValue`
- 原：`ParagraphStyleApisFeatureSwitchDynamicPermutationShouldApply`

拆分后新增细粒度测试，覆盖：

- `Set/Increase/DecreaseIndentation`（ParagraphIndex、CaretOffset）
- `SetParagraphSpaceBefore/After`（ParagraphIndex、CaretOffset）
- `SetLineSpacing`（ParagraphIndex、CaretOffset）
- `SetHorizontalTextAlignment`（ParagraphIndex、CaretOffset、当前光标）
- 重复设置最后值读取（Indentation / ParagraphBefore / LineSpacing / Alignment）
- 功能开关动态排列（`SetParagraphSpaceAfter`、`AlignHorizontalCenter`、`HorizontalAlign`）

# 验证方式（构建/测试/手工验证）

- 构建验证：执行工作区构建，结果为“生成成功”。

# 后续建议

1. 后续新增便捷 API 时，优先按“单行为测试”直接落地，避免先写聚合测试再拆分。
2. 可为段落 API 建立统一测试模板（输入、动作、单断言）以提升维护效率。
