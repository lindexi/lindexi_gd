# 指导布局

## 背景

指导布局也称指导渲染。`LightTextEditorPlus.Core` 不直接负责具体平台渲染，因此在核心层统一使用“指导布局”这一命名。

指导布局的目标有两部分：

1. 在一次文本布局完成后，输出一份稳定的、平台无关的布局快照
2. 允许上层将一份指导布局重新设置回文本库，用来指导后续布局与渲染

## 提供的信息

当前版本提供以下最基础也是最重要的信息：

- 文档当前的 `ArrangingType`
- 一共有多少段 `ParagraphCount`
- 一共有多少行 `LineCount`
- 每段有多少行
- 每行包含多少个 `CharData` 字符
- 每行的起始点和内容起始点
- 每段、每行的内容范围和外接范围

其中“文本框应该被布局为几行，一行有多少个 CharData 字符”属于指导布局的核心最小信息集。

## 获取方式

### 从 `TextEditorCore` 获取

```csharp
TextEditorCore textEditorCore = ...;
GuidingLayoutInfo guidingLayoutInfo = textEditorCore.GetCurrentGuidingLayoutInfo();
```

调用前要求文本已经完成布局。若文本仍然是脏的，则应先等待布局完成，或者走既有的渲染信息获取流程。

### 设置到 `TextEditorCore`

```csharp
TextEditorCore textEditorCore = ...;
GuidingLayoutInfo guidingLayoutInfo = ...;

bool isSuccess = textEditorCore.SetGuidingLayoutInfoForNextUpdateLayout(guidingLayoutInfo);
```

设置成功后，后续布局会优先按照指导布局中的段落与每行字符数量进行排版。

## 数据结构

### `GuidingLayoutInfo`

表示整个文档的一次指导布局快照。

包含：

- `ArrangingType`
- `ParagraphList`
- `ParagraphCount`
- `LineCount`
- `CharCount`

### `GuidingParagraphLayoutInfo`

表示单个段落的指导布局结果。

包含：

- `ParagraphIndex`
- `ContentBounds`
- `OutlineBounds`
- `LineList`
- `LineCount`
- `CharCount`

### `GuidingLineLayoutInfo`

表示单行的指导布局结果。

包含：

- `ParagraphIndex`
- `LineIndex`
- `CharCount`
- `ContentBounds`
- `OutlineBounds`
- `StartPoint`
- `ContentStartPoint`

其中：

- `StartPoint` 表示该行在当前排版坐标系下的行起始点
- `ContentStartPoint` 表示扣除缩进、对齐空白后的真正内容起始点

## 行数和字符数的理解

- 空文本：对外依然表现为一段一行，但该行 `CharCount` 为 0
- 空段落：同样表现为一段一行，但该行 `CharCount` 为 0
- 多段文本：按段输出，每段独立统计其行数与字符数
- 一段多行：通过 `LineList` 可以直接拿到每一行的字符数
- 对齐变化：不改变行内字符数，但会影响 `ContentStartPoint`
- 横排、竖排、蒙文竖排：都通过统一结构输出，只是点位含义跟随当前 `ArrangingType`

## 适用场景

指导布局适合用于：

- 文本框自身根据核心层结果决定外部布局
- 平台层渲染前后的信息对齐
- 调试某段文本实际断成了几行
- 做跨平台一致性校验
- 编写针对排版行为的单元测试
- 将一份既有布局结果复用于另一个文本框的布局渲染

## 校验规则

在设置指导布局之前，文本库会校验其是否与当前文本状态匹配。当前会检查：

- `ArrangingType` 是否一致
- 段落数量是否一致
- 每段字符总数是否一致
- 每行字符数之和是否等于对应段落字符总数
- 空段是否仍然满足一段一行且字符数为 0

校验失败时：

- 本次布局不会应用该指导布局
- 会通过 `ITextLogger` 输出警告日志
- 当前这次布局继续使用默认布局逻辑

## 当前限制

当前指导布局已经支持反向驱动布局，但驱动粒度主要还是“每段几行、每行多少个字符”。

也就是说：

- 可以读取当前布局结果
- 可以按指导布局指定的每行字符数重新布局
- 但还不能精确强制每个字符使用给定的点位、边界或对齐偏移

如果后续需要支持更强的指导能力，例如精确复用每行坐标、每行边界甚至字符点位，则需要继续扩展输入模型与布局约束规则。
