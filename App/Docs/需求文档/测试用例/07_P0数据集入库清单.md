## 四、#4 50MB+ 大图（5 张）

### 用途

- 验证 AC-12 性能指标（20MB JPEG 加载 ≤ 500ms）
- 验证 M10 性能测试
- 验证 IF-B `[正常 → 大图加载中 → 正常]` 过渡

### 数据明细

| # | 文件名 | 像素 | 大小 | 格式 | 用途 |
|---|---|---|---|---|---|
| 1 | large_50mb_01.jpg | 8000×6000 | 50MB | JPEG | 验证 50MB 加载 ≤ 500ms（理论推算） |
| 2 | large_50mb_02.jpg | 10000×7500 | 50MB | JPEG | 同上，不同尺寸 |
| 3 | large_60mb_01.png | 8000×6000 | 60MB | PNG | PNG 解码性能对比 |
| 4 | large_80mb_01.png | 10000×8000 | 80MB | PNG | 极限尺寸 PNG 解码 |
| 5 | large_100mb_01.jpg | 12000×9000 | 100MB | JPEG | 极端压力测试 |

### AC 溯源

- **AC-12**（性能）— 加载时间严格 ≤ 500ms（50MB 推算）
- **R10**（性能波动）— 每张测 5 次取中位数

### 性能测试方法

1. 重启机器后冷启动 → 计时到第一帧图片渲染（≤ 1.5s）
2. 单独打开大图 → 计时到完整渲染（≤ 500ms，50MB 推算）
3. 每张图测试 5 次，取中位数（R10 应对）
4. 内存占用：使用 ProcMon / atop 监控

### 来源说明

公开图库 + 自制大图（使用 ImageMagick `convert -size 12000x9000 plasma:fractal output.jpg`）。

## 八、#8 巨型目录（1 套，1000+ 文件）

### 用途

- 验证 AC-12 性能（翻看切换 ≤ 300ms 在大数据集下）
- 验证 AC-04a 翻看边界（1000+ 张文件的边界快捷键）
- 验证 M4 翻看性能 + IF-B 大数据集过渡

### 数据明细

| # | 内容 | 数量 | 文件结构 | 总大小 |
|---|---|---|---|---|
| 1 | img0001.jpg ~ img0500.jpg | 500 张 | 800×600 JPEG | ~150MB |
| 2 | img0501.jpg ~ img1000.jpg | 500 张 | 800×600 JPEG | ~150MB |
| **合计** | **1000 张** | **单目录** | **~300MB** |

### AC 溯源

- **AC-04a**（翻看边界）— 1000+ 文件的首/末边界禁用验证
- **AC-12**（性能）— 1000 文件目录下的翻看切换 ≤ 300ms

### 制作方法

```bash
# Linux 批量生成脚本
mkdir large_dir
for i in $(seq -f "%04g" 1 1000); do
  convert -size 800x600 plasma:fractal "large_dir/img${i}.jpg"
done
```

### 性能验证方法

1. 打开 img0500.jpg（在 1000 文件目录中间位置）
2. → → → 连续翻看 → 计时每次切换
3. PageUp 跳到首张 → Home → 计时
4. End 跳到末张 → ← → 计时
5. 每项操作测 5 次取中位数
