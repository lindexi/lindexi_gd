## 一、需求概述

### 1.1 需求背景

作为自研 OS 的上层应用生态建设的一部分，需要一款预装的图片查看器，为普通用户提供日常照片查看能力。这是 OS 应用矩阵中的基础组件之一。

### 1.2 落地目标

交付一款**板正**的图片查看器——规矩、正统、专业、克制。如同 Windows 经典照片查看器给人的感觉：不搞社交分享、不搞云同步、不堆花哨特效，就是认认真真把「看图」这件事做到位。界面简洁（非极简），克制但完整。

### 1.3 业务价值

- 补齐 OS 基础体验闭环，用户拿到系统就能直接双击打开图片
- 作为系统默认图片查看器，降低用户安装第三方工具的需求
- 为后续上层应用生态建立品质标杆

## 二、需求范围

### 2.1 明确包含

| 模块 | 内容 |
|------|------|
| 图片查看 | 打开、展示、缩放、旋转（视图旋转，不保存）、拖拽平移 |
| 目录翻看 | 识别同目录图片，支持上/下一张切换（文件名自然排序，到头即停） |
| 全屏模式 | 进入/退出全屏 |
| 系统集成 | 注册为系统默认图片查看器；右键菜单「用图片查看器打开」（Windows + Linux） |
| 窗口管理 | 标题栏显示「图片名 - 图片查看器」；允许多实例运行；默认大小居中 |
| 图片信息 | 状态栏常驻显示分辨率、文件大小、格式 |
| 格式支持 | 依赖 ImageShape 库，该库支持的格式即为产品支持的格式 |

### 2.2 明确不包含

- ❌ 文件夹浏览器 / 缩略图网格 / 左侧目录树
- ❌ 删除图片功能
- ❌ 编辑功能（裁剪、调色、滤镜等）
- ❌ 社交分享、云同步
- ❌ 命令行参数支持（本期不做）
- ❌ 文件管理器联动（文件管理器尚未开发）
- ❌ macOS 右键菜单集成（本期仅 Windows + Linux）
- ❌ 批量处理
- ❌ 幻灯片自动播放
- ❌ 图片格式转换
- ❌ **任何二次确认对话框**（v1.0 阶段不提供；保留"克制"气质）

## 四、核心功能需求

### 4.1 图片打开

| 规则项 | 说明 |
|--------|------|
| 触发方式 | ① 在文件管理器中双击关联格式图片文件；② 右键图片 →「用图片查看器打开」 |
| 默认显示 | 打开后图片**适应窗口**显示（等比缩放至窗口内完整可见） |
| 窗口大小 | 固定默认大小，居中屏幕。**不记住**上次关闭时的尺寸和位置 |
| 多实例 | **允许多实例**。每次打开图片均创建独立窗口，不替换已有窗口内容；不限制采用多进程或单进程多窗口实现 |
| 标题栏 | 格式：`{图片文件名} - 图片查看器`（例如 `DSC001.jpg - 图片查看器`） |
| 格式兜底 | 不支持的格式弹出提示：「无法打开此文件，不支持的图片格式」 |

> **AC-01a（v1.1 新增）**：切换图片时，缩放比例自动重置为「适应窗口」，不继承上一张的缩放状态。切换时不弹任何"是否保留上一张设置"的提示（即不提供二次确认对话框）。

### 4.2 缩放

| 规则项 | 说明 |
|--------|------|
| 操作方式 | **按住 Ctrl + 鼠标滚轮**缩放 |
| 缩放范围 | **10% ~ 3200%** |
| 缩放基准点 | 以鼠标光标所在位置为中心缩放 |
| 双击行为 | 双击图片切换至 **1:1 实际大小**（100%）显示；再次双击恢复适应窗口 |
| 适应窗口 | 提供按钮/快捷键恢复「适应窗口」模式 |
| 缩放指示 | 缩放比例变化时短暂显示百分比（如右下角浮现「150%」，1.5 秒后消失） |

> 缩放范围 10%~3200%：低于 10% 图片已无法辨识内容，高于 3200% 对于像素图没有实际意义。若 ImageShape 能渲染矢量图（如 SVG），矢量图缩放建议无上限。

### 4.3 旋转

| 规则项 | 说明 |
|--------|------|
| 旋转行为 | **仅视图旋转**，不修改原文件。图片在内存中旋转展示 |
| 操作方式 | 提供顺时针 90° / 逆时针 90° 旋转按钮（含快捷键） |
| 持久性 | 关闭图片即丢失旋转状态，下次打开恢复原方向 |
| 翻看行为 | 旋转后翻到下一张再翻回来，旋转状态**不保留** |
| 保存询问 | **不提供**「是否保存旋转结果」的对话框（详见 §2.2） |

> 仅视图旋转是最安全的选择——不会因误操作损坏用户原图文件。后续如有需求，可增加「另存为」功能来保存旋转后的版本（已纳入 v2.0 候选，见 §八）。

### 4.4 目录翻看

| 规则项 | 说明 |
|--------|------|
| 触发条件 | 打开一张图片后，自动识别同目录下其他图片文件 |
| 排序规则 | **文件名自然升序**（`img1 < img2 < img10`，而非字典序 `img1 < img10 < img2`） |
| 操作方式 | 键盘 ← → 方向键 / 界面按钮 切换上一张/下一张 |
| 边界行为 | 到目录第一张时「上一张」按钮灰掉；到最后一张时「下一张」按钮灰掉。**不循环** |
| 过滤逻辑 | 仅枚举 ImageShape 支持的格式，跳过不支持的格式（静默跳过，不弹提示） |

> **AC-04a（v1.1 新增）**：到达边界时不循环翻看，按钮处于禁用状态，快捷键 `←` `→` `Home` `End` 在边界处不响应（不弹任何提示，不循环）。

### 4.5 全屏模式

| 规则项 | 说明 |
|--------|------|
| 进入全屏 | 快捷键 F11 / 界面按钮 |
| 退出全屏 | ESC / F11 / 界面按钮 |
| 全屏行为 | 图片居中，背景纯黑，隐藏标题栏和菜单栏（若有） |
| 全屏下的操作 | 缩放、旋转、翻看、拖拽平移均可用 |
| 全屏下的 UI | 鼠标移动时短暂显示底部工具栏（2 秒无操作自动隐藏） |

### 4.6 拖拽平移

| 规则项 | 说明 |
|--------|------|
| 触发条件 | 图片放大后超出窗口可视区域时 |
| 操作方式 | 鼠标拖拽平移视图 |
| 光标样式 | 平移时显示抓手光标（Hand） |
| 边界约束 | 拖拽到图片边缘后不再移动 |

> **AC-06a（v1.1 新增）**：全屏模式下，无修饰键的滚轮 = 翻看（上一张/下一张）；`Ctrl+滚轮` = 缩放。两种行为不混叠。非全屏模式下，滚轮统一为翻看（与 4.4 一致）。

### 4.7 图片基本信息

| 规则项 | 说明 |
|--------|------|
| 显示内容 | 分辨率（宽×高）、文件大小、图片格式 |
| 展示方式 | 状态栏常驻显示于窗口底部 |
| 显示格式示例 | `3840 × 2160 ｜ 4.2 MB ｜ JPEG` |

### 4.8 系统集成

| 规则项 | 说明 |
|--------|------|
| 默认查看器 | 安装后注册为关联图片格式的默认打开程序（Windows + Linux） |
| 右键菜单 | 图片文件右键菜单增加「用图片查看器打开」选项（Windows + Linux） |
| 文件关联 | 关联范围为 ImageShape 库支持的所有格式 |

## 五、非功能需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 冷启动速度 | ≤ 1.5 秒（不含系统级开销） |
| 图片加载 | 常规 JPEG/PNG（≤ 20MB）500ms 内完成渲染 |
| 翻看切换 | 上一张/下一张切换延迟 ≤ 300ms（含预加载策略） |
| 缩放响应 | 滚轮缩放帧级响应，无明显卡顿 |
| 内存占用 | 空闲 ≤ 80MB；打开 20MB 图片 ≤ 200MB |

### 5.2 兼容性

| 平台 | 版本要求 |
|------|----------|
| Windows | Windows 10 及以上 |
| Linux | 主流发行版（Ubuntu 20.04+ / Debian 11+ / Fedora 36+） |

#### 5.2.1 兼容性测试最小覆盖矩阵（v1.1 新增）

| 维度 | Windows 最小覆盖 | Linux 最小覆盖 |
|------|------------------|----------------|
| 发行版/版本 | Win10 21H2 / Win11 23H2 | Ubuntu 22.04 LTS / Debian 12 / Fedora 38 |
| 桌面环境 | 默认 + 高对比度模式 | GNOME / KDE Plasma / XFCE |
| 字体回退 | 无微软雅黑时回退 | 无 Noto Sans CJK 时回退 |
| HiDPI 缩放 | 100% / 150% / 200% | 100% / 150% / 200% |
| 文件关联 | 默认关联生效 | MIME 类型注册生效 |
| 右键菜单 | 「用图片查看器打开」可用 | `.desktop` 文件注册可用 |
| 快捷键差异 | `Alt+F4` 退出 | `Ctrl+Q` 退出 |
| 主题 | 跟随系统亮/暗主题 | 跟随系统亮/暗主题 |

### 5.3 安全性

- 不联网，无任何网络请求
- 不修改、不删除用户原文件
- 仅读取用户打开的图片文件
- 不弹出任何形式的"操作确认"对话框（与 §2.2 一致）

### 5.4 易用性

- 所有操作均有键盘快捷键，且符合行业通用习惯
- 界面元素清晰，功能入口直观
- 空状态 / 错误状态有明确提示文案
- 板正气质：无动画炫技、无弹窗广告、无引导教程、无功能堆砌
- **不提供任何二次确认对话框**（v1.1 新增约束，与 §2.2 一致）

### 5.5 国际化

- 本期仅简体中文，但代码层面预留 i18n 扩展能力

## 七、验收标准

| 编号 | 功能 | 验收条件 |
|------|------|----------|
| AC-01 | 图片打开 | 双击 JPEG/PNG/BMP/GIF 文件，窗口默认大小居中打开，图片适应窗口完整显示，标题栏显示「图片名 - 图片查看器」 |
| AC-01a | 切换图片重置缩放 | 切换图片时，缩放比例自动重置为「适应窗口」，不继承上一张的缩放状态，不弹任何提示 |
| AC-02 | 格式兼容 | ImageShape 支持的所有格式均可正常打开；不支持的格式弹出明确提示 |
| AC-03 | Ctrl+滚轮缩放 | 缩放范围 10%~3200%，以光标为中心缩放，无卡顿 |
| AC-04 | 双击 1:1 | 双击图片切换至 100% 实际大小，再次双击恢复适应窗口 |
| AC-04a | 翻看边界停止 | 到达边界时不循环翻看，按钮处于禁用状态，快捷键 `←` `→` `Home` `End` 在边界处不响应 |
| AC-05 | 旋转 | 顺时针/逆时针 90° 旋转，仅视图旋转，关闭后恢复原方向；不弹出"是否保存"对话框 |
| AC-06 | 全屏 | F11 进入/退出全屏，背景纯黑，鼠标移动显示工具栏，2 秒无操作隐藏 |
| AC-06a | 全屏滚轮行为 | 全屏模式下，无修饰键的滚轮 = 翻看；`Ctrl+滚轮` = 缩放。两种行为不混叠 |
| AC-07 | 拖拽平移 | 放大后鼠标拖拽平移，边界不越界，光标为抓手掌 |
| AC-08 | 多实例 | 已有窗口时再打开新图片，创建新的独立窗口，原窗口内容保持不变 |
| AC-09 | 窗口不记忆 | 关闭后重新打开，窗口恢复默认大小居中，不保留上次位置和尺寸 |
| AC-10 | 系统关联 | Windows/Linux 下图片文件默认关联本应用打开，右键菜单包含入口 |
| AC-11 | 图片信息 | 状态栏常驻显示当前图片分辨率、文件大小、格式 |
| AC-12 | 性能 | 冷启动 ≤ 1.5s，20MB JPEG 加载 ≤ 500ms，翻看切换 ≤ 300ms |
| AC-13 | 无二次确认 | v1.0 全流程不弹出任何"操作确认"对话框（与 §2.2 / §5.4 一致） |

> AC 总数：v1.0 为 13 条；v1.1 追加 3 条明确化条款（AC-01a / AC-04a / AC-06a）+ 1 条约束性条款（AC-13），合计 14 条。

## 八、后续迭代建议

| 优先级 | 建议项 | 理由 |
|--------|--------|------|
| 高 | macOS 支持 | Avalonia 具备跨平台能力，补全主流桌面平台覆盖 |
| 高 | 命令行参数 | `image-viewer /path/to/image` 便于与其他工具联动 |
| 中 | 打印功能 | 基础打印是图片查看器的常见延伸 |
| 中 | 另存为/复制到 | 旋转后想保存的用户需求可能会出现（v1.0 暂不提供） |
| 中 | 浅色/深色主题 | 跟随系统或手动切换（v1.0 仅暗色） |
| 低 | 幻灯片自动播放 | 部分用户有需求但非核心 |
| 低 | EXIF 信息查看 | 摄影爱好者可能关注拍摄参数 |

### 8.1 v2.0 快捷键预留位（v1.1 新增）

为避免 v2.0 引入新功能时与现有快捷键冲突，提前预留以下快捷键位（**v1.0 不实现，仅作占位声明**）：

| 快捷键 | 预留功能 | v1.0 状态 |
|--------|---------|-----------|
| `Ctrl+S` | 保存（v2.0「另存为」时启用） | 不实现 |
| `Ctrl+Shift+S` | 另存为 | 不实现 |
| `Delete` | 删除（v2.0「删除」时启用） | 不实现 |
| `Ctrl+X` | 剪切 | 不实现 |
| `Ctrl+N` | 新建窗口（v2.0 多窗口时启用） | 不实现 |
| `Ctrl+T` | 新建标签页（v2.0 标签页时启用） | 不实现 |
| `Ctrl+W` | 关闭标签页 | 不实现 |
| `Ctrl+P` | 打印（v2.0 打印时启用） | 不实现 |

> 上述快捷键在 v1.0 阶段**不响应任何操作**，且不与系统/浏览器全局快捷键冲突。预留目的是让 v2.0 扩展时无需重构 UX 04 文档。
