## 关键设计决策总结

| # | 决策 | 简要理由 |
|---|------|---------|
| VD-1 | 暖暗灰基底（#2C2C2C / #1A1A1A） | 纯黑太冷，暖灰兼容大多数图片色温；图片区更深形成"画框"层次 |
| VD-2 | 强调色 #4A90D9（非 Windows 蓝、非 macOS 蓝） | 跨平台中立；饱和度温和，不刺眼 |
| VD-3 | 纯线性图标、1.5px 描边、16px | 面性图标在暗色下视觉重量过大 |
| VD-4 | 系统原生字体、偏小字号（11-16px） | 零加载成本；小字让 UI 存在感降低 |
| VD-5 | 圆角 0px（容器）/ 4px（按钮）/ 6-8px（浮层/对话框） | 能直就直，必需圆的用小圆角 |
| VD-6 | 不用 spinner，用静态"正在加载..." | 符合"克制"气质；短时加载不需要动画 |
| VD-7 | S8 文件损坏用橙色而非红色 | 语义区分：损坏≠用户错误 |
| VD-8 | 不设计旋转保存对话框 | 遵循需求 §4.3"仅视图旋转，关闭即丢失" |
| VD-9 | 原生系统标题栏 | 板正——窗口应该像窗口；跨平台交给 OS |
| VD-10 | v1.0 仅暗色主题 | 看图场景暗色最优；亮色留给 v2.0 |
