# 需求分析文档

## 1. 项目目标

构建一个基于 WPF 的桌面 AI Bot 聊天工具，支持本地会话管理、消息编辑、聊天持续上下文、基础设置维护，并为后续接入真实模型服务预留清晰扩展点。

## 2. 角色与使用场景

### 2.1 目标用户

- 需要在桌面侧与 AI Bot 进行多轮对话的普通用户。
- 需要调试模型配置、缓存路径和日志路径的开发者或高级用户。

### 2.2 典型场景

1. 用户打开应用，查看历史会话并继续某一轮对话。
2. 用户新建会话，与 AI Bot 进行多轮聊天。
3. 用户发现已发送或已生成的消息需要修正，可选中消息并编辑保存。
4. 用户在模型响应过程中主动停止生成。
5. 用户在设置页修改缓存目录、日志目录与模型名称。

## 3. 功能需求

### 3.1 主界面

主界面至少包含以下能力：

- 查看历史会话列表。
- 新建会话。
- 删除或管理已有会话。
- 选中历史会话后继续聊天。
- 显示当前会话的消息流。
- 发送新消息。
- 在响应过程中执行停止。
- 选中已有消息后进行编辑并保存。
- 展示当前状态提示，例如“正在生成”“已停止”“已保存编辑”。

### 3.2 设置界面

设置界面至少包含以下能力：

- 配置缓存路径。
- 配置日志路径。
- 配置所使用的模型名称。
- 保存设置。
- 恢复默认设置。

## 4. 非功能需求

- 使用 MVVM 组织结构，保证可维护性。
- 聊天记录与设置均需支持本地持久化。
- AI 调用实现需要可替换，允许先接入模拟服务，后续替换真实服务。
- 发送、停止等交互应具备清晰的状态反馈。
- 目录结构应支持后续扩展更多页面或服务。

## 5. 页面划分

### 5.1 主界面内容

- 会话列表区：展示历史会话、更新时间、消息数。
- 聊天展示区：展示消息内容与角色。
- 输入区：输入消息、发送、停止。
- 消息编辑区：编辑当前选中的消息并保存。

### 5.2 设置界面内容

- 缓存路径输入框。
- 日志路径输入框。
- 模型名称输入框。
- 保存与恢复默认按钮。

## 6. 数据对象需求

- `ChatSession`：会话标识、标题、创建时间、更新时间、消息集合。
- `ChatMessage`：消息标识、角色、内容、时间。
- `AppSettings`：缓存路径、日志路径、模型名称。

## 7. 技术实现要求

- 使用 `UserControl` 分离主界面与设置界面内容。
- 使用 `ContentControl + DataTemplate` 或等价方式实现页面承载。
- 通过接口抽象 AI 服务、设置服务、聊天仓储、日志服务。
