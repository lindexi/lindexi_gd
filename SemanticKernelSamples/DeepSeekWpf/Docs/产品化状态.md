# DeepSeekWpf 产品化状态

## 当前已具备

- 依赖 `AgentLib`，由 `CopilotChatManager` 管理运行期会话、流式回复、推理内容、取消和 Token 用量。
- 使用 `AgentApiManagerConfiguration` 配置多个 OpenAI 协议终结点与模型。
- 正式配置文件位于 `%LOCALAPPDATA%\DeepSeekWpf\agent-api.json`，首次运行自动创建模板。
- Debug 构建在正式配置不可用时尝试 `LindexiAgentConfiguration.LoadDefault()`，该回退不参与 Release 构建。
- 应用日志位于设置的日志目录；聊天日志和 XML 历史快照分别写入日志目录与数据目录下的 `ChatHistory`。
- 捕获 WPF UI、后台线程和未观察 Task 异常，并在启动配置失败时给出可见提示。
- 支持 `win-x64`、`win-arm64` 的自包含单文件发布属性。

## 配置流程

1. 启动应用并进入“设置”。
2. 点击“编辑配置文件”。
3. 填写 `PrimaryModel`、终结点、API Key 和模型定义。
4. 点击“校验配置”。
5. 配置通过后重启应用使其生效。

不要将包含真实 API Key 的 `%LOCALAPPDATA%\DeepSeekWpf\agent-api.json` 提交到仓库或打包到发布产物。

## 发布命令

- x64：`dotnet publish -c Release -r win-x64`
- Arm64：`dotnet publish -c Release -r win-arm64`

发布前应替换正式图标、确定产品版本、签名可执行文件，并在干净 Windows 环境执行安装/启动/联网验证。

## 尚未完成的产品化缺口

### P0：发布前必须处理

- **凭据保护**：API Key 当前以明文 JSON 存储。正式产品应接入 Windows Credential Manager、DPAPI 或独立密钥服务，并支持凭据迁移与删除。
- **历史恢复**：AgentLib 当前会写出 XML 历史和 AgentSessionState，但没有公开的会话恢复 API；应用重启后不会自动恢复到 `CopilotChatManager.ChatSessions`。
- **历史删除一致性**：当前删除只影响运行期集合，不删除已写出的日志/历史文件。
- **隐私与错误脱敏**：AgentLib 的异常预设消息可能包含完整异常文本；发布前应统一用户可见错误、日志级别及 API Key/请求头脱敏策略。
- **发布签名与安装器**：尚未提供代码签名、MSIX/安装器、卸载清理和升级策略。

### P1：建议首个正式版本处理

- 模型选择与能力展示、连接测试、超时和重试策略。
- Markdown/代码块渲染、复制、搜索、导出和无障碍支持。
- 多实例写文件协调、历史损坏恢复、数据保留期限与清理机制。
- 网络代理、证书错误、限流、配额、离线状态和服务端错误的分类提示。
- 自动化测试：配置解析、启动失败、发送/取消、UI 绑定、历史写盘和发布烟雾测试。
- 崩溃报告、可选遥测、隐私政策、许可声明和第三方依赖清单。

### P2：持续演进

- 多语言、主题、字体与高 DPI 适配。
- 工具调用审批 UI、附件、多模态和工作区能力。
