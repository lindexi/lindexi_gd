# README

## 开发须知

- 每完成一件事情，都向文档进行记录
- 当文档较多时，重新阅读文档，进行整理
- 避免文档内容陈旧，应该根据代码内容进行更新，保持文档与代码的一致性
- 经验文档不要出现类似“本次更改”这样的描述，因为经验文档是沉淀可复用的方法论、最佳实践和后续项目可参考的经验，应该保持内容的通用性和时效性

## 文档目录

- `过程文件记录/`：记录每个实施步骤的目标、输入、输出和完成情况。
- `经验文档记录/`：沉淀可复用的方法论、最佳实践和后续项目可参考的经验。
