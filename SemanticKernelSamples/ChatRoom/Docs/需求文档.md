# 需求文档

## 1. 项目概述

### 1.1 项目定位

这是一个桌面聊天室应用。多个 AI 角色在同一个聊天室中自动对话，人类可以作为其中一个角色随时插话。每个角色拥有独立的人设、记忆和模型。

### 1.2 参考对象

AutoGen / CrewAI 等框架的 GroupChat 模式。

---

## 2. 核心功能需求

### 2.1 多角色自动对话

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-01 | 支持在聊天室中添加多个 AI 角色，每个角色拥有独立的系统提示词、记忆内容和模型配置 | P0 |
| F-02 | 角色按可配置的发言策略自动轮流发言，默认按固定顺序轮流 | P0 |
| F-03 | 每个角色发言时，只看到自己上次发言之后其他角色或人类产生的增量公开消息，看不到其他角色的内部思考过程 | P0 |
| F-04 | 角色发言通过 AI 模型流式输出 | P0 |
| F-05 | 支持最大对话轮次限制（默认 10 轮） | P1 |
| F-06 | 某个角色发言失败时，生成系统错误消息，不中断整个聊天室循环 | P0 |
| F-07 | 每个聊天室默认内置一个"助手"角色 | P0 |

### 2.2 人类插话

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-10 | 人类可以随时在聊天室中插话，消息直接追加到共享对话历史 | P0 |
| F-11 | 人类插话后自动启动新一轮自动循环，让 AI 角色回复 | P0 |
| F-12 | 人类消息中可以通过 @角色名 提及特定角色，被提及的角色优先回复 | P1 |
| F-13 | 人类插话后，当前正在发言的角色完成本轮回复后暂停；随后"助手"角色立刻回复人类内容，同时所有配置为"始终参与"的角色各回复一次，仅在被 @ 时才发言的角色不参与此轮回复 | P1 |

### 2.3 @Mention 机制

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-20 | 支持 @RoleName 和 @[Role Name] 两种 @mention 格式 | P1 |
| F-21 | 消息追加到聊天室时自动解析 @mention，记录被提及的角色 | P1 |
| F-22 | @mention 触发链式调用：角色 A @ B → B 发言 → B @ C → C 发言 | P1 |
| F-23 | 角色可配置参与模式：始终参与（默认参与轮流）或仅在被 @ 时才发言 | P2 |

### 2.4 角色管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-30 | 通过 UI 添加新角色，配置角色名、系统提示词、模型提供商、模型、记忆内容、是否人类角色 | P0 |
| F-31 | 通过 UI 编辑已有角色的所有属性 | P0 |
| F-32 | 通过 UI 删除角色 | P0 |
| F-33 | AI 可在对话中通过工具动态管理角色（列举/创建/编辑角色） | P2 |
| F-34 | 新建角色默认使用全局配置的模型提供商和首选模型 | P0 |

### 2.5 会话管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-40 | 创建新会话，自动包含助手角色 | P0 |
| F-41 | 列出历史会话（标题、创建时间、角色数、消息数） | P0 |
| F-42 | 打开历史会话，恢复角色定义和消息历史 | P0 |
| F-43 | 删除历史会话 | P0 |
| F-44 | 会话持久化到本地磁盘 | P0 |
| F-45 | 新建会话时关闭当前会话 | P0 |

### 2.6 设置

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-50 | 编辑全局默认模型 | P0 |
| F-51 | 编辑模型提供商列表（提供商名、API 地址、密钥、主模型） | P0 |
| F-52 | 添加/删除模型提供商 | P0 |
| F-53 | 编辑持久化路径、默认最大轮次 | P1 |
| F-54 | 设置变更后保存到配置文件 | P0 |
| F-55 | 设置页返回后，配置变更立即生效，无需重启 | P0 |
| F-56 | 每个提供商下可配置多个模型 | P1 |

### 2.7 模型管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-60 | 支持多个 LLM 提供商，每个提供商有独立的 API 地址和密钥 | P0 |
| F-61 | 所有角色共享同一组已注册的模型提供商 | P0 |
| F-62 | 全局首选模型可在运行时切换 | P0 |
| F-63 | 支持模型轻量标记，用于区分快速模型和完整模型 | P2 |
| F-64 | 新建角色能正确使用已注册的模型提供商 | P0 |

## 3. 界面需求

### 3.1 主界面布局（三栏式）

- **左栏**：历史会话列表，支持新建/打开/删除会话，底部有"设置"按钮
- **中栏**：聊天消息区，底部输入框和发送/停止按钮
- **右栏**：角色列表，支持添加/编辑/删除角色

### 3.2 设置界面

- 顶部：标题 + 返回按钮
- 全局默认模型：下拉选择
- 路径配置：持久化路径、默认最大轮次
- 模型提供商列表：每个提供商可编辑名称、API 地址、密钥、主模型
- 添加/删除提供商按钮

### 3.3 角色编辑界面

- 顶部：标题 + 取消/保存按钮
- 基本信息：角色 ID（只读）、角色名称、是否人类角色
- 系统提示词：多行文本框
- 模型配置：提供商、模型
- 角色记忆：多行文本框（可选）

### 3.4 视觉风格

- 主题色：蓝色系，白色/浅灰背景
- 圆角卡片式布局
- 角色头像：彩色圆形 + 角色名首字
- 消息气泡：角色名 + 时间戳 + 内容
- 状态色：成功/警告/错误/信息
